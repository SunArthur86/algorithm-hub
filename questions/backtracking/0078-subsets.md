# 78. 子集

> 难度：**Medium** | 分类：回溯 | 标签：位运算, 数组, 回溯

> 🔗 力扣链接：[https://leetcode.cn/problems/subsets/](https://leetcode.cn/problems/subsets/)

## 📝 题目描述

给你一个不含重复元素的整数数组 `nums`，返回它的所有子集（幂集）。比如 `nums = [1,2,3]`，结果是 `[[],[1],[1,2],[1,2,3],[1,3],[2],[2,3],[3]]`。

- 题号：78
- 题目名：子集
- 难度：中等
- 题型：回溯
- 为什么选这题：它是回溯法中"收集所有中间状态"的代表作——不只是收集叶子节点，而是收集决策树上每个节点。

## 🧠 思考路径

和组合题很像，但区别在于：组合只收集长度为 `k` 的路径，而子集要收集所有长度的路径。也就是说，递归树上的每一个节点都是一个合法结果。

代码模板几乎一样，唯一的不同是：不再等 `len(path) == k` 才收集，而是每进入一层递归就先把当前 `path` 加进结果。

同样可以用"选或不选"的视角理解：对于每个元素，要么选它加入子集，要么不选。这形成了一棵二叉决策树，所有从根到叶子的路径就对应所有子集。

## 💻 Java 题解

### 解法 1：回溯（选/不选）

从index开始，每个元素都可以选或不选。每次进入递归先收集当前子集。

- ⏱ 时间复杂度：`O(2ⁿ×n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public List<List<Integer>> subsets(int[] nums) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(nums, 0, new ArrayList<>(), result);
        return result;
    }
    private void backtrack(int[] nums, int start, List<Integer> path, List<List<Integer>> result) {
        result.add(new ArrayList<>(path)); // 每步都收集
        for (int i = start; i < nums.length; i++) {
            path.add(nums[i]);
            backtrack(nums, i + 1, path, result);
            path.remove(path.size() - 1);
        }
    }
}
```

**关键点：**
- 每次进入都收集（包括空集）
- for循环从start开始避免重复
- 不需要终止条件——遍历完自然结束

**执行步骤：**
1. 收集当前path
1. 从start遍历选下一个元素
1. 递归+回溯


## ⚠️ 易错点

- 收集结果要在循环之前，不是之后。空集 `[]` 也是合法子集，必须收集。
- 收集时要做深拷贝（`path[:]` 或 `new ArrayList<>(path)`），否则后续的 `pop` 会修改已收集的结果。
- `start` 参数保证元素的选取顺序是递增的，避免出现 `[2,1]` 和 `[1,2]` 的重复。

- 和全排列不同——for从start开始不是0

## 📊 复杂度分析

- 时间复杂度：`O(n × 2ⁿ)`，共有 `2ⁿ` 个子集，每个子集平均 `O(n)` 时间拷贝。
- 空间复杂度：`O(n)`，递归栈深度最多 `n`。

# 131. 分割回文串

> 难度：**Medium** | 分类：回溯 | 标签：字符串, 动态规划, 回溯

> 🔗 力扣链接：[https://leetcode.cn/problems/palindrome-partitioning/](https://leetcode.cn/problems/palindrome-partitioning/)

## 📝 题目描述

给你一个字符串 `s`，请把它分成若干子串，使得每个子串都是回文串。返回所有可能的分割方案。

比如 `"aab"` 可以分成 `[["a","a","b"],["aa","b"]]`。重点不是"能不能判断回文"，而是"怎么系统地枚举出所有合法的切法"。

- 题号：131
- 题目名：分割回文串
- 难度：中等
- 题型：回溯
- 为什么选这题：它是回溯法的经典入门题，能帮你建立"做选择 → 走到底 → 退回来换一条路"的核心直觉。

## 🧠 思考路径

最直觉的想法：从第一个字符开始，尝试不同长度的切割。切出一段回文子串后，对剩下的部分继续切；如果某一段不是回文，就跳过这种切法。

这个过程天然是一棵决策树：每个位置决定"在哪里切一刀"。树的每一层是当前切分的位置，每条路径是一种完整的分割方案。

关键在于：走到某一步时，如果当前的切法已经让某段不是回文了，后面的尝试毫无意义——直接回头。这就是"回溯"的精髓：**发现此路不通，立刻掉头，不继续浪费**。

所以这题的关键词不是"切字符串"，而是"在决策树上做深度优先搜索，遇到非法状态就剪枝回退"。

## 💻 Java 题解

### 解法 1：回溯 + 回文判断

从start遍历end，如果s[start..end]是回文，递归分割end+1后面的部分。

- ⏱ 时间复杂度：`O(n×2ⁿ)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public List<List<String>> partition(String s) {
        List<List<String>> result = new ArrayList<>();
        backtrack(s, 0, new ArrayList<>(), result);
        return result;
    }
    private void backtrack(String s, int start, List<String> path, List<List<String>> result) {
        if (start == s.length()) { result.add(new ArrayList<>(path)); return; }
        for (int end = start; end < s.length(); end++) {
            if (isPalindrome(s, start, end)) {
                path.add(s.substring(start, end + 1));
                backtrack(s, end + 1, path, result);
                path.remove(path.size() - 1);
            }
        }
    }
    private boolean isPalindrome(String s, int l, int r) {
        while (l < r) if (s.charAt(l++) != s.charAt(r--)) return false;
        return true;
    }
}
```

**关键点：**
- 遍历所有可能的分割点
- 只对回文子串递归
- start==s.length() 收集结果

**执行步骤：**
1. 从start遍历end
1. 判断s[start..end]是否回文
1. 是→递归分割剩余
1. 回溯


## ⚠️ 易错点

- 回溯后别忘了 `pop` 撤销选择，否则路径会越攒越多。
- 回文判断可以用 `s == s[::-1]` 快速实现，也可以预处理一个二维 DP 表来加速重复查询。
- 子串范围是 `s[start:end+1]`，注意右端点是闭区间。
- 收集结果时要拷贝一份 `path[:]`，不能直接加入引用，否则后续 pop 会影响已存结果。

- substring(start, end+1) 注意+1

- 只对回文子串才继续分割

## 📊 复杂度分析

- 时间复杂度：`O(n · 2ⁿ)`。最坏情况下每个位置都切或不切，有 `2ⁿ` 种分割方案，每种方案需要 `O(n)` 判断回文。
- 空间复杂度：`O(n)`。递归深度最多 `n` 层，`path` 最多存 `n` 个子串。

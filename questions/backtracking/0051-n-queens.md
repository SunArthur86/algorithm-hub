# 51. N 皇后

> 难度：**Hard** | 分类：回溯 | 标签：数组, 回溯

> 🔗 力扣链接：[https://leetcode.cn/problems/n-queens/](https://leetcode.cn/problems/n-queens/)

## 📝 题目描述

给你一个 `n × n` 的棋盘，要往上面放 `n` 个皇后。规则很简单：任意两个皇后不能在同一行、同一列、同一条对角线上。请你找出所有合法的摆法，每种摆法用字符串数组表示（`'Q'` 表示皇后，`'.'` 表示空位）。

- 题号：51
- 题目名：N 皇后
- 难度：困难
- 题型：回溯
- 为什么选这题：它是回溯算法的"教科书题"，把"逐行做选择、冲突就撤回"的思想体现得淋漓尽致。

## 🧠 思考路径

最朴素的想法：把皇后一个一个放到棋盘上，每种位置组合都试一遍。但 `n × n` 个格子选 `n` 个，组合数爆炸，根本不现实。

观察规则：每行只能放一个皇后（否则同行冲突）。所以可以逐行放置——第 0 行放一个、第 1 行放一个……放完 `n` 行就得到一个解。每行只需要选"放在第几列"。

再进一步：放某行的皇后时，怎么快速判断是否和已有皇后冲突？列冲突用一个集合记录已占列；主对角线（左上到右下）的特征是 `行号 - 列号` 相同；副对角线（右上到左下）的特征是 `行号 + 列号` 相同。三个集合就能 O(1) 判断冲突。

如果某一行所有列都冲突，说明前面选错了，需要"回退"到上一行换个位置——这就是回溯。

## 💻 Java 题解

### 解法 1：回溯 + 列/对角线标记

逐行放置皇后。用colUsed、diag1、diag2三个数组检查冲突。O(n)检查。

- ⏱ 时间复杂度：`O(n!)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public List<List<String>> solveNQueens(int n) {
        List<List<String>> result = new ArrayList<>();
        boolean[] col = new boolean[n], d1 = new boolean[2*n], d2 = new boolean[2*n];
        char[][] board = new char[n][n];
        for (char[] row : board) Arrays.fill(row, '.');
        solve(n, 0, col, d1, d2, board, result);
        return result;
    }
    private void solve(int n, int row, boolean[] col, boolean[] d1, boolean[] d2,
                        char[][] board, List<List<String>> result) {
        if (row == n) {
            List<String> solution = new ArrayList<>();
            for (char[] r : board) solution.add(new String(r));
            result.add(solution);
            return;
        }
        for (int c = 0; c < n; c++) {
            int diag1 = row + c, diag2 = row - c + n - 1;
            if (col[c] || d1[diag1] || d2[diag2]) continue;
            board[row][c] = 'Q'; col[c] = true; d1[diag1] = true; d2[diag2] = true;
            solve(n, row + 1, col, d1, d2, board, result);
            board[row][c] = '.'; col[c] = false; d1[diag1] = false; d2[diag2] = false;
        }
    }
}
```

**关键点：**
- 逐行放置
- 3个数组标记列和对角线
- 对角线索引: row+c 和 row-c+n-1

**执行步骤：**
1. 逐行遍历每列
1. 检查列和对角线冲突
1. 放置+标记
1. 递归下一行
1. 回溯


## ⚠️ 易错点

- 对角线冲突判断：主对角线用 `row - col`，副对角线用 `row + col`，不能搞反。搞反了某些冲突就检测不到。
- 回溯时必须撤销选择：把列、两条对角线从集合中移除，否则后面的分支会被错误地判定为冲突。
- 生成棋盘时，每一行都要新建一个字符数组来填充，不能复用同一个对象。

- 对角线索引公式：d1=row+c, d2=row-c+n-1

- char数组转String构建棋盘

## 📊 复杂度分析

- 时间复杂度：`O(n!)`。第一行有 `n` 个选择，第二行最多 `n-1` 个，以此类推。实际运行因剪枝远小于 `n!`。
- 空间复杂度：`O(n)`。递归栈深度为 `n`，三个集合最多各存 `n` 个元素。

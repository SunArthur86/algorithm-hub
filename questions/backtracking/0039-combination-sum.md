# 39. 组合总和

> 难度：**Medium** | 分类：回溯 | 标签：数组, 回溯

> 🔗 力扣链接：[https://leetcode.cn/problems/combination-sum/](https://leetcode.cn/problems/combination-sum/)

## 📝 题目描述

给你一个没有重复元素的整数数组 `candidates` 和一个目标值 `target`。从 `candidates` 中选出一些数（每个数可以选多次），让它们的和正好等于 `target`。返回所有不重复的组合。

组合里的数字可以按任意顺序排列，但两个组合不能完全一样（顺序不同算同一个）。

- 题号：39
- 题目名：组合总和
- 难度：中等
- 题型：回溯
- 为什么选这题：它是回溯搜索组合的入门经典题——每个数字可以无限次使用，需要用"起始下标"来避免重复组合，理解了这个技巧，后面一堆组合题都能解。

## 🧠 思考路径

想到"所有组合"，自然想到回溯。问题是：怎么避免生成重复的组合？

比如 `candidates = [2, 3, 6, 7]`，`target = 7`。如果先选了 3 再选 2，和先选了 2 再选 3，结果都是 `[2, 3]`，算同一个组合。

解决办法是：每次递归时，只从当前下标 `start` 开始往后的元素里选（包括 `start` 本身，因为可以重复使用）。这样保证了组合里的元素是按 `candidates` 中的顺序出现的，不会产生逆序的重复。

具体来说，在每一层递归里，从 `start` 开始遍历 `candidates`。对于每个候选数字 `candidates[i]`，如果加上它之后总和不超过 `target`，就把它加入路径，然后从 `i`（不是 `i+1`）继续递归——因为可以重复选。回溯时把它从路径中移除。

## 💻 Java 题解

### 解法 1：回溯（可重复选取）

从start开始选，选了后递归还是从start（不是start+1）因为可以重复。减去当前值，等于0时收集。

- ⏱ 时间复杂度：`O(n^(T/M))`
- 💾 空间复杂度：`O(T/M)`

```java
class Solution {
    public List<List<Integer>> combinationSum(int[] candidates, int target) {
        List<List<Integer>> result = new ArrayList<>();
        backtrack(candidates, target, 0, new ArrayList<>(), result);
        return result;
    }
    private void backtrack(int[] candidates, int remain, int start, List<Integer> path, List<List<Integer>> result) {
        if (remain == 0) { result.add(new ArrayList<>(path)); return; }
        if (remain < 0) return;
        for (int i = start; i < candidates.length; i++) {
            path.add(candidates[i]);
            backtrack(candidates, remain - candidates[i], i, path, result); // i 不是 i+1
            path.remove(path.size() - 1);
        }
    }
}
```

**关键点：**
- 递归传 i 不是 i+1（可重复选）
- remain<0 剪枝
- 排序后可更早剪枝

**执行步骤：**
1. 遍历从start开始
1. 减去当前值
1. 递归（还是从i）
1. 回溯


## ⚠️ 易错点

- 递归调用时 `start` 传的是 `i` 而不是 `i + 1`，因为同一个数字可以重复使用。如果传 `i + 1`，每个数字就只能用一次了。
- 加入结果时要拷贝 `path`（`path[:]` 或 `new ArrayList<>(path)`），不能直接加引用，否则后续回溯会修改结果里的列表。
- 剪枝条件 `remain < 0` 可以优化为在 `for` 循环里直接跳过 `candidates[i] > remain` 的情况，减少无效递归。

- 递归传 i 不是 i+1——允许重复选同一元素

## 📊 复杂度分析

- 时间复杂度：`O(S)`，其中 `S` 是所有可行解的长度之和。最坏情况下没有精确的紧界，但通常认为是指数级。
- 空间复杂度：`O(target / min(candidates))`，递归栈的最大深度。如果最小元素是 1，最深可以是 `target`。

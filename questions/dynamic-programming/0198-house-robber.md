# 198. 打家劫舍

> 难度：**Medium** | 分类：动态规划 | 标签：数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/house-robber/](https://leetcode.cn/problems/house-robber/)

## 📝 题目描述

给你一个数组 `nums`，`nums[i]` 代表第 i 家的金额。你不能同时抢相邻的两家。求能抢到的最大总金额。

- 题号：198
- 题目名：打家劫舍
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是最经典的"一维 DP、两个选择"问题——每个位置只有"抢"或"不抢"两个动作，是所有 DP 入门者必须掌握的模板。

## 🧠 思考路径

站在第 i 家门口，有两个选择：

- 抢这家：那第 i-1 家不能抢，最大金额是"前 i-2 家的最大金额 + 当前金额"。
- 不抢这家：最大金额就是"前 i-1 家的最大金额"。

所以 `dp[i] = max(dp[i-1], dp[i-2] + nums[i])`。

初始化：`dp[0] = nums[0]`，`dp[1] = max(nums[0], nums[1])`。

空间优化：只需要两个变量 `prev2` 和 `prev1` 滚动更新。

## 💻 Java 题解

### 解法 1：动态规划（空间优化）

dp[i]=max(不偷=dp[i-1], 偷=dp[i-2]+nums[i])。两个变量滚动。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int rob(int[] nums) {
        if (nums.length == 1) return nums[0];
        int prev2 = nums[0], prev1 = Math.max(nums[0], nums[1]);
        for (int i = 2; i < nums.length; i++) {
            int curr = Math.max(prev1, prev2 + nums[i]);
            prev2 = prev1; prev1 = curr;
        }
        return prev1;
    }
}
```

**关键点：**
- dp[i]=max(dp[i-1], dp[i-2]+nums[i])
- 偷或不偷两种选择
- 两个变量滚动

**执行步骤：**
1. base case: dp[0]=nums[0], dp[1]=max(nums[0],nums[1])
1. 递推
1. 返回dp[n-1]


## ⚠️ 易错点

- 数组只有 1 个元素时要特殊处理，否则 `nums[1]` 会越界。
- `prev2` 代表"前 i-2 家的最大值"，`prev1` 代表"前 i-1 家的最大值"，滚动时先更新 `prev2` 再更新 `prev1`。
- 不能同时抢相邻两家，但可以隔一家抢。

- n=1直接返回nums[0]

## 📊 复杂度分析

- 时间复杂度：`O(n)`，遍历一次。
- 空间复杂度：`O(1)`，只用了两个变量。

## 📖 形象类比

可以类比成《水浒传》里时迁（鼓上蚤）夜里偷盗。

时迁沿着一条街挨家挨户偷，但有个规矩：不能连续偷两家（太容易被发现）。每到一家门口，他要想：是偷这家（那上一家不能偷，只能用上上家的成果），还是跳过这家（保留上一家的成果）？他每次都选收益更大的方案。

对应关系是：

- "每家的财产"对应 `nums[i]`
- "偷这家"对应 `prev2 + nums[i]`
- "跳过这家"对应 `prev1`
- "选收益更大的"对应 `max(prev1, prev2 + nums[i])`

这段记忆的关键是：每个位置只有两个选择，取较大的那个。

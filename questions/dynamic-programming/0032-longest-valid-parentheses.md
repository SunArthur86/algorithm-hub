# 32. 最长有效括号

> 难度：**Hard** | 分类：动态规划 | 标签：栈, 字符串, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/longest-valid-parentheses/](https://leetcode.cn/problems/longest-valid-parentheses/)

## 📝 题目描述

给你一个只包含 `(` 和 `)` 的字符串。你要找出其中最长的、括号完全匹配的子串，返回它的长度。

比如 `")()())"` 中，最长的有效括号子串是 `"()()"`，长度为 4。

- 题号：32
- 题目名：最长有效括号
- 难度：困难
- 题型：栈
- 为什么选这题：它是括号匹配问题的进阶版，从"判断是否合法"升级到"找最长的合法子串"，栈和动态规划两种思路都能解，是理解栈维护"区间边界"的绝佳题目。

## 🧠 思考路径

最朴素的想法是枚举所有子串，逐个判断是否合法。这显然是 `O(n³)`，不行。

想一个更好的办法。括号匹配天然适合用栈——遇到 `(` 就入栈，遇到 `)` 就看栈顶有没有 `(` 来配对。但光判断合法不够，我们还需要知道"连续合法的长度"。

关键观察：如果用栈来存字符本身，弹出配对之后，栈空了就不知道这段连续合法有多长了。但如果栈里存的是下标呢？

栈里存下标。初始时把 `-1` 入栈作为"基准线"。遍历字符串，遇到 `(` 就把下标入栈；遇到 `)` 就先弹出栈顶。弹出之后如果栈不空，当前下标减去栈顶就是一段有效长度；如果栈空了，说明当前的 `)` 没人配对，就把它的下标作为新的基准线入栈。

这样，栈顶始终是"最后一个没有被匹配的右括号的下标"（或初始的 -1），它就是当前有效区间的左边界。

## 💻 Java 题解

### 解法 1：动态规划

s[i]=')'时：若s[i-1]='('则dp[i]=dp[i-2]+2。若s[i-1]=')'且s[i-dp[i-1]-1]='('则dp[i]=dp[i-1]+2+dp[i-dp[i-1]-2]。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int longestValidParentheses(String s) {
        int maxLen = 0;
        int[] dp = new int[s.length()];
        for (int i = 1; i < s.length(); i++) {
            if (s.charAt(i) == ')') {
                if (s.charAt(i - 1) == '(') {
                    dp[i] = (i >= 2 ? dp[i - 2] : 0) + 2;
                } else if (i - dp[i - 1] - 1 >= 0 && s.charAt(i - dp[i - 1] - 1) == '(') {
                    dp[i] = dp[i - 1] + 2 + (i - dp[i - 1] - 2 >= 0 ? dp[i - dp[i - 1] - 2] : 0);
                }
                maxLen = Math.max(maxLen, dp[i]);
            }
        }
        return maxLen;
    }
}
```

**关键点：**
- 两种情况：()和))
- 情况2：检查匹配的( 位置
- 加上匹配( 之前的dp值

**执行步骤：**
1. s[i]==)时
1. s[i-1]==( → dp[i]=dp[i-2]+2
1. s[i-1]==) → 检查匹配位置


## ⚠️ 易错点

- 栈里存的是下标，不是字符。初始哨兵 `-1` 不能忘，否则第一个 `()` 的长度算不出来。
- 遇到 `)` 时要先弹出，再判断栈是否为空。不是先判断再弹出。
- 栈空时要把当前 `)` 的下标压入，作为新一轮的"左边界"，不能直接跳过。

- 边界检查：i-dp[i-1]-1>=0

- 加上匹配位置之前的dp值

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个字符最多入栈、出栈各一次。
- 空间复杂度：`O(n)`。最坏情况下栈里存了所有 `(` 的下标。

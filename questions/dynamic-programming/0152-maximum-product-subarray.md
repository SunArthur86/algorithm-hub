# 152. 乘积最大子数组

> 难度：**Medium** | 分类：动态规划 | 标签：数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/maximum-product-subarray/](https://leetcode.cn/problems/maximum-product-subarray/)

## 📝 题目描述

给你一个整数数组 `nums`，找一个连续子数组，让这个子数组里所有元素的乘积最大，返回这个最大乘积。

重点不在"乘法怎么算"，而在"负数会翻转最大最小，怎么同时维护两个状态"。

- 题号：152
- 题目名：乘积最大子数组
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是最大子数组和的进阶版，乘法引入了"负负得正"的问题，状态必须同时跟踪最大和最小。

## 🧠 思考路径

如果只有正数，这题和最大子数组和完全一样——每一步要么接着乘，要么重新开始。但负数搞乱了这件事：一个很大的负数乘上另一个负数，可能变成一个更大的正数。

所以我们必须同时记住当前位置的"最大乘积"和"最小乘积"。当遇到负数时，最大和最小会互换角色。状态转移是：

- `maxDP[i] = max(nums[i], maxDP[i-1] * nums[i], minDP[i-1] * nums[i])`
- `minDP[i] = min(nums[i], maxDP[i-1] * nums[i], minDP[i-1] * nums[i])`

每一步更新两个值，全局取最大的那个 `maxDP`。

所以这题的关键词不是"乘积"，而是"负数让最大变最小，必须两手都抓"。

## 💻 Java 题解

### 解法 1：动态规划（双状态）

维护maxProd和minProd。负数时交换。maxProd=max(num, maxProd*num)，minProd=min(num, minProd*num)。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int maxProduct(int[] nums) {
        int maxProd = nums[0], minProd = nums[0], result = nums[0];
        for (int i = 1; i < nums.length; i++) {
            int num = nums[i];
            if (num < 0) { int t = maxProd; maxProd = minProd; minProd = t; } // 负数交换
            maxProd = Math.max(num, maxProd * num);
            minProd = Math.min(num, minProd * num);
            result = Math.max(result, maxProd);
        }
        return result;
    }
}
```

**关键点：**
- 同时维护最大积和最小积
- 负数时交换max和min
- 更新result

**执行步骤：**
1. 遇负数交换max和min
1. 更新maxProd和minProd
1. 更新全局最大值


## ⚠️ 易错点

- 遇到负数时必须先交换再计算，否则交换后用的是已更新的值。
- 空间可以优化到 O(1)，只需要两个变量，不需要整个 DP 数组。
- 不要忘记 `nums[i]` 本身也可能成为新的起点——比如前面乘积很小，不如从当前重新开始。

- 负数会让最小变最大——必须交换

- 先交换再计算

## 📊 复杂度分析

- 时间复杂度：`O(n)`。遍历一次数组。
- 空间复杂度：`O(1)`。只用了常数个变量。

# 139. 单词拆分

> 难度：**Medium** | 分类：动态规划 | 标签：数组, 哈希表, 字符串, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/word-break/](https://leetcode.cn/problems/word-break/)

## 📝 题目描述

给你一个字符串 `s` 和一个字典 `wordDict`（字符串列表）。判断 `s` 能否被拆分成若干个字典中出现的单词。单词可以重复使用。

比如 `s = "leetcode"`, `wordDict = ["leet", "code"]`，返回 `True`。重点不在"拆出所有方案"，而在"能不能拆"。

- 题号：139
- 题目名：单词拆分
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是子串匹配类 DP 的代表题，"能否拼出来"的判断模式在很多场景都会出现。

## 🧠 思考路径

最直觉的想法是递归：从 `s` 开头尝试匹配字典里的每个单词，匹配成功就对剩余部分递归判断。但这样会有大量重复计算——比如 `"aaaaab"` 匹配 `"a"` 时，`"aaaab"` 会被反复计算。

加上记忆化就能变成 `O(n²)`，但 DP 更简洁。

设 `dp[i]` 表示 `s[0..i-1]`（前 `i` 个字符）能否被拆分。对于每个位置 `i`，枚举所有 `j < i`：如果 `dp[j]` 为 `True`（前 `j` 个字符能拆分），且 `s[j..i-1]` 在字典里，那 `dp[i] = True`。

初始条件 `dp[0] = True`（空字符串一定能拆分）。

所以这题的关键词不是"拆字符串"，而是"用 DP 记录每个前缀是否可拆，逐步推出整串"。

## 💻 Java 题解

### 解法 1：动态规划

dp[i]=true表示s[0..i)可拆分。对每个i，检查是否存在单词w使得s[i-w.len..i)==w 且 dp[i-w.len]==true。

- ⏱ 时间复杂度：`O(n×m)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public boolean wordBreak(String s, List<String> wordDict) {
        Set<String> dict = new HashSet<>(wordDict);
        boolean[] dp = new boolean[s.length() + 1];
        dp[0] = true;
        for (int i = 1; i <= s.length(); i++)
            for (int j = 0; j < i; j++)
                if (dp[j] && dict.contains(s.substring(j, i))) {
                    dp[i] = true; break;
                }
        return dp[s.length()];
    }
}
```

**关键点：**
- dp[0]=true 空串可拆分
- 双重循环i和j
- substring(j,i)

**执行步骤：**
1. dp[0]=true
1. 对每个i枚举j
1. dp[j]==true且s[j..i)在字典中→dp[i]=true


## ⚠️ 易错点

- `dp` 数组长度是 `n + 1`，不是 `n`，因为 `dp[0]` 对应空串。
- `s[j:i]` 是左闭右开，对应 `s` 的第 `j` 到第 `i-1` 个字符。
- 字典要转成 `set`，否则每次 `in` 查询是 `O(m)`（m 是字典大小），总复杂度会变成 `O(n² · m)`。
- `dp[0] = True` 是边界条件，不能漏，否则后面全为 `False`。

- dp[0]=true 是关键base case

## 📊 复杂度分析

- 时间复杂度：`O(n²)`。两层循环，每次子串查询 `O(1)`（set）。
- 空间复杂度：`O(n + m)`。`dp` 数组 `O(n)`，字典集合 `O(m)`。

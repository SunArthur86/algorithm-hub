# 48. 旋转图像

> 难度：**Medium** | 分类：矩阵 | 标签：数组, 数学, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/rotate-image/](https://leetcode.cn/problems/rotate-image/)

## 📝 题目描述

给你一个 `n × n` 的二维矩阵，要求你把它顺时针旋转 90 度。必须原地修改，不能另开一个矩阵。

比如把 `[[1,2,3],[4,5,6],[7,8,9]]` 旋转后变成 `[[7,4,1],[8,5,2],[9,6,3]]`。

- 题号：48
- 题目名：旋转图像
- 难度：中等
- 题型：矩阵
- 为什么选这题：它是矩阵操作的经典题，"转置 + 翻转"的组合技巧非常优雅，面试中出现频率极高。

## 🧠 思考路径

最朴素的想法是另开一个矩阵，把每个元素从旧位置搬到新位置。新位置的规律是：原来在 `(i, j)` 的元素，旋转后到 `(j, n-1-i)`。但这需要额外空间。

能不能原地做？有两种思路：

**思路一：先转置，再水平翻转。** 矩阵转置（沿主对角线翻转）后，再沿垂直中线左右翻转，等价于顺时针旋转 90 度。比如 `(i, j)` 转置变成 `(j, i)`，翻转变成 `(j, n-1-i)`，正好就是旋转后的位置。每步都是简单的两两交换，原地完成。

**思路二：四元素一轮换。** 每次取四个"互为旋转对应"的元素进行循环交换：`(i,j) → (j,n-1-i) → (n-1-i,n-1-j) → (n-1-j,i) → (i,j)`。只需遍历四分之一矩阵即可完成全部旋转。

两种方法都行，思路一更直观，思路二更一步到位。这里选用思路一。

## 💻 Java 题解

### 解法 1：转置 + 翻转

先沿主对角线转置（matrix[i][j]↔matrix[j][i]），再对每行做水平翻转。

- ⏱ 时间复杂度：`O(n²)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public void rotate(int[][] matrix) {
        int n = matrix.length;
        // 转置
        for (int i = 0; i < n; i++)
            for (int j = i + 1; j < n; j++) {
                int t = matrix[i][j]; matrix[i][j] = matrix[j][i]; matrix[j][i] = t;
            }
        // 水平翻转每行
        for (int i = 0; i < n; i++) {
            int l = 0, r = n - 1;
            while (l < r) {
                int t = matrix[i][l]; matrix[i][l] = matrix[i][r]; matrix[i][r] = t;
                l++; r--;
            }
        }
    }
}
```

**关键点：**
- 转置：沿主对角线交换
- 翻转：每行左右翻转
- 两步组合等价于旋转90°

**执行步骤：**
1. 转置：matrix[i][j]↔matrix[j][i]
1. 每行水平翻转
1. 验证：(i,j)→(j,i)→(j,n-1-i)


## ⚠️ 易错点

- 转置时内层循环从 `i + 1` 开始，不是从 `0` 开始。如果从 `0` 开始，会交换两次等于没交换。
- 如果要求逆时针旋转 90 度，则是"先转置，再上下翻转"（翻转行而非列），不要搞混方向。
- Java 翻转行时不能直接 `Arrays.reverse()`（Java 数组没有这个方法），需要手动双指针交换。
- Python 的 `matrix[i].reverse()` 是原地操作，不需要赋值回去。

- 转置时 j 从 i+1 开始防止交换两次

- 翻转是对每行操作不是整体

## 📊 复杂度分析

- 时间复杂度：`O(n²)`。转置遍历 `n²/2` 次，翻转遍历 `n²/2` 次，总计 `O(n²)`。
- 空间复杂度：`O(1)`。原地操作，只用了一个临时变量。

## 🏢 实际业务应用

- **图像处理**：用户在手机上旋转照片时，图片的像素矩阵需要原地旋转。90 度、180 度、270 度旋转是最基本的图像变换操作，JPEG 解码后的像素操作就是矩阵旋转。
- **游戏开发**：俄罗斯方块中方块的旋转、棋盘游戏中棋盘的视角切换，本质都是矩阵旋转。原地旋转可以避免频繁创建新数组带来的内存压力。
- **地图渲染**：导航 App 切换地图朝向（北朝上、车头朝上）时，瓦片地图的二维数组需要旋转后重新渲染。
- **电子表格**：Excel 中"转置粘贴"功能把行变列、列变行，就是矩阵转置操作；而旋转则是转置加翻转的组合。

- **图像处理**：用户在手机上旋转照片时，图片的像素矩阵需要原地旋转。90 度、180 度、270 度旋转是最基本的图像变换操作，JPEG 解码后的像素操作就是矩阵旋转。
- **游戏开发**：俄罗斯方块中方块的旋转、棋盘游戏中棋盘的视角切换，本质都是矩阵旋转。原地旋转可以避免频繁创建新数组带来的内存压力。

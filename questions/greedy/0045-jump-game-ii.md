# 45. 跳跃游戏 II

> 难度：**Medium** | 分类：贪心算法 | 标签：贪心, 数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/jump-game-ii/](https://leetcode.cn/problems/jump-game-ii/)

## 📝 题目描述

给你一个非负整数数组，每个元素表示你在那个位置最多能跳几步。你从第一个位置出发，目标是到达最后一个位置。问最少需要跳几次。

比如 `[2,3,1,1,4]`：从下标 0 跳一步到下标 1（跳了 1 步），再从下标 1 跳三步到下标 4（跳了 2 步），总共 2 步。

- 题号：45
- 题目名：跳跃游戏 II
- 难度：中等
- 题型：贪心
- 为什么选这题：它是贪心策略最优雅的体现——每一步都选"当前能到的最远位置"，不需要回溯，也不需要 DP。

## 🧠 思考路径

最朴素的想法是 BFS 或 DP：把每个位置能跳到的所有位置都标记下来，层层扩展，直到到达终点。能做，但时间或空间开销大。

贪心的思路更巧妙。想象你在跑道上跑步，每一跳有一个"覆盖范围"。在当前覆盖范围内，你不需要急着跳——可以先观察范围内的所有位置，看哪个位置能让你"下一步跳得更远"。等到走到了当前覆盖范围的边界，就不得不跳了（步数加一），同时把新的覆盖范围更新为之前观察到的最远位置。

这样，整个过程只需要维护两个值：当前跳跃的边界 `end` 和能到达的最远位置 `farthest`。每到边界就跳一步，更新边界。一遍扫过去，`O(n)` 搞定。

## 💻 Java 题解

### 解法 1：贪心（BFS层序）

维护当前跳跃边界currEnd。遍历到边界时必须跳，步数+1，新边界=currFarthest。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int jump(int[] nums) {
        int jumps = 0, currEnd = 0, currFarthest = 0;
        for (int i = 0; i < nums.length - 1; i++) {
            currFarthest = Math.max(currFarthest, i + nums[i]);
            if (i == currEnd) { jumps++; currEnd = currFarthest; }
        }
        return jumps;
    }
}
```

**关键点：**
- 遍历到n-1（最后一个不用跳）
- i==currEnd 时跳一步
- 新边界=currFarthest

**执行步骤：**
1. 遍历更新最远
1. 到达边界时跳跃
1. 更新边界


## ⚠️ 易错点

- 循环范围是 `range(n - 1)` 而不是 `range(n)`。到了最后一个位置就不需要再跳了，如果循环到最后一个元素，可能会多算一步。
- `farthest` 要在每次循环中都更新，不要只在 `i == end` 时才更新。在到达边界之前，我们要持续追踪范围内最远能到哪里。
- 不要把本题和"跳跃游戏 I"搞混：I 是判断能否到达终点，II 是求最少步数。解法不同。
- 初始时 `end` 和 `farthest` 都是 0，如果数组长度为 1，循环不执行，返回 0 步（已在终点）。

- 遍历到n-2不是n-1——最后位置不用再跳

## 📊 复杂度分析

- 时间复杂度：`O(n)`。只遍历一次数组，每个位置只处理一次。
- 空间复杂度：`O(1)`。只用了三个整数变量。

## 🏢 实际业务应用

- **网络路由优化**：数据包在网络中经过多个路由器转发，每个路由器的"跳数"有限制（类似每步能跳多远）。路由协议需要找到跳数最少的路径到达目标网络。BGP 协议中的 AS-PATH 长度优化就是类似的贪心策略。
- **物流中转规划**：快递从仓库出发，经过多个中转站到达目的地。每个中转站有一定的配送范围。系统需要计算最少的中转次数，模型和本题完全一致。
- **游戏关卡设计**：平台跳跃游戏中，玩家在平台间跳跃前进，每个平台的弹跳力决定能跳多远。关卡设计师需要验证从起点到终点是否可达，以及最少需要几跳。
- **加油站选择问题**：汽车沿公路行驶，油箱容量决定了每次加满油能行驶的最大距离。如何规划加油次数最少，和跳跃游戏的贪心解法如出一辙。

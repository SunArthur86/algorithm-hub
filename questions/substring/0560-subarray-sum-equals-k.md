# 560. 和为 K 的子数组

> 难度：**Medium** | 分类：子串 | 标签：数组, 哈希表, 前缀和

> 🔗 力扣链接：[https://leetcode.cn/problems/subarray-sum-equals-k/](https://leetcode.cn/problems/subarray-sum-equals-k/)

## 📝 题目描述

给定一个整数数组 `nums` 和一个整数 `k`，统计数组中连续子数组的和等于 `k` 的个数。

子数组是数组中连续的一段。

- 题号：560
- 题目名：和为 K 的子数组
- 难度：中等
- 题型：哈希
- 为什么选这题：它是"前缀和 + 哈希"的经典代表作，这个组合在面试中出现频率极高。

## 🧠 思考路径

暴力做法：枚举所有子数组的起点和终点，计算每个子数组的和，看是否等于 `k`。时间 `O(n²)`。

优化思路：用前缀和。定义 `prefix[i]` 为 `nums[0] + nums[1] + ... + nums[i-1]`。那么子数组 `nums[j..i]` 的和等于 `prefix[i+1] - prefix[j]`。要让它等于 `k`，就是 `prefix[j] = prefix[i+1] - k`。

所以对于每个位置 `i`，我只需要知道"之前有多少个前缀和等于 `prefix[i+1] - k`"。这正是哈希表擅长的——用哈希表记录每个前缀和出现的次数，一边遍历一边查询。

## 💻 Java 题解

### 解法 1：前缀和 + 哈希表（最优）

边遍历边计算前缀和，用哈希表记录每个前缀和出现的次数。对当前位置 j，查找 prefix-k 出现了多少次，即为以 j 结尾的和为 k 的子数组个数。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int subarraySum(int[] nums, int k) {
        Map<Integer, Integer> map = new HashMap<>();
        map.put(0, 1); // 前缀和为0出现1次（空前缀）
        int sum = 0, count = 0;
        for (int num : nums) {
            sum += num;
            count += map.getOrDefault(sum - k, 0);
            map.merge(sum, 1, Integer::sum);
        }
        return count;
    }
}
```

**关键点：**
- map.put(0,1) 处理从头开始的子数组
- sum-k 表示前面有多少个前缀和使得差值为k
- map.merge 简化计数

**执行步骤：**
1. 初始化 map{0:1}
1. 遍历累加 sum
1. count += map[sum-k]
1. map[sum]++


## ⚠️ 易错点

- 必须先查询再更新哈希表。如果先更新再查询，可能会把当前位置的前缀和自己也算进去。
- 哈希表初始要放入 `{0: 1}`，否则当子数组从第一个元素开始时会漏算。
- 本题数组元素可以为负数，所以前缀和不一定单调递增，不能用滑动窗口。

- 忘记 map.put(0,1) 会导致漏算从头开始的子数组

- 不能用滑动窗口——数组有负数

## 📊 复杂度分析

- 时间复杂度：`O(n)`，一趟遍历，每次哈希查询 O(1)。
- 空间复杂度：`O(n)`，最坏情况下哈希表存储 `n` 个不同的前缀和。

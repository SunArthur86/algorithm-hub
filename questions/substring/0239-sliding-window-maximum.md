# 239. 滑动窗口最大值

> 难度：**Hard** | 分类：子串 | 标签：队列, 数组, 滑动窗口, 单调队列, 堆

> 🔗 力扣链接：[https://leetcode.cn/problems/sliding-window-maximum/](https://leetcode.cn/problems/sliding-window-maximum/)

## 📝 题目描述

给你一个整数数组和滑动窗口大小 `k`，窗口从数组最左侧滑到最右侧，每次滑动一格。返回每次滑动窗口中的最大值。

重点不在"怎么遍历窗口"，而在"怎么在 `O(1)` 时间内得到窗口内的最大值，而不是每次都重新扫描整个窗口"。

- 题号：239
- 题目名：滑动窗口最大值
- 难度：困难
- 题型：单调栈
- 为什么选这题：它是单调队列（双端队列 + 单调性）的教科书题，把"维护窗口内的候选最大值"这个思想吃透，所有滑动窗口类问题都能迎刃而解。

## 🧠 思考路径

暴力做法是对每个窗口都扫描一遍找最大值，时间 `O(n * k)`，窗口大了就很慢。

关键观察：当一个更大的数进入窗口后，它前面那些比它小的数就永远不可能成为最大值了——它们既比当前数小，又会比当前数更早离开窗口。这些"没有希望的候选者"可以直接淘汰。

这就引出了单调队列：一个双端队列，存储数组下标，保持队列中的值单调递减。队头始终是当前窗口的最大值。具体操作：
- 新元素入队时，从队尾弹出所有比它小的元素（它们不可能成为最大值了）。
- 如果队头元素已经滑出窗口（下标 < 当前窗口左端），从队头弹出。
- 每个窗口的最大值就是队头元素对应的值。

每个元素最多入队出队各一次，所以整体时间 `O(n)`。

这题的关键词不是"找最大值"，而是"维护一个单调递减的候选队列，过期的从前面删，太弱的从后面删"。

## 💻 Java 题解

### 解法 1：单调队列（双端队列）

维护一个单调递减的双端队列。队首是最大值的索引。新元素入队时，从队尾弹出所有比它小的。队首超出窗口范围则弹出。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(k)`

```java
class Solution {
    public int[] maxSlidingWindow(int[] nums, int k) {
        int n = nums.length;
        int[] result = new int[n - k + 1];
        Deque<Integer> dq = new ArrayDeque<>(); // 存索引
        for (int i = 0; i < n; i++) {
            // 移除超出窗口的队首
            while (!dq.isEmpty() && dq.peekFirst() < i - k + 1)
                dq.pollFirst();
            // 维护单调递减：弹出比当前小的
            while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i])
                dq.pollLast();
            dq.offerLast(i);
            // 窗口满后记录队首（最大值）
            if (i >= k - 1)
                result[i - k + 1] = nums[dq.peekFirst()];
        }
        return result;
    }
}
```

**关键点：**
- 队列存索引不是值
- 维护单调递减
- 队首始终是窗口最大值

**执行步骤：**
1. 遍历每个元素
1. 移除超出窗口范围的队首
1. 从队尾弹出比当前元素小的
1. 当前索引入队
1. 窗口满后取队首为结果


## ⚠️ 易错点

- 队列里存的是下标，不是值。比较大小用 `nums[dq[-1]]`，不是 `dq[-1]`。
- 移除过期元素的条件是 `dq[0] < i - k + 1`（严格小于），不是 `<=`。
- 先移除过期，再移除弱者，最后加入当前元素——顺序不能乱。
- 只有当 `i >= k - 1` 时才开始记录结果（窗口还没形成时不算）。

- 队列存索引不是值——需要用索引判断是否过期

- 不是大顶堆——大顶堆删除超时元素是 O(k)

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个元素最多入队出队各一次。
- 空间复杂度：`O(k)`。队列最多存 `k` 个下标。

## 📖 形象类比

可以把它类比成《水浒传》里梁山泊每轮选出最强战将的场景。

每轮比武（对应每个窗口），只有站上擂台的人（队列中的候选者）有机会。新来的战将（新元素）先挑战擂台上比他弱的人，弱的直接淘汰（从队尾弹出）。比他强的留在台上。如果台上最强的人已经退场（过期，从队头弹出），下一个就顶上。每轮站在擂台最前面的就是当前最强的。

对应关系是：

- "擂台上的候选者"对应单调队列中的下标
- "新来的挑战弱的"对应弹出比当前小的队尾
- "退场的人离开"对应弹出过期的队头
- "擂台最前面的就是最强的"对应队头是窗口最大值

这段记忆的关键是：队列里只留"强者"和"还没过期的候选者"，弱的被后来者淘汰，过期的被时间淘汰。

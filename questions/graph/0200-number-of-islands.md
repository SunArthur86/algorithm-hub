# 200. 岛屿数量

> 难度：**Medium** | 分类：图论 | 标签：深度优先搜索, 广度优先搜索, 并查集, 数组, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/number-of-islands/](https://leetcode.cn/problems/number-of-islands/)

## 📝 题目描述

给你一个由 `'1'`（陆地）和 `'0'`（水）组成的二维网格，计算网格中岛屿的数量。岛屿是四面相连的 `'1'` 组成的连通区域。

- 题号：200
- 题目名：岛屿数量
- 难度：中等
- 题型：图论
- 为什么选这题：它是"网格图上的 BFS/DFS 染色"问题的模板——找出图中连通块的数量，掌握了它，所有"网格搜索"题都能解。

## 🧠 思考路径

遍历整个网格。遇到 `'1'` 时，说明发现了一个新岛屿，计数器加 1。然后用 BFS 或 DFS 把这个 `'1'` 所在的整个连通块全部"淹没"（标记为 `'0'`），这样后续遍历就不会重复计数。

关键是：每发现一个未访问的 `'1'`，就触发一次"扩散染色"，把相连的陆地全部标记。扩散的次数就是岛屿的数量。

## 💻 Java 题解

### 解法 1：DFS 洪水填充

遍历网格，遇到'1'就DFS把整个岛标记为'0'（沉岛），岛屿数+1。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(m×n)`

```java
class Solution {
    public int numIslands(char[][] grid) {
        int count = 0;
        for (int i = 0; i < grid.length; i++)
            for (int j = 0; j < grid[0].length; j++)
                if (grid[i][j] == '1') {
                    dfs(grid, i, j);
                    count++;
                }
        return count;
    }
    private void dfs(char[][] grid, int i, int j) {
        if (i < 0 || i >= grid.length || j < 0 || j >= grid[0].length || grid[i][j] != '1')
            return;
        grid[i][j] = '0'; // 沉岛
        dfs(grid, i+1, j); dfs(grid, i-1, j);
        dfs(grid, i, j+1); dfs(grid, i, j-1);
    }
}
```

**关键点：**
- 沉岛法：访问过的直接改为'0'
- 四个方向DFS
- 每个岛只需触发一次DFS

**执行步骤：**
1. 遍历每个格子
1. 遇到'1'→DFS标记整个岛为'0'
1. 计数+1


## ⚠️ 易错点

- 边界检查：DFS 递归前必须判断下标是否越界。
- 标记时机：BFS 中要在入队时就标记为 `'0'`，不要在出队时标记，否则同一个格子可能被重复入队。
- 四个方向是上下左右（4 连通），不是对角线（8 连通）。
- 网格是 `char` 类型，比较时用 `'1'` 和 `'0'`，不是 `1` 和 `0`。

- grid是char不是int，用'1'不是1

- DFS终止条件要先检查边界

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`，每个格子最多被访问一次。
- 空间复杂度：`O(m × n)`（DFS 递归栈最坏情况，或 BFS 队列最大长度）。

# 994. 腐烂的橘子

> 难度：**Medium** | 分类：图论 | 标签：广度优先搜索, 数组, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/rotting-oranges/](https://leetcode.cn/problems/rotting-oranges/)

## 📝 题目描述

给你一个 `m × n` 的网格，每个格子有三种状态：`0`（空）、`1`（新鲜橘子）、`2`（腐烂橘子）。每分钟，腐烂橘子会让上下左右相邻的新鲜橘子腐烂。返回所有橘子都腐烂所需的最少分钟数。如果不可能让所有橘子都腐烂，返回 `-1`。

- 题号：994
- 题目名：腐烂的橘子
- 难度：中等
- 题型：图论
- 为什么选这题：它是 BFS 多源最短路径的经典入门题——从多个起点同时扩散，每一步影响相邻格子，计算"感染"全部目标所需的最少步数。

## 🧠 思考路径

这题的模型是"多源 BFS"。初始时所有腐烂的橘子都是 BFS 的起点，它们同时向外扩散。

做法是：先把所有初始腐烂橘子的坐标加入队列，然后进行 BFS。每一轮（一层 BFS）代表一分钟，所有当前队列中的腐烂橘子同时感染相邻的新鲜橘子。

BFS 结束后，检查网格中是否还有新鲜橘子。如果有，说明有些橘子被隔离了（被空格子围住），返回 `-1`。如果没有，返回 BFS 经过的轮数。

## 💻 Java 题解

### 解法 1：多源 BFS

所有初始腐烂橘子同时入队，BFS逐层扩散。层数即分钟数。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(m×n)`

```java
class Solution {
    public int orangesRotting(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        Queue<int[]> q = new LinkedList<>();
        int fresh = 0;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++) {
                if (grid[i][j] == 2) q.offer(new int[]{i, j});
                else if (grid[i][j] == 1) fresh++;
            }
        int minutes = 0;
        int[][] dirs = {{0,1},{0,-1},{1,0},{-1,0}};
        while (!q.isEmpty() && fresh > 0) {
            int size = q.size();
            for (int i = 0; i < size; i++) {
                int[] cur = q.poll();
                for (int[] d : dirs) {
                    int r = cur[0]+d[0], c = cur[1]+d[1];
                    if (r>=0 && r<m && c>=0 && c<n && grid[r][c]==1) {
                        grid[r][c] = 2;
                        fresh--;
                        q.offer(new int[]{r, c});
                    }
                }
            }
            minutes++;
        }
        return fresh == 0 ? minutes : -1;
    }
}
```

**关键点：**
- 多源同时BFS
- 层数=分钟数
- fresh==0 判断是否全部腐烂

**执行步骤：**
1. 所有腐烂橘子入队
1. BFS逐层扩散
1. 每层minutes++
1. fresh>0 则返回-1


## ⚠️ 易错点

- BFS 要按层处理，不能直接把新腐烂的橘子也当作"本分钟"的——用 `for _ in range(len(queue))` 控制每层。
- 如果一开始没有新鲜橘子，应返回 0（不需要任何时间）。
- 如果一开始没有腐烂橘子但有新鲜橘子，返回 -1。

- 初始没有新鲜橘子时返回0

- queue判空和 fresh>0 双条件

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`，每个格子最多被访问一次。
- 空间复杂度：`O(m × n)`，最坏情况下队列中包含所有格子。

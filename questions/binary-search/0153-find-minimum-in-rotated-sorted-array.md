# 153. 寻找旋转排序数组中的最小值

> 难度：**Medium** | 分类：二分查找 | 标签：数组, 二分查找

> 🔗 力扣链接：[https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/)

## 📝 题目描述

给你一个升序数组，它在某个位置被旋转了。比如 `[0,1,2,4,5,6,7]` 在下标 3 处旋转变成 `[4,5,6,7,0,1,2]`。数组中元素各不相同。你要找到其中的最小值。

重点不在"遍历找最小"，而在"怎么利用有序性用二分做到 O(log n)"。

- 题号：153
- 题目名：寻找旋转排序数组中的最小值
- 难度：中等
- 题型：二分查找
- 为什么选这题：它展示了二分查找不只能找"某个值"，还能找"转折点"，是二分思维的扩展。

## 🧠 思考路径

暴力遍历是 O(n)，但题目说数组本质是有序的，只是被旋转了一下。我们能不能利用这个有序性？

观察旋转数组的特征：它被分成了两段，每段内部都是升序的，但第二段的起点就是最小值。如果拿中间元素和右端点比：

- `nums[mid] > nums[right]`：mid 在左半段，最小值在 mid 右边。
- `nums[mid] <= nums[right]`：mid 在右半段（或刚好是最小值），最小值在 mid 或它左边。

这样每次都能把搜索范围缩小一半，这就是标准二分的变体。

所以这题的关键词不是"找最小值"，而是"和右端点比较，判断最小值在哪一侧"。

## 💻 Java 题解

### 解法 1：二分（与右端比较）

每次与nums[right]比较。nums[mid]>nums[right]→最小在右半(left=mid+1)，否则在左半(right=mid)。

- ⏱ 时间复杂度：`O(log n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int findMin(int[] nums) {
        int left = 0, right = nums.length - 1;
        while (left < right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] > nums[right]) left = mid + 1;
            else right = mid;
        }
        return nums[left];
    }
}
```

**关键点：**
- 与nums[right]比较不是nums[left]
- right=mid 不是 mid-1（mid可能是答案）
- 循环条件 left<right

**执行步骤：**
1. 二分
1. nums[mid]>nums[right]→left=mid+1
1. 否则right=mid
1. 返回nums[left]


## ⚠️ 易错点

- 循环条件是 `left < right`，不是 `left <= right`，否则可能死循环。
- 当 `nums[mid] <= nums[right]` 时，`right = mid`（不是 `mid - 1`），因为 mid 本身可能就是最小值。
- 这题和右端点比较，不是和左端点比较。和左端点比较的逻辑不同，容易写错。

- right=mid不是mid-1因为mid可能就是最小值

## 📊 复杂度分析

- 时间复杂度：`O(log n)`。每次折半。
- 空间复杂度：`O(1)`。

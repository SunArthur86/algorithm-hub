# 35. 搜索插入位置

> 难度：**Easy** | 分类：二分查找 | 标签：数组, 二分查找

> 🔗 力扣链接：[https://leetcode.cn/problems/search-insert-position/](https://leetcode.cn/problems/search-insert-position/)

## 📝 题目描述

给你一个排序数组和一个目标值 `target`。如果 `target` 在数组中存在，返回它的下标；如果不存在，返回它应该被插入的位置下标，使得插入后数组仍然有序。

要求时间复杂度 `O(log n)`。

- 题号：35
- 题目名：搜索插入位置
- 难度：简单
- 题型：二分查找
- 为什么选这题：它是二分查找最干净的入门题——不是找"有没有"，而是找"该插在哪里"，帮你理解二分结束后 `left` 指针的含义。

## 🧠 思考路径

最直接的想法是遍历数组，找到第一个大于等于 `target` 的位置。但这是 `O(n)`，题目要求 `O(log n)`，所以要用二分。

标准二分在找到 `target` 时就返回了。但这题还要处理 `target` 不存在的情况——返回"该插入的位置"。

观察二分的循环 `while left <= right`：循环结束时，`left` 一定停在"第一个大于 `target` 的位置"或者"等于 `target` 的位置"上。换句话说，`left` 就是我们要的答案。

具体来说，当 `nums[mid] >= target` 时，我们让 `right = mid - 1`（往左压）；当 `nums[mid] < target` 时，让 `left = mid + 1`（往右走）。这样最终 `left` 会精确停在该插入的位置。

## 💻 Java 题解

### 解法 1：二分查找

标准二分。循环结束后left即为插入位置。

- ⏱ 时间复杂度：`O(log n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int searchInsert(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) return mid;
            else if (nums[mid] < target) left = mid + 1;
            else right = mid - 1;
        }
        return left; // 插入位置
    }
}
```

**关键点：**
- while left<=right
- 找不到时left就是插入点
- mid=left+(right-left)/2 防溢出

**执行步骤：**
1. 标准二分
1. target不存在时返回left


## ⚠️ 易错点

- 循环条件是 `left <= right`（不是 `<`），否则当 `target` 比所有元素都大时，`left` 会停在 `n`，但如果用 `<` 可能少判断一轮。
- 最终返回 `left` 不是 `right`。当 `target` 大于所有元素时，`left` 会等于 `n`，而 `right` 会等于 `n-1`，返回 `right` 就错了。
- 当数组为空时直接返回 0，但二分框架本身也能处理（`left=0, right=-1`，循环不执行，返回 0）。

- 循环条件是<=不是<

- 找不到返回left不是-1

## 📊 复杂度分析

- 时间复杂度：`O(log n)`。标准二分，每次排除一半。
- 空间复杂度：`O(1)`。只用常数个变量。

# 62. 不同路径

> 难度：**Medium** | 分类：多维动态规划 | 标签：数学, 动态规划, 组合数学

> 🔗 力扣链接：[https://leetcode.cn/problems/unique-paths/](https://leetcode.cn/problems/unique-paths/)

## 📝 题目描述

一个 `m × n` 的网格，你站在左上角，目标是走到右下角。每次只能向右或向下走一步。问：有多少种不同的路径？

核心约束是"只能往右或往下"。这意味着每个格子只能从上方或左方到达，路径数就是这两个方向来的路径数之和。

- 题号：62
- 题目名：不同路径
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是最经典的二维动态规划入门题，能帮你建立"从终点倒推到起点"的 DP 直觉。

## 🧠 思考路径

先看小规模。如果是 `2 × 2` 的网格，从左上到右下有 2 条路：先右再下，或先下再右。

再想递推关系。站在某个格子上，你只可能从上面走下来或从左边走过来。所以到达这个格子的路径总数 = 上面格子的路径数 + 左面格子的路径数。这就是经典的 DP 递推式：`dp[i][j] = dp[i-1][j] + dp[i][j-1]`。

边界条件很简单：第一行的格子只能从左边来，所以都只有 1 条路；第一列的格子只能从上面来，也都只有 1 条路。

进一步优化空间：计算 `dp[i][j]` 只需要上一行的值和本行左边的值，所以用一维数组就够了，从左到右滚动更新。

再进一步，这题其实可以用组合数学直接算。从左上到右下一共走 `m+n-2` 步，其中 `m-1` 步向下，`n-1` 步向右。路径数就是从 `m+n-2` 里选 `m-1` 的组合数 `C(m+n-2, m-1)`。

## 💻 Java 题解

### 解法 1：动态规划（空间优化）

dp[j]=dp[j]+dp[j-1]。一维数组滚动。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int uniquePaths(int m, int n) {
        int[] dp = new int[n];
        Arrays.fill(dp, 1);
        for (int i = 1; i < m; i++)
            for (int j = 1; j < n; j++)
                dp[j] += dp[j - 1];
        return dp[n - 1];
    }
}
```

**关键点：**
- 一维数组
- dp[j]+=dp[j-1]
- 第一行全1初始化

**执行步骤：**
1. 初始化全1
1. 逐行累加
1. 返回dp[n-1]


## ⚠️ 易错点

- 不要把 `m` 和 `n` 搞反了。`m` 是行数，`n` 是列数，初始化时数组长度应该是 `n`。
- 暴力递归会超时。`m = 23, n = 12` 时路径数已经是上千万，递归深度和重复计算都会爆掉。
- 用组合数学解法时，注意先除后乘防止溢出，Python 3 的 `math.comb` 已经处理好了。

- 一维数组从1开始j

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`。遍历整个网格一次。
- 空间复杂度：`O(n)`。用一维数组滚动更新。

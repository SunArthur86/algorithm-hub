# 1143. 最长公共子序列

> 难度：**Medium** | 分类：多维动态规划 | 标签：字符串, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/longest-common-subsequence/](https://leetcode.cn/problems/longest-common-subsequence/)

## 📝 题目描述

给你两个字符串 `text1` 和 `text2`，找到它们的最长公共子序列的长度。子序列是指从原字符串中删除一些（或不删除）字符后，不改变剩余字符相对顺序得到的新字符串。

例如 `"abcde"` 和 `"ace"` 的最长公共子序列是 `"ace"`，长度为 3。

- 题号：1143
- 题目名：最长公共子序列
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是字符串 DP 的**母题**，几乎所有字符串匹配/比对问题都从这道题延伸出去。

## 🧠 思考路径

经典二维 DP 问题。定义 `dp[i][j]` 为 `text1` 的前 `i` 个字符和 `text2` 的前 `j` 个字符的最长公共子序列长度。

对于每对 `(i, j)`：
- 如果 `text1[i-1] == text2[j-1]`，那么这个字符可以加入公共子序列：`dp[i][j] = dp[i-1][j-1] + 1`。
- 如果不等，那么取去掉 `text1[i-1]` 或去掉 `text2[j-1]` 两种情况的最大值：`dp[i][j] = max(dp[i-1][j], dp[i][j-1])`。

初始条件：`dp[0][j] = 0` 和 `dp[i][0] = 0`（空串与任何串的公共子序列长度为 0）。

## 💻 Java 题解

### 解法 1：动态规划

字符相等：dp[i][j]=dp[i-1][j-1]+1。不等：dp[i][j]=max(dp[i-1][j], dp[i][j-1])。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(m×n)`

```java
class Solution {
    public int longestCommonSubsequence(String text1, String text2) {
        int m = text1.length(), n = text2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++) {
                if (text1.charAt(i-1) == text2.charAt(j-1))
                    dp[i][j] = dp[i-1][j-1] + 1;
                else
                    dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
            }
        return dp[m][n];
    }
}
```

**关键点：**
- 相等取对角线+1
- 不等取上和左的最大
- dp从1开始，字符串从0索引

**执行步骤：**
1. 双重循环
1. 字符相等→dp[i-1][j-1]+1
1. 不等→max(up,left)


## ⚠️ 易错点

- `dp` 数组大小是 `(m+1) × (n+1)`，不是 `m × n`，因为要处理空串的边界。
- `text1[i-1]` 和 `text2[j-1]` 的下标是 `-1` 偏移，因为 `dp` 的索引从 1 开始。
- 不要把子序列和子串搞混：子串要求连续，子序列不要求。这道题是子序列。
- 空间优化时只能用一维数组，但必须用临时变量保存左上角的值（因为会被覆盖）。

- dp索引从1开始，字符串索引差1

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`。两层嵌套循环，每个格子 O(1) 转移。
- 空间复杂度：`O(m × n)`。可以优化到 `O(min(m, n))`，用一维滚动数组。

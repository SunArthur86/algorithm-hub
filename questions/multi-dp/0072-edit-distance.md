# 72. 编辑距离

> 难度：**Medium** | 分类：多维动态规划 | 标签：字符串, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/edit-distance/](https://leetcode.cn/problems/edit-distance/)

## 📝 题目描述

给你两个字符串 `word1` 和 `word2`，你可以对 `word1` 执行三种操作：插入一个字符、删除一个字符、替换一个字符。问最少需要几次操作才能把 `word1` 变成 `word2`。

- 题号：72
- 题目名：编辑距离
- 难度：中等
- 题型：动态规划
- 为什么选这题：它是字符串 DP 的巅峰之作，理解了它，就能举一反三处理最长公共子序列、回文等变体。

## 🧠 思考路径

先缩小问题规模。假设我已经知道把 `word1` 的前 `i` 个字符变成 `word2` 的前 `j` 个字符需要多少步，那 `i+1` 和 `j+1` 的情况能不能从它推出？

可以。如果 `word1[i-1] == word2[j-1]`，这两个字符对齐了，不用额外操作，直接继承 `dp[i-1][j-1]`。如果不等，就有三种选择：替换（从 `dp[i-1][j-1]` 加一步）、删除（从 `dp[i-1][j]` 加一步）、插入（从 `dp[i][j-1]` 加一步），取三者最小值。

于是问题变成了一张二维表格的填充问题，每个格子只依赖它左边、上边、左上的三个格子。

## 💻 Java 题解

### 解法 1：动态规划

字符相等：dp[i][j]=dp[i-1][j-1]。不等：dp[i][j]=1+min(替换dp[i-1][j-1], 删除dp[i-1][j], 插入dp[i][j-1])。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(m×n)`

```java
class Solution {
    public int minDistance(String word1, String word2) {
        int m = word1.length(), n = word2.length();
        int[][] dp = new int[m + 1][n + 1];
        for (int i = 0; i <= m; i++) dp[i][0] = i;
        for (int j = 0; j <= n; j++) dp[0][j] = j;
        for (int i = 1; i <= m; i++)
            for (int j = 1; j <= n; j++) {
                if (word1.charAt(i-1) == word2.charAt(j-1))
                    dp[i][j] = dp[i-1][j-1];
                else
                    dp[i][j] = 1 + Math.min(dp[i-1][j-1], Math.min(dp[i-1][j], dp[i][j-1]));
            }
        return dp[m][n];
    }
}
```

**关键点：**
- 三种操作：替换/删除/插入
- base case: dp[i][0]=i, dp[0][j]=j
- 相等不操作

**执行步骤：**
1. 初始化第一行第一列
1. 字符相等→dp[i-1][j-1]
1. 不等→1+min(三方向)


## ⚠️ 易错点

- 边界初始化不能忘：`dp[i][0] = i` 表示把 `word1` 的前 `i` 个字符全删变成空串；`dp[0][j] = j` 表示在空串上插入 `j` 个字符。
- 状态转移时"插入"是 `dp[i][j-1] + 1`（对 `word2` 少匹配一个），"删除"是 `dp[i-1][j] + 1`（对 `word1` 少处理一个），别搞反。
- 可以用一维数组优化空间，但要注意左上角的值需要用临时变量保存。

- 三种操作分别对应三个方向：dp[i-1][j-1], dp[i-1][j], dp[i][j-1]

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`，其中 `m` 和 `n` 分别是两个字符串的长度。
- 空间复杂度：`O(m × n)`。可以优化到 `O(min(m, n))`，用一维数组滚动。

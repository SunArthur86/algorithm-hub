# 41. 缺失的第一个正数

> 难度：**Hard** | 分类：普通数组 | 标签：数组, 哈希表

> 🔗 力扣链接：[https://leetcode.cn/problems/first-missing-positive/](https://leetcode.cn/problems/first-missing-positive/)

## 📝 题目描述

给你一个可能包含负数、零和重复元素的数组。你要找出"从 1 开始，第一个没出现在数组里的正整数"。

比如 `[3, 4, -1, 1]`，里面有 1、3、4，但缺了 2，所以答案是 2。再比如 `[7, 8, 9, 11, 12]`，最小的正整数 1 就没出现，答案直接是 1。

- 题号：41
- 题目名：缺失的第一个正数
- 难度：困难
- 题型：数组
- 为什么选这题：它用数组下标本身当哈希表，是原地算法的经典范例，能打开"用位置编码信息"的思路。

## 🧠 思考路径

最朴素的想法是把所有正整数从小到大试一遍，看哪个不在数组里。但正整数可以无穷大，没法穷举。换个思路：先把数组排序，再从 1 开始往后扫，第一个对不上的就是答案。排序能做到，但时间 `O(n log n)`，而且题目要求 `O(n)`。

再进一步：如果数组长度是 `n`，那么答案一定在 `[1, n+1]` 这个范围内。为什么？因为如果 1 到 n 全都出现了，答案就是 `n+1`；如果中间有缺的，答案一定不超过 `n`。这个观察把搜索范围从无穷大缩小到了 `n+1`。

既然答案只和 `[1, n]` 有关，那我们能不能用数组的下标来"标记"某个数存不存在？比如把数字 1 放到下标 0 的位置，数字 2 放到下标 1 的位置……让每个"值在 `[1, n]` 范围内的元素"归位。归位之后，从头扫一遍，第一个 `nums[i] != i + 1` 的位置，`i + 1` 就是答案。

这就是"原地哈希"：不用额外空间，拿数组自己的位置当哈希表来用。

## 💻 Java 题解

### 解法 1：原地哈希（标记法）

遍历数组，将每个在 [1,n] 范围内的数交换到正确位置 nums[i]→nums[nums[i]-1]。最终扫描第一个 nums[i]!=i+1 的位置。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int firstMissingPositive(int[] nums) {
        int n = nums.length;
        for (int i = 0; i < n; i++) {
            while (nums[i] >= 1 && nums[i] <= n && nums[nums[i] - 1] != nums[i]) {
                int temp = nums[nums[i] - 1];
                nums[nums[i] - 1] = nums[i];
                nums[i] = temp;
            }
        }
        for (int i = 0; i < n; i++) {
            if (nums[i] != i + 1) return i + 1;
        }
        return n + 1;
    }
}
```

**关键点：**
- 把 nums[i] 放到索引 nums[i]-1 的位置
- while 循环持续交换直到位置正确
- 最终扫描找不匹配的

**执行步骤：**
1. 遍历每个位置
1. 若值在 [1,n] 且目标位置值不对，交换
1. 扫描第一个 nums[i]!=i+1
1. 都匹配则返回 n+1


## ⚠️ 易错点

- 交换时必须用 `while` 而不是 `if`。一个位置被换入新值后，新值可能也需要归位，只有 `while` 才能保证一轮处理到底。
- 必须加条件 `nums[nums[i] - 1] != nums[i]`，否则遇到重复元素（如 `[1, 1]`）会无限交换。
- Java 交换两数时不能写成 `a = b; b = a`，必须用临时变量，否则值被覆盖。

- while 条件必须包含 nums[nums[i]-1]!=nums[i] 防死循环

- 值不在 [1,n] 的跳过

## 📊 复杂度分析

- 时间复杂度：`O(n)`。虽然用了 `while` 循环，但每次交换都把一个元素放到正确位置，每个位置最多被放好一次，所以总交换次数不超过 `n`。
- 空间复杂度：`O(1)`。只用了常数个临时变量，原地修改数组。

## 🏢 实际业务应用

- **分布式 ID 分配系统**：在分片数据库中分配用户 ID 时，需要快速判断某个范围的 ID 是否已全部用完，找出最小可用的 ID 槽位。这套"归位扫描"思路可以快速定位缺口。
- **内存管理器**：操作系统分配内存页时，维护一个页号池，需要找到第一个未被分配的页号。把页号当作数组下标来标记，可以 `O(n)` 找到最小空闲页。
- **任务调度器**：优先级队列中的任务可能被取消或完成，需要快速找到当前可用的最小优先级编号，用于新任务插入。原地标记的思路避免了额外的哈希表开销。
- **网络端口分配**：服务器监听端口时，需要找到当前未被占用的最小端口号，和本题"找最小缺失正整数"是同一个问题。

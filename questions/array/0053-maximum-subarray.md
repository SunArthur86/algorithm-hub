# 53. 最大子数组和

> 难度：**Medium** | 分类：普通数组 | 标签：数组, 分治, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/maximum-subarray/](https://leetcode.cn/problems/maximum-subarray/)

## 📝 题目描述

给你一个整数数组，里面可能有正有负。找一个连续的子数组（至少一个元素），使得这些元素加起来的和最大，返回这个最大和。

重点不是"子数组在哪"，而是"最大和是多少"。

- 题号：53
- 题目名：最大子数组和
- 难度：中等
- 题型：动态规划
- 为什么选这题：Kadane 算法是动态规划最经典的入门范式之一，"要不要带上前一段"这个决策极具代表性。

## 🧠 思考路径

最朴素的想法：枚举所有可能的子数组，算每个的和，取最大的。两层循环，`O(n²)`，数组一大就慢。

换个角度想：从左往右扫描，站在位置 `i`，我要决定"以 `i` 结尾的最大子数组和是多少"。有两种选择：要么把前面那段接上（前面的和是正的，带着它只会更大），要么从 `i` 重新开始（前面的和是负的，带上它只会更小）。

所以 `dp[i] = max(nums[i], dp[i-1] + nums[i])`——要么自己干，要么继承前一段。最终答案就是所有 `dp[i]` 中的最大值。

再进一步：`dp[i]` 只依赖 `dp[i-1]`，不需要整个数组，一个变量就够了。这就是 Kadane 算法。

## 💻 Java 题解

### 解法 1：Kadane算法（动态规划）

遍历时维护当前连续和。如果当前和为负，不如从当前元素重新开始。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int maxSubArray(int[] nums) {
        int maxSum = nums[0], currSum = nums[0];
        for (int i = 1; i < nums.length; i++) {
            currSum = Math.max(nums[i], currSum + nums[i]);
            maxSum = Math.max(maxSum, currSum);
        }
        return maxSum;
    }
}
```

**关键点：**
- currSum=max(nums[i], currSum+nums[i])
- 负数前缀不如丢弃
- O(1)空间

**执行步骤：**
1. 遍历每个元素
1. 当前和=max(自己, 当前和+自己)
1. 更新全局最大值


## ⚠️ 易错点

- 初始值不能设为 `0`，必须设为 `nums[0]`。如果数组全是负数，答案应该是最大的那个负数，而不是 `0`。
- `cur` 和 `best` 的更新顺序不能反：先更新 `cur`，再用 `cur` 更新 `best`。
- 不要把 `cur = max(nums[i], cur + nums[i])` 写成 `cur += nums[i]`，后者没有"断开重开"的逻辑。

- 初始值用nums[0]不是0（可能有全负数组）

- 先更新currSum再更新maxSum

## 📊 复杂度分析

- 时间复杂度：`O(n)`。只遍历数组一次，每个元素做 O(1) 操作。
- 空间复杂度：`O(1)`。只用了两个变量 `cur` 和 `best`。

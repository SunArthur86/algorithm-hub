# 155. 最小栈

> 难度：**Medium** | 分类：栈 | 标签：栈, 设计

> 🔗 力扣链接：[https://leetcode.cn/problems/min-stack/](https://leetcode.cn/problems/min-stack/)

## 📝 题目描述

设计一个栈，除了标准的 `push`、`pop`、`top` 操作外，还要支持 `getMin`，能在 O(1) 时间内拿到栈里当前的最小值。

重点不在"栈怎么实现"，而在"怎么在不遍历的情况下随时知道最小值"。

- 题号：155
- 题目名：最小栈
- 难度：中等
- 题型：栈
- 为什么选这题：它是"用空间换时间"的经典范例，用辅助栈把 O(n) 的最小值查询变成 O(1)。

## 🧠 思考路径

普通的栈，找最小值需要遍历，O(n)。要 O(1) 拿到，就必须在每次 push/pop 的时候维护好最小值信息。

最直观的想法：用一个额外的变量存当前最小值。但 pop 的时候，如果弹出的正好是最小值，怎么知道"次小值"是多少？

答案是再用一个栈——最小值栈。每次 push 的时候，把当前的最小值也压入最小值栈；每次 pop 的时候，两个栈同步弹出。这样最小值栈的栈顶始终是当前栈的最小值。

所以这题的关键词不是"栈"，而是"主栈管数据，辅栈管最小值，同步进出"。

## 💻 Java 题解

### 解法 1：辅助栈

主栈存数据，辅助栈同步存当前最小值。push时辅助栈push min(val, 当前栈顶)，pop时两个栈同步弹。

- ⏱ 时间复杂度：`O(1) all ops`
- 💾 空间复杂度：`O(n)`

```java
class MinStack {
    Deque<Integer> stack, minStack;
    public MinStack() { stack = new ArrayDeque<>(); minStack = new ArrayDeque<>(); }
    public void push(int val) {
        stack.push(val);
        minStack.push(minStack.isEmpty() ? val : Math.min(val, minStack.peek()));
    }
    public void pop() { stack.pop(); minStack.pop(); }
    public int top() { return stack.peek(); }
    public int getMin() { return minStack.peek(); }
}
```

**关键点：**
- 辅助栈同步记录最小值
- push时与辅助栈顶比较取min

**执行步骤：**
1. push: 两栈同时push
1. pop: 两栈同时pop
1. getMin: 返回辅助栈顶


## ⚠️ 易错点

- `min_stack` 每次 push 都要压值，不能只在更小的时候压——否则 pop 时两个栈长度不一致，对应关系就乱了。
- 比较时用 `<=` 不是 `<`，保证相等时也压入，否则弹出后可能丢失最小值。
- 两个栈必须同步 pop，不能只弹一个。

- push第一个元素时辅助栈为空要特殊处理

## 📊 复杂度分析

- 时间复杂度：所有操作都是 `O(1)`。
- 空间复杂度：`O(n)`。最坏情况最小值栈和主栈一样长。

## 📖 形象类比

可以把它类比成《三国演义》里刘备行军时粮草官的记账方式。

刘备的军队一路行进，每到一处就建一个粮仓（push）。粮草官同时记两本账：一本记每个粮仓有多少粮（主栈），另一本记"到这一站为止，最少的存粮是多少"（辅助栈）。撤军时（pop），两本账同步翻一页。刘备问"现在最缺粮的是多少"（getMin），粮草官看第二本账最后一页就行。

对应关系是：

- "粮仓"对应栈里的元素
- "粮草账本"对应主栈 `stack`
- "最少存粮账本"对应辅助栈 `min_stack`
- "同步翻页"对应同步 pop
- "看最后一页就知最少"对应 O(1) 查询

这段记忆的关键是：记账时多记一笔"到目前最少多少"，查的时候就不必翻遍整本账。

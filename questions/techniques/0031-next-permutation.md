# 31. 下一个排列

> 难度：**Medium** | 分类：技巧 | 标签：数组, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/next-permutation/](https://leetcode.cn/problems/next-permutation/)

## 📝 题目描述

给你一个整数数组，它代表一个排列。你要把它改成"字典序中刚好比它大的下一个排列"。如果当前排列已经是最大的（完全降序），就把它变成最小的（完全升序）。

比如 `[1, 2, 3]` 的下一个排列是 `[1, 3, 2]`，`[3, 2, 1]` 已经是最大的了，下一个就回到 `[1, 2, 3]`。

修改必须就地进行，不能额外开数组。

- 题号：31
- 题目名：下一个排列
- 难度：中等
- 题型：数组
- 为什么选这题：它是"就地修改数组"的经典题，考察从右往左扫描和双指针交换的思维，也是理解排列字典序的关键一步。

## 🧠 思考路径

先想"下一个排列比当前排列大在哪里"。从右往左看，如果一段后缀是降序的，那它已经是这一段能排出的最大值，想变大只能往左找。

所以第一步：从右往左找到第一个"升高的位置"，也就是找到第一对 `nums[i] < nums[i+1]` 的 `i`。这个位置就是我们能做文章的地方。

找到 `i` 之后，我们想让 `nums[i]` 变大一点点。多大呢？比它大的最小值。所以在 `i` 右边的降序后缀里，从右往左找第一个比 `nums[i]` 大的元素，和 `nums[i]` 交换。

交换之后，`i` 右边那段仍然是降序的（因为我们选的是"刚好比 `nums[i]` 大"的元素来交换），但我们要的是"下一个排列"，也就是只比原来大一点点，所以把 `i` 右边翻转成升序即可。

如果从头到尾都没找到 `nums[i] < nums[i+1]`，说明整个数组是降序的，直接全部翻转成升序。

## 💻 Java 题解

### 解法 1：三步法

1. 从右找第一个i使得nums[i]<nums[i+1]。2. 从右找第一个j使得nums[j]>nums[i]。3. 交换i和j，反转i+1到末尾。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public void nextPermutation(int[] nums) {
        // 1. 找下降点
        int i = nums.length - 2;
        while (i >= 0 && nums[i] >= nums[i + 1]) i--;
        if (i >= 0) {
            // 2. 找比nums[i]大的最小值
            int j = nums.length - 1;
            while (nums[j] <= nums[i]) j--;
            // 3. 交换
            swap(nums, i, j);
        }
        // 4. 反转i+1到末尾
        reverse(nums, i + 1, nums.length - 1);
    }
    private void swap(int[] nums, int i, int j) { int t = nums[i]; nums[i] = nums[j]; nums[j] = t; }
    private void reverse(int[] nums, int l, int r) { while (l < r) swap(nums, l++, r--); }
}
```

**关键点：**
- 从右找第一个下降
- 从右找第一个比它大的
- 交换后反转后面

**执行步骤：**
1. 找i（第一个下降点）
1. 找j（大于nums[i]的最右值）
1. 交换
1. 反转i+1到末尾


## ⚠️ 易错点

- 找 `i` 的时候条件是 `nums[i] >= nums[i+1]`（注意是 `>=` 不是 `>`），要跳过所有相等和降序的位置，找到第一个严格升高的地方。
- 找 `j` 的时候条件是 `nums[j] <= nums[i]`（注意是 `<=` 不是 `<`），因为等于 `nums[i]` 的元素交换后没有意义。
- 如果整个数组是降序的，`i` 会变成 `-1`，这时跳过交换步骤，直接翻转整个数组。

- i<0说明整体降序→全反转得到最小

- 反转不是排序——后面已经是降序

## 📊 复杂度分析

- 时间复杂度：`O(n)`。最多扫描两遍数组（找 `i` 和找 `j`），翻转也是 `O(n)`。
- 空间复杂度：`O(1)`。只用了常数个变量，所有操作就地进行。

## 🏢 实际业务应用

- 代码生成工具中按字典序枚举所有排列时，用这个算法从当前排列生成下一个，不需要额外空间存储所有排列。
- 游戏里的关卡排序或卡牌组合排序，需要"找出比当前组合稍大的下一个组合"时，就是下一个排列的逻辑。
- 数据库查询优化器在枚举多表连接的顺序时，用排列生成来穷举所有 join 顺序，下一个排列算法负责按字典序推进。

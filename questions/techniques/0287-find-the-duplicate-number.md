# 287. 寻找重复数

> 难度：**Medium** | 分类：技巧 | 标签：位运算, 数组, 双指针, 二分查找

> 🔗 力扣链接：[https://leetcode.cn/problems/find-the-duplicate-number/](https://leetcode.cn/problems/find-the-duplicate-number/)

## 📝 题目描述

给你一个包含 `n + 1` 个整数的数组 `nums`，数字在 `[1, n]` 范围内。至少有一个重复的数，找出那个重复的数。要求不修改数组、O(1) 空间。

重点不在"怎么排序或用哈希"，而在"怎么把数组看成链表来找环"。

- 题号：287
- 题目名：寻找重复数
- 难度：中等
- 题型：链表
- 为什么选这题：它把数组下标当成指针，把问题转化成"链表找环"，是"换个角度建模"的经典案例。

## 🧠 思考路径

每个元素的值在 `[1, n]` 范围内，所以可以把 `nums[i]` 看成"从节点 i 指向节点 nums[i] 的指针"。这样数组就变成了一个链表（或更准确地说，一个有环的有向图）。

因为数组有 `n+1` 个元素但值只有 `n` 种可能，所以一定有重复，这意味着链表一定有环。重复的数就是环的入口。

找环的入口用 Floyd 龟兔赛跑算法：快指针每次走两步，慢指针每次走一步，相遇后，一个指针回到起点，两个指针每次各走一步，再次相遇就是环的入口。

所以这题的关键词不是"找重复"，而是"数组即链表，重复即环入口"。

## 💻 Java 题解

### 解法 1：Floyd判圈（快慢指针）

把nums[i]看作i→nums[i]的链表。重复元素造成环。快慢指针找相遇点，再找入环点。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int findDuplicate(int[] nums) {
        // 快慢指针找相遇点
        int slow = nums[0], fast = nums[0];
        do {
            slow = nums[slow];
            fast = nums[nums[fast]];
        } while (slow != fast);
        // 找入环点
        slow = nums[0];
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;
    }
}
```

**关键点：**
- nums[i]作为链表指针
- do-while找相遇点
- 第二次同速找入环点=重复值

**执行步骤：**
1. 快慢指针找相遇点
1. 一个指针回头
1. 同速走找入环点


## ⚠️ 易错点

- 初始时 `slow = fast = nums[0]`，不是 0。0 不在值域 `[1, n]` 中。
- 快指针是 `nums[nums[fast]]`，不是 `nums[fast * 2]`。
- 第二阶段两个指针每次各走一步，不是一快一慢。

- do-while不是while（初始slow==fast）

- 入环点就是重复元素

## 📊 复杂度分析

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(1)`。

# 169. 多数元素

> 难度：**Easy** | 分类：技巧 | 标签：数组, 哈希表, 分治, 计数, 排序

> 🔗 力扣链接：[https://leetcode.cn/problems/majority-element/](https://leetcode.cn/problems/majority-element/)

## 📝 题目描述

给你一个数组 `nums`，大小为 `n`，其中有个元素出现的次数超过 `⌊n/2⌋`（多数元素）。找到并返回这个元素。

题目保证多数元素一定存在。

重点不在"计数谁最多"，而在"怎么不用额外空间，一遍找出过半的元素"。

- 题号：169
- 题目名：多数元素
- 难度：简单
- 题型：位运算
- 为什么选这题：Boyer-Moore 投票算法是它的最优解法，O(1) 空间找众数，思路极其巧妙。

## 🧠 思考路径

最容易想到的是用哈希表计数，O(n) 时间 O(n) 空间。排序取中位数也可以，O(n log n) 时间 O(1) 空间。

但有个更妙的算法：Boyer-Moore 投票法。想象一个场景：每个多数元素投一票，每个其他元素投反对票。因为多数元素超过一半，它们最终一定不会被完全抵消。

具体做法：维护一个候选者 `candidate` 和一个计数器 `count`。遍历数组，如果 `count == 0`，把当前元素设为候选者。如果当前元素等于候选者，`count++`；否则 `count--`。最后剩下的候选者就是多数元素。

所以这题的关键词不是"计数"，而是"多数元素一定抵消不完，候选人就是答案"。

## 💻 Java 题解

### 解法 1：Boyer-Moore 投票

维护候选人和计数器。相同+1，不同-1，计数器归零换候选人。多数元素一定最后留下。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int majorityElement(int[] nums) {
        int candidate = nums[0], count = 1;
        for (int i = 1; i < nums.length; i++) {
            if (count == 0) { candidate = nums[i]; count = 1; }
            else if (nums[i] == candidate) count++;
            else count--;
        }
        return candidate;
    }
}
```

**关键点：**
- count归零换候选人
- 多数元素出现>n/2次一定存活
- 无需验证

**执行步骤：**
1. 遍历
1. 相同+1不同-1
1. 归零换人
1. 返回候选人


## ⚠️ 易错点

- 题目保证多数元素存在，所以不需要二次验证。如果不保证，需要再遍历一次确认。
- `count == 0` 时换候选者，不是 `count < 0`。
- 初始 `candidate` 可以是任意值（因为 `count` 从 0 开始，第一个元素就会成为候选者）。

- 题目保证一定有多数元素，所以不用验证

## 📊 复杂度分析

- 时间复杂度：`O(n)`。遍历一次。
- 空间复杂度：`O(1)`。只用两个变量。

- "宋江派系过半"对应多数元素超过 n/2

这段记忆的关键是：超过半数的人不会被完全抵消。

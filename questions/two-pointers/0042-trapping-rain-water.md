# 42. 接雨水

> 难度：**Hard** | 分类：双指针 | 标签：栈, 数组, 双指针, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/trapping-rain-water/](https://leetcode.cn/problems/trapping-rain-water/)

## 📝 题目描述

给你一组柱子的高度图，比如 `[0,1,0,2,1,0,1,3,2,1,2,1]`。想象这些柱子之间下了一场雨，问柱子之间能接住多少雨水。

关键在于：每个位置能接多少水，取决于它左边最高的柱子和右边最高的柱子中较矮的那个，再减去当前位置自身的高度。

- 题号：42
- 题目名：接雨水
- 难度：困难
- 题型：双指针
- 为什么选这题：它是双指针最经典的难题，把"木桶原理"和"左右夹逼"结合得天衣无缝。

## 🧠 思考路径

最朴素的想法是对每个位置，往左扫描找左边最高柱子，往右扫描找右边最高柱子，然后算出该位置能接多少水。每个位置要扫两遍，总共 `O(n²)`。

进一步优化：可以预处理两个数组，`leftMax[i]` 记录位置 `i` 左边的最高值，`rightMax[i]` 记录右边的最高值。这样每个位置的答案就是 `min(leftMax[i], rightMax[i]) - height[i]`。时间降到 `O(n)`，但空间是 `O(n)`。

还能不能省空间？这就是双指针的精妙之处。我们用左右两个指针从两端往中间走，同时维护左边的历史最大值 `leftMax` 和右边的历史最大值 `rightMax`。关键观察：如果 `leftMax < rightMax`，那么左边指针所指位置的水量完全由 `leftMax` 决定（因为右边一定有一个更高的柱子挡住了），可以直接算出。反之亦然。这样一边走一边算，空间只需 `O(1)`。

## 💻 Java 题解

### 解法 1：动态规划（左右最大值数组）

预处理 leftMax[] 和 rightMax[] 数组。每个位置雨水量 = min(leftMax[i], rightMax[i]) - height[i]。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int trap(int[] height) {
        int n = height.length;
        if (n == 0) return 0;
        int[] leftMax = new int[n], rightMax = new int[n];
        leftMax[0] = height[0];
        for (int i = 1; i < n; i++)
            leftMax[i] = Math.max(leftMax[i - 1], height[i]);
        rightMax[n - 1] = height[n - 1];
        for (int i = n - 2; i >= 0; i--)
            rightMax[i] = Math.max(rightMax[i + 1], height[i]);
        int water = 0;
        for (int i = 0; i < n; i++)
            water += Math.min(leftMax[i], rightMax[i]) - height[i];
        return water;
    }
}
```

**关键点：**
- 预处理左右最大值数组
- 雨水量 = min(leftMax,rightMax) - height
- 三次遍历

**执行步骤：**
1. 正向遍历求 leftMax[]
1. 逆向遍历求 rightMax[]
1. 每个位置计算雨水量并累加


### 解法 2：双指针（空间优化）

左右指针从两端出发，维护 leftMax 和 rightMax 变量。移动较小的一边，因为较小边的雨水已经可以确定。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int trap(int[] height) {
        int left = 0, right = height.length - 1;
        int leftMax = 0, rightMax = 0, water = 0;
        while (left < right) {
            if (height[left] < height[right]) {
                leftMax = Math.max(leftMax, height[left]);
                water += leftMax - height[left];
                left++;
            } else {
                rightMax = Math.max(rightMax, height[right]);
                water += rightMax - height[right];
                right--;
            }
        }
        return water;
    }
}
```

**关键点：**
- O(1) 空间
- 移动较小边
- leftMax-height[left] >= 0

**执行步骤：**
1. left=0, right=n-1
1. 比较两边高度
1. 较小边：更新 max，累加雨水，移动指针


## ⚠️ 易错点

- 先移动指针，再更新最大值和算水量。顺序不能反，否则 `leftMax`/`rightMax` 的更新时机不对。
- 移动指针时 `left++` 和 `right--` 不要写反：`leftMax` 更小时移动左指针，`rightMax` 更小时移动右指针。
- 不要忘了处理空数组的边界情况。
- `water += leftMax - height[left]` 中的 `leftMax` 是更新后的值，保证结果非负。

- 双指针中 leftMax 是左侧实际最大值，不是假设值

- 空数组直接返回 0

## 📊 复杂度分析

- 时间复杂度：`O(n)`。左右指针各最多走 `n` 步，相遇即停。
- 空间复杂度：`O(1)`。只用了 `left`、`right`、`leftMax`、`rightMax`、`water` 五个变量。

# 98. 验证二叉搜索树

> 难度：**Medium** | 分类：二叉树 | 标签：树, 深度优先搜索, 二叉搜索树, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/validate-binary-search-tree/](https://leetcode.cn/problems/validate-binary-search-tree/)

## 📝 题目描述

给你一棵二叉树的根节点 `root`，判断它是否是一棵有效的二叉搜索树（BST）。

BST 的定义：对于每个节点，左子树中所有节点的值都严格小于该节点的值，右子树中所有节点的值都严格大于该节点的值。左右子树也必须是 BST。

注意是"所有"节点，不是"直接子节点"。也就是说，某个右子树中的左后代也必须大于根。

- 题号：98
- 题目名：验证二叉搜索树
- 难度：中等
- 题型：二叉树
- 为什么选这题：BST 是二叉树中最重要的变种，验证 BST 是理解"上下界约束逐层传递"的关键一步。

## 🧠 思考路径

一种思路是利用 BST 的性质：中序遍历的结果必须是严格递增的。所以先做一遍中序遍历，再检查结果数组是否严格递增。简单直观，但需要额外的 `O(n)` 空间。

更优雅的做法是在遍历的过程中直接传递上下界。对于每个节点，它必须在一个合法的范围内（`lower < node.val < upper`）。进入左子树时，上界更新为当前节点值；进入右子树时，下界更新为当前节点值。

这样不需要额外的数组，也不需要先遍历完再检查，一次遍历即可。

## 💻 Java 题解

### 解法 1：递归 + 范围约束

每个节点的值必须在 (min, max) 范围内。左子树上限更新为当前值，右子树下限更新为当前值。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    public boolean isValidBST(TreeNode root) {
        return valid(root, null, null);
    }
    private boolean valid(TreeNode node, Integer min, Integer max) {
        if (node == null) return true;
        if (min != null && node.val <= min) return false;
        if (max != null && node.val >= max) return false;
        return valid(node.left, min, (Integer) node.val)
            && valid(node.right, (Integer) node.val, max);
    }
}
```

**关键点：**
- 用 Integer 而非 int（处理边界 null）
- 左子树 max=当前值
- 右子树 min=当前值

**执行步骤：**
1. 传入 (min, max) 范围
1. 当前值不在范围内返回 false
1. 递归更新范围约束


### 解法 2：中序遍历（迭代）

BST中序遍历必须严格递增。用一个 prev 变量记录前一个值。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    long prev = Long.MIN_VALUE;
    public boolean isValidBST(TreeNode root) {
        if (root == null) return true;
        if (!isValidBST(root.left)) return false;
        if (root.val <= prev) return false;
        prev = root.val;
        return isValidBST(root.right);
    }
}
```

**关键点：**
- prev 初始化为 Long.MIN_VALUE
- 中序遍历必须严格递增

**执行步骤：**
1. 中序遍历
1. 与 prev 比较
1. 更新 prev


## ⚠️ 易错点

- BST 要求严格大于/小于，不是大于等于/小于等于。`node.val <= lower` 或 `node.val >= upper` 就不合法。
- Java 中节点值是 `int`，范围边界用 `long` 避免溢出。`Long.MIN_VALUE` 和 `Long.MAX_VALUE`。
- 不要只检查"左子节点 < 当前 < 右子节点"。这是必要条件但不是充分条件。必须检查整个子树。
- 如果用中序遍历法，注意比较时要用严格递增（`>` 不是 `>=`）。

- 不能只比较节点和左右子节点——子树中可能有更远的违反

- 用 long 避免整数边界值

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点访问一次。
- 空间复杂度：`O(h)`。递归栈深度等于树的高度。最坏 `O(n)`，平衡树 `O(log n)`。

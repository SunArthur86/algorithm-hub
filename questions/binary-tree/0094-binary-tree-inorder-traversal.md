# 94. 二叉树的中序遍历

> 难度：**Easy** | 分类：二叉树 | 标签：栈, 树, 深度优先搜索, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/binary-tree-inorder-traversal/](https://leetcode.cn/problems/binary-tree-inorder-traversal/)

## 📝 题目描述

给你一棵二叉树的根节点 `root`，返回它的中序遍历结果。

中序遍历的顺序是：先遍历左子树，再访问根节点，最后遍历右子树。即"左 → 根 → 右"。

对于一棵 BST（二叉搜索树），中序遍历的结果天然是递增排序的，这个性质在后续很多题中都会用到。

- 题号：94
- 题目名：二叉树的中序遍历
- 难度：简单
- 题型：二叉树
- 为什么选这题：中序遍历是二叉树三大遍历之一，掌握递归和迭代两种写法，是后续所有树题的基石。对于 BST，中序遍历的结果恰好是升序序列。

## 🧠 思考路径

最直觉的写法是递归：定义一个辅助函数，先递归左子树，把根节点的值加入结果，再递归右子树。代码极其简洁，3 行核心逻辑。

但面试中通常会追问迭代写法。递归的本质是用系统栈保存"返回地址"，迭代写法就是用显式栈来模拟这个过程。

迭代中序遍历的思路：
1. 从根节点出发，一直往左走，把沿途节点都压入栈。
2. 栈顶弹出，访问该节点（加入结果）。
3. 转向该节点的右子树，重复步骤 1。

这个过程模拟了"先走到底，再回头，再转向右边"的递归行为。

## 💻 Java 题解

### 解法 1：递归

递归访问左子树、记录当前节点、递归访问右子树。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        inorder(root, result);
        return result;
    }
    private void inorder(TreeNode node, List<Integer> result) {
        if (node == null) return;
        inorder(node.left, result);
        result.add(node.val);
        inorder(node.right, result);
    }
}
```

**关键点：**
- 左→根→右顺序
- 递归终止条件 node==null

**执行步骤：**
1. 递归左子树
1. 访问当前节点
1. 递归右子树


### 解法 2：迭代（栈）

用栈模拟递归。一路向左走到底，弹出访问，再转向右子树。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    public List<Integer> inorderTraversal(TreeNode root) {
        List<Integer> result = new ArrayList<>();
        Deque<TreeNode> stack = new ArrayDeque<>();
        TreeNode curr = root;
        while (curr != null || !stack.isEmpty()) {
            while (curr != null) {
                stack.push(curr);
                curr = curr.left;
            }
            curr = stack.pop();
            result.add(curr.val);
            curr = curr.right;
        }
        return result;
    }
}
```

**关键点：**
- 一路向左压栈
- 弹出时访问
- 转向右子树

**执行步骤：**
1. 一路向左压栈
1. 弹出栈顶访问
1. 转向右子树继续


## ⚠️ 易错点

- 迭代写法中，内层 `while` 是"一路往左压栈"，外层 `while` 是"整个遍历还没结束"。两个循环的条件不同，不要混用。
- 弹出节点后要转向右子树，而不是左子树。`curr = curr.right` 这一步容易漏。
- 栈里存的是节点引用，不是值。弹出后才能访问值。
- 递归写法中不要忘了 `base case`（节点为空直接返回）。

- 迭代法外层 while 条件是 curr!=null || !stack.isEmpty()

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点访问一次。
- 空间复杂度：`O(h)`。`h` 是树的高度，栈的最大深度等于树的高度。最坏情况（链状树）为 `O(n)`，平衡树为 `O(log n)`。

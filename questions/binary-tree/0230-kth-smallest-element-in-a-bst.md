# 230. 二叉搜索树中第 K 小的元素

> 难度：**Medium** | 分类：二叉树 | 标签：树, 深度优先搜索, 二叉搜索树, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/kth-smallest-element-in-a-bst/](https://leetcode.cn/problems/kth-smallest-element-in-a-bst/)

## 📝 题目描述

给你一棵二叉搜索树的根节点和一个整数 `k`，找出树中第 `k` 小的元素值。

重点不在"怎么排序"，而在"二叉搜索树的中序遍历天然就是升序，走到第 k 个就行"。

- 题号：230
- 题目名：二叉搜索树中第K小的元素
- 难度：中等
- 题型：二叉树
- 为什么选这题：它把二叉搜索树的中序遍历性质用到了极致——BST 的中序遍历天然有序，第 K 个就是第 K 小，是理解 BST 核心特性的必做题。

## 🧠 思考路径

二叉搜索树的性质是：左子树所有节点值 < 根节点值 < 右子树所有节点值。这个性质意味着，对 BST 做中序遍历（左 → 根 → 右），输出的序列天然就是从小到大排序的。

所以找第 `k` 小的元素，只需要做中序遍历，遍历到第 `k` 个时停下来，返回那个节点的值。

可以用递归实现中序遍历，用一个全局计数器 `count` 记录当前是第几个，到 `k` 时记录答案。也可以用迭代方式（显式栈）实现，这样可以在找到答案时立即停止，不用遍历完整棵树。

如果这棵树会被频繁查询第 `k` 小，可以在每个节点上额外维护"左子树节点数"，这样就能二分搜索，每次 `O(log n)` 查询。但单次查询用中序遍历就够了。

这题的关键词不是"排序找第K个"，而是"BST 的中序遍历 = 升序，走到第 K 个就停"。

## 💻 Java 题解

### 解法 1：中序遍历（提前终止）

中序遍历BST，遍历到第k个时返回。不需要遍历完整棵树。

- ⏱ 时间复杂度：`O(h+k)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    int count = 0, result = 0;
    public int kthSmallest(TreeNode root, int k) {
        inorder(root, k);
        return result;
    }
    private void inorder(TreeNode node, int k) {
        if (node == null) return;
        inorder(node.left, k);
        count++;
        if (count == k) { result = node.val; return; }
        if (count < k) inorder(node.right, k);
    }
}
```

**关键点：**
- 中序遍历BST是升序
- count==k 时提前终止

**执行步骤：**
1. 中序遍历
1. 计数到 k 时记录结果
1. 提前终止避免不必要的遍历


## ⚠️ 易错点

- 中序遍历的顺序是"左 → 根 → 右"，别写成"根 → 左 → 右"（前序）。
- `k` 从 1 开始计数，不是从 0 开始。
- 递归版本中，找到答案后要提前终止，不然会多遍历无用的节点。用返回值或标志变量控制。
- 迭代法中，内层 `while` 是一路向左压栈，外层 `while` 是整体遍历控制。

- 找到后要终止递归避免浪费

## 📊 复杂度分析

- 时间复杂度：`O(H + k)`，`H` 为树高。先走 `H` 步到底，再访问 `k` 个节点。最坏 `O(n)`。
- 空间复杂度：`O(H)`，栈的深度等于树高。最坏 `O(n)`（退化为链表）。

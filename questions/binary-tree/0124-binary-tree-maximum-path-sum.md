# 124. 二叉树中的最大路径和

> 难度：**Hard** | 分类：二叉树 | 标签：树, 深度优先搜索, 动态规划, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/binary-tree-maximum-path-sum/](https://leetcode.cn/problems/binary-tree-maximum-path-sum/)

## 📝 题目描述

给你一棵二叉树，每个节点包含一个整数（可能为负）。路径定义为从树中任意节点出发，沿着父子连线到达任意节点的序列。同一条路径中每个节点最多出现一次。求所有路径中节点值之和的最大值。

路径不一定要经过根节点，也不一定要到叶子节点。可以是树中任意一段连续的"向上路径"。

- 题号：124
- 题目名：二叉树中的最大路径和
- 难度：困难
- 题型：二叉树
- 为什么选这题：它是二叉树递归中最考验"返回值 vs 全局答案"区分能力的题目，帮你理解"递归函数返回什么"和"最终答案是什么"可以不同。

## 🧠 思考路径

路径在树中的形状只有两种：要么是一条"直线"（从某个节点往上走），要么是一条"人字形"（从左子树上来，经过当前节点，再下到右子树）。

递归函数的返回值应该是"从当前节点往下走的最大贡献"（直线），而全局答案可能是一条"人字形"路径。

对每个节点，计算：
- 左子树能贡献的最大和 `left_gain = max(0, dfs(left))`（如果左子树贡献是负数，不如不要）
- 右子树能贡献的最大和 `right_gain = max(0, dfs(right))`
- 经过当前节点的"人字形"路径和 = `node.val + left_gain + right_gain`，用这个更新全局答案
- 返回给父节点的"直线"贡献 = `node.val + max(left_gain, right_gain)`（只能选一侧往上传）

## 💻 Java 题解

### 解法 1：DFS + 全局最大值

DFS返回以当前节点为端点的最大贡献值。过程中用 left+right+val 更新全局最大路径和。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    int maxSum = Integer.MIN_VALUE;
    public int maxPathSum(TreeNode root) {
        dfs(root);
        return maxSum;
    }
    private int dfs(TreeNode node) {
        if (node == null) return 0;
        int left = Math.max(0, dfs(node.left));   // 负贡献直接剪掉
        int right = Math.max(0, dfs(node.right));
        maxSum = Math.max(maxSum, left + right + node.val); // 横跨路径
        return Math.max(left, right) + node.val; // 只能选一条路向上
    }
}
```

**关键点：**
- 负贡献取0（不选）
- 横跨路径 = left+right+val
- 返回只能选一边向上

**执行步骤：**
1. DFS求左右子树最大贡献
1. 更新全局最大（横跨路径）
1. 返回 max(left,right)+val


## ⚠️ 易错点

- 递归返回值和全局答案是两个不同的东西。返回值是"向上传的直线贡献"，全局答案是"可能的人字形路径"。
- 子树贡献为负时要取 0，不要直接用负值。负贡献只会拖累路径和。
- `max_sum` 初始化为负无穷，不是 0。因为节点值可能全是负数，最大路径可能就是某个单独的负数节点。
- 不要在递归返回值中包含"左右两侧都选"的情况。返回值只能选一侧。

- 负数贡献要取0

- 返回值和更新全局最大值不同——返回只能选一边

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点只访问一次。
- 空间复杂度：`O(h)`。递归栈深度等于树高。

# 215. 数组中的第K个最大元素

> 难度：**Medium** | 分类：堆 | 标签：数组, 分治, 快速选择, 堆, 排序

> 🔗 力扣链接：[https://leetcode.cn/problems/kth-largest-element-in-an-array/](https://leetcode.cn/problems/kth-largest-element-in-an-array/)

## 📝 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，返回数组中第 `k` 大的元素。

重点不在"会不会排序取第 k 个"，而在"怎么只关心前 k 个大的元素，不排序整个数组"。

- 题号：215
- 题目名：数组中的第K个最大元素
- 难度：中等
- 题型：堆
- 为什么选这题：它是堆的实战入门题，也是面试最高频的 Top-K 问题，必须拿下。

## 🧠 思考路径

最直接的想法，是把数组排序后取第 `k` 个。时间复杂度 `O(n log n)`，能用但不够优雅。

更好的想法是用"最小堆"维护前 `k` 个最大的元素。建一个大小为 `k` 的最小堆，堆顶是当前最小的元素。遍历数组时，如果当前元素比堆顶大，就替换堆顶（弹出小的，推入大的）。遍历结束后，堆顶就是第 `k` 大的元素。

为什么用最小堆而不是最大堆？因为我们要保留前 `k` 个最大的，堆顶是最小的那个"守门员"——只有比它大的才能进来，进来之后它继续守门。

时间复杂度是 `O(n log k)`，当 `k << n` 时远优于排序。空间复杂度 `O(k)`。

还有一个更快的算法：快速选择（QuickSelect），基于快排的分区思想，平均 `O(n)`，但最坏 `O(n²)`。

这题的关键词不是"排序取第 k 个"，而是"用最小堆当守门员，只留前 k 大"。

## 💻 Java 题解

### 解法 1：小顶堆（大小K）

维护大小为K的小顶堆。堆满后新元素比堆顶大就替换。最终堆顶即第K大。

- ⏱ 时间复杂度：`O(n log k)`
- 💾 空间复杂度：`O(k)`

```java
class Solution {
    public int findKthLargest(int[] nums, int k) {
        PriorityQueue<Integer> pq = new PriorityQueue<>();
        for (int num : nums) {
            pq.offer(num);
            if (pq.size() > k) pq.poll();
        }
        return pq.peek();
    }
}
```

**关键点：**
- 小顶堆大小K
- 堆顶是K个中最小的=第K大
- pq.size()>k 弹出堆顶

**执行步骤：**
1. 遍历每个元素入堆
1. 堆超过k弹出最小
1. 返回堆顶


### 解法 2：快速选择（Hoare）

类似快排的partition，每次只递归包含第K大的一侧。平均O(n)。

- ⏱ 时间复杂度：`O(n) avg`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int findKthLargest(int[] nums, int k) {
        return quickSelect(nums, 0, nums.length - 1, k);
    }
    private int quickSelect(int[] nums, int lo, int hi, int k) {
        int pivot = nums[lo], i = lo + 1, j = hi;
        while (i <= j) {
            if (nums[i] < pivot && nums[j] > pivot) swap(nums, i++, j--);
            if (nums[i] >= pivot) i++;
            if (nums[j] <= pivot) j--;
        }
        swap(nums, lo, j);
        if (j == k - 1) return nums[j];
        return j < k - 1 ? quickSelect(nums, j + 1, hi, k) : quickSelect(nums, lo, j - 1, k);
    }
    private void swap(int[] nums, int i, int j) { int t = nums[i]; nums[i] = nums[j]; nums[j] = t; }
}
```

**关键点：**
- partition只递归一侧
- 降序排列，找第K大=k-1位置
- 平均O(n)

**执行步骤：**
1. partition选pivot
1. 判断pivot位置
1. 只递归目标侧


## ⚠️ 易错点

- 是"第 k 大"不是"第 k 小"。第 1 大就是最大值，第 `n` 大就是最小值。
- 最小堆的堆顶是最小值，是"前 k 大中最小的那个"——恰好就是第 k 大。
- `heapreplace` 是先弹出再推入，比分开调用 `heappop` + `heappush` 更高效。
- 快速选择法平均 `O(n)` 但最坏 `O(n²)`，面试时两种方法都要会。

- 第K大对应降序索引k-1

- 快速选择最坏O(n²)，随机化pivot可优化

## 📊 复杂度分析

- 时间复杂度：堆法 `O(n log k)`；快速选择法平均 `O(n)`，最坏 `O(n²)`。
- 空间复杂度：堆法 `O(k)`；快速选择法 `O(1)`（原地分区）。

# 347. 前 K 个高频元素

> 难度：**Medium** | 分类：堆 | 标签：数组, 哈希表, 分治, 桶排序, 计数, 快速选择, 堆, 排序

> 🔗 力扣链接：[https://leetcode.cn/problems/top-k-frequent-elements/](https://leetcode.cn/problems/top-k-frequent-elements/)

## 📝 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，返回数组中出现频率前 k 高的元素。你可以按任意顺序返回答案。

- 题号：347
- 题目名：前 K 个高频元素
- 难度：中等
- 题型：堆
- 为什么选这题：它是"堆"最经典的应用场景——从海量数据中提取 Top K，面试出场率极高。

## 🧠 思考路径

第一步肯定要先统计频率——用哈希表，遍历一遍数组就能得到每个元素的出现次数。

问题变成：从一堆"元素→频率"的键值对中，找出频率最高的 k 个。

方法一：全部排序。按频率从高到低排序，取前 k 个。时间 `O(n log n)`，简单但不是最优。

方法二：用堆（优先队列）。维护一个大小为 k 的最小堆。遍历所有频率对，把当前元素加入堆；如果堆的大小超过 k，就把堆顶（频率最小的）弹出去。遍历结束后，堆里剩下的就是频率最高的 k 个。时间 `O(n log k)`，当 k 远小于 n 时优势明显。

方法三：桶排序。频率最大不会超过 n，所以建 n+1 个桶，第 i 个桶放所有频率为 i 的元素。从高频率桶往低频率桶收集，收集够 k 个就停。时间 `O(n)`，但空间开销大一些。

这里给堆解法，因为它最通用、面试最常考。

## 💻 Java 题解

### 解法 1：频率Map + 小顶堆

统计频率后，用大小K的小顶堆按频率排序，保留频率最高的K个。

- ⏱ 时间复杂度：`O(n log k)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int[] topKFrequent(int[] nums, int k) {
        Map<Integer, Integer> freq = new HashMap<>();
        for (int n : nums) freq.merge(n, 1, Integer::sum);
        PriorityQueue<int[]> pq = new PriorityQueue<>((a, b) -> a[1] - b[1]);
        for (var entry : freq.entrySet()) {
            pq.offer(new int[]{entry.getKey(), entry.getValue()});
            if (pq.size() > k) pq.poll();
        }
        int[] result = new int[k];
        for (int i = 0; i < k; i++) result[i] = pq.poll()[0];
        return result;
    }
}
```

**关键点：**
- 先统计频率
- 小顶堆按频率排序
- 堆大小K保留最高频

**执行步骤：**
1. 统计频率
1. 入小顶堆（按频率）
1. 保留K个
1. 返回堆中元素


## ⚠️ 易错点

- 堆的类型一定要是最小堆（默认就是），不是最大堆。因为我们要"淘汰频率最小的"。
- Java 中 PriorityQueue 默认就是最小堆，比较器 `freq.get(a) - freq.get(b)` 是升序。
- 返回的是元素值，不是频率值。
- 题目保证答案唯一，不需要处理频率相同的情况。

- Comparator比较频率不是值

- 堆中存[val, freq]对

## 📊 复杂度分析

- 时间复杂度：`O(n log k)`。统计频率 `O(n)`，堆操作最多 n 次，每次 `O(log k)`。
- 空间复杂度：`O(n)`。哈希表存频率 `O(n)`，堆最多 `O(k)`。

# 142. 环形链表 II

> 难度：**Medium** | 分类：链表 | 标签：哈希表, 链表, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/linked-list-cycle-ii/](https://leetcode.cn/problems/linked-list-cycle-ii/)

## 📝 题目描述

给定一个链表，返回链表开始入环的第一个节点。如果链表无环，则返回 `null`。

不允许修改链表。重点不在"判断有没有环"（那是 141 题），而在"环从哪里开始"。

- 题号：142
- 题目名：环形链表 II
- 难度：中等
- 题型：链表
- 为什么选这题：它在 141 题基础上加了"找环入口"的需求，快慢指针相遇后的数学推导非常精妙。

## 🧠 思考路径

先用 141 题的方法，快慢指针找到相遇点。但相遇点不一定是环的入口。

设链表头到环入口距离为 `a`，环入口到相遇点距离为 `b`，相遇点到环入口剩余距离为 `c`。环的长度 = `b + c`。

慢指针走的距离：`a + b`。
快指针走的距离：`a + b + k(b + c)`（快指针在环里绕了 `k` 圈）。
快指针走的距离是慢指针的两倍：`a + b + k(b + c) = 2(a + b)`。

化简得：`a = (k - 1)(b + c) + c`。

这意味着：从链表头走 `a` 步等于从相遇点走 `c + (k-1)圈` 的距离，都到达环入口。

所以，让一个指针从链表头出发，另一个指针从相遇点出发，两者每次都走一步，它们一定会在环入口相遇。

所以这题的关键词不是"找到环在哪"，而是"相遇后，一个回起点一个留在原地，同步走就会在入口碰面"。

## 💻 Java 题解

### 解法 1：快慢指针 + 数学推导

快慢指针相遇后，一个指针从头出发，另一个从相遇点出发，同速走再次相遇即为环入口。数学证明：a=(n-1)(b+c)+c。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
public class Solution {
    public ListNode detectCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) {
                ListNode p = head;
                while (p != slow) { p = p.next; slow = slow.next; }
                return p;
            }
        }
        return null;
    }
}
```

**关键点：**
- 第一次相遇后
- 一个指针从头开始
- 同速走再次相遇即入口

**执行步骤：**
1. 快慢指针找相遇点
1. 一个指针放回头部
1. 同速走直到相遇
1. 相遇点即环入口


## ⚠️ 易错点

- 第二阶段两个指针都是从"链表头"和"相遇点"同时出发，不是从"头"和"环入口"出发。
- 找到相遇点后，`slow` 不要重置为 `head`，应该留在相遇点，新开一个 `ptr` 从 `head` 出发。
- 数学推导中 `k ≥ 1`（快指针至少绕了一圈），但不影响结论。
- 链表只有两个节点形成环时，也要正确处理。

- 相遇判断不能在初始位置——要先走

- 第二次相遇用同速不是快慢

## 📊 复杂度分析

- 时间复杂度：`O(n)`。找相遇点最多 `O(n)`，找入口最多 `O(n)`。
- 空间复杂度：`O(1)`。只用了几个指针变量。

# 141. 环形链表

> 难度：**Easy** | 分类：链表 | 标签：哈希表, 链表, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/linked-list-cycle/](https://leetcode.cn/problems/linked-list-cycle/)

## 📝 题目描述

给你一个链表的头节点 `head`，判断链表中是否有环。如果链表存在环，则返回 `True`；否则返回 `False`。

环的定义：链表中某个节点的 `next` 指针通过一系列操作可以再次回到该节点。重点不在"找到环在哪"，而在"判断有没有环"。

- 题号：141
- 题目名：环形链表
- 难度：简单
- 题型：链表
- 为什么选这题：它是"快慢指针"最经典的应用，一句话就能建立"两个速度不同的人在环形跑道上必相遇"的直觉。

## 🧠 思考路径

最直觉的做法是用一个集合记录走过的节点：每走一步就查一下当前节点是否出现过。出现过就说明有环。时间和空间都是 `O(n)`。

但题目鼓励 `O(1)` 空间。有没有不用额外空间的办法？

想象两个人在环形跑道上跑步，一个人跑得快，一个人跑得慢。只要跑道是环形的，快的人一定会从后面追上慢的人。如果跑道是直线，快的人会先到终点，两人永远不会相遇。

这就是快慢指针法：慢指针每走一步，快指针走两步。如果链表有环，快指针最终会和慢指针在环内某个位置相遇。如果没环，快指针会先到达链表末尾。

所以这题的关键词不是"遍历链表"，而是"两个速度不同的指针在环里必相遇"。

## 💻 Java 题解

### 解法 1：快慢指针（Floyd判圈）

slow 走一步，fast 走两步。如果有环，fast 必然追上 slow。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
public class Solution {
    public boolean hasCycle(ListNode head) {
        ListNode slow = head, fast = head;
        while (fast != null && fast.next != null) {
            slow = slow.next;
            fast = fast.next.next;
            if (slow == fast) return true;
        }
        return false;
    }
}
```

**关键点：**
- 快指针走两步，慢指针走一步
- 相遇则有环
- fast 或 fast.next 为 null 则无环

**执行步骤：**
1. slow=head, fast=head
1. slow+1, fast+2
1. slow==fast 返回 true
1. fast 到 null 返回 false


## ⚠️ 易错点

- 循环条件是 `fast and fast.next`，不能只写 `fast`，因为快指针走两步，需要保证 `fast.next` 不为 null。
- 判断相遇用 `is`（Python）/ `==`（Java）比较的是节点引用，不是节点值。
- 快慢指针初始都指向 `head`，不能快指针从 `head.next` 开始，否则可能跳过相遇点。
- 如果链表为空或只有一个节点（且无环），应该直接返回 `False`。

- 要先走再比较，否则初始就相等

- while 条件检查 fast 和 fast.next

## 📊 复杂度分析

- 时间复杂度：`O(n)`。无环时快指针最多走 `n/2` 步到达末尾；有环时在环内最多走环长度的若干圈后相遇。
- 空间复杂度：`O(1)`。只用了两个指针变量。

## 🏢 实际业务应用

- 死锁检测：操作系统把进程的资源等待关系建模为有向图（类似链表），检测图中是否有环就是检测是否存在死锁。
- 垃圾回收中的引用计数：对象之间的循环引用形成环，需要检测和处理。
- 网络路由环路检测：路由表中如果存在环路，数据包会无限转发，快慢指针的思想可以用于检测路径中的环。

可以把它类比成《西游记》里孙悟空和猪八戒在迷宫中探路。

孙悟空跑得快（快指针），一次跨两间房；猪八戒走得慢（慢指针），一次走一间房。迷宫要么是一条直通出口的路（无环链表），要么会绕回已经走过的房间（有环链表）。如果是直线，孙悟空一定会先到出口；如果迷宫有环形通道，孙悟空绕了一圈后一定会从后面追上还在慢走的猪八戒——两人一碰面，就知道这里有环。

- "迷宫"对应链表
- "孙悟空"对应快指针，每次走两步
- "猪八戒"对应慢指针，每次走一步
- "碰面"对应 `slow == fast`，判定有环
- "到出口"对应 `fast == null`，判定无环

这段记忆的关键是：两个速度不同的人在环形跑道上一定会相遇。

# 206. 反转链表

> 难度：**Easy** | 分类：链表 | 标签：递归, 链表

> 🔗 力扣链接：[https://leetcode.cn/problems/reverse-linked-list/](https://leetcode.cn/problems/reverse-linked-list/)

## 📝 题目描述

给你一个单链表的头节点 `head`，反转这个链表，返回反转后的头节点。

重点不在"会不会翻转概念"，而在"怎么用三个指针安全地把每个节点的指向反过来"。

- 题号：206
- 题目名：反转链表
- 难度：简单
- 题型：链表
- 为什么选这题：它是链表操作最重要的基础题，没有之一。搞懂它，链表的"指针搬移"就算入门了。

## 🧠 思考路径

最直接的想法，是把链表的值存到数组里，倒过来再建链表。这确实能过，但浪费了 `O(n)` 的额外空间，而且失去了练习指针操作的机会。

正解的思路很纯粹：遍历链表，每到一个节点，就把它指向的下一个节点改成指向它前面的节点。问题在于：修改 `next` 指针时，原来的下一个节点就丢了。所以需要三个指针配合工作：`prev`（前驱）、`curr`（当前）、`next`（暂存后继）。

也可以递归地做：假设后面的部分已经反转好了，我只需要让当前节点的 `next` 指向自己，再把自己指向 `null`。递归的终止条件是链表为空或只有一个节点。

这题的关键词不是"反转"，而是"三个指针的精准配合"。

## 💻 Java 题解

### 解法 1：迭代（推荐）

用 prev=null, curr=head。每次保存 next，翻转 curr.next=prev，然后 prev=curr, curr=next。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        ListNode prev = null, curr = head;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr;
            curr = next;
        }
        return prev;
    }
}
```

**关键点：**
- 三指针：prev, curr, next
- 先存 next 再翻转
- prev 最终是新头节点

**执行步骤：**
1. prev=null, curr=head
1. 保存 next=curr.next
1. 翻转 curr.next=prev
1. 前进 prev=curr, curr=next


### 解法 2：递归

递归反转后面的链表，然后让当前节点的 next 指回自己。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public ListNode reverseList(ListNode head) {
        if (head == null || head.next == null) return head;
        ListNode newHead = reverseList(head.next);
        head.next.next = head;
        head.next = null;
        return newHead;
    }
}
```

**关键点：**
- 递归到底再回溯翻转
- head.next.next=head 是核心
- head.next=null 断开原方向

**执行步骤：**
1. 递归到链表末尾
1. 回溯时 head.next.next=head
1. head.next=null


## ⚠️ 易错点

- 一定要在修改 `curr.next` 之前暂存 `curr.next`，否则后继节点就丢了。
- 迭代法返回的是 `prev` 不是 `curr`，因为 `curr` 最终会变成 `null`。
- 递归法中 `head.next.next = head` 是让下一个节点指向自己，不要写成 `head.next = head`。
- 空链表和单节点链表直接返回 `head`。

- 迭代法忘记先保存 next

- 递归法忘记 head.next=null

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点访问一次。
- 空间复杂度：迭代法 `O(1)`；递归法 `O(n)`（递归栈深度）。

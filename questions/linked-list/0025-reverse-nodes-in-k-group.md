# 25. K 个一组翻转链表

> 难度：**Hard** | 分类：链表 | 标签：递归, 链表

> 🔗 力扣链接：[https://leetcode.cn/problems/reverse-nodes-in-k-group/](https://leetcode.cn/problems/reverse-nodes-in-k-group/)

## 📝 题目描述

给你一条链表和一个整数 K，每 K 个节点为一组进行翻转。如果最后剩余的节点不足 K 个，就保持原样不动。

比如链表 `1→2→3→4→5`，K=3，结果是 `3→2→1→4→5`。前三个人翻转为 `3→2→1`，后两个不够三个，保持 `4→5` 不动。

- 题号：25
- 题目名：K 个一组翻转链表
- 难度：困难
- 题型：链表
- 为什么选这题：它是链表操作的集大成者——翻转、分组、指针重连、边界处理全都有，能把链表基本功一次拉满。

## 🧠 思考路径

先拆成子问题：能不能先翻转一条链表的前 K 个节点？这是经典的"头插法"翻转链表，用三个指针 `prev`、`curr`、`next` 就能搞定。

然后考虑分组：翻完一组之后，前一组的尾巴要接到下一组的翻转结果上。这就需要记住前一组的尾节点（翻转后它变成了这组的最后一个节点），等下一组翻完，直接把前一组尾巴的 next 指过去。

还要处理边界：翻之前先数一下够不够 K 个节点，不够就不翻直接接上去。

这样整件事就拆成了三个步骤：先数够不够 K 个 → 翻转这 K 个 → 递归处理剩下的。递归的思路最清晰：翻转前 K 个，返回新的头，然后递归处理后半段，把前半段的尾接上递归结果。

## 💻 Java 题解

### 解法 1：迭代分组翻转

每次找到 k 个节点的区间，翻转这一段，连接前后。用 dummy 简化。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public ListNode reverseKGroup(ListNode head, int k) {
        ListNode dummy = new ListNode(0, head);
        ListNode prev = dummy;
        while (true) {
            // 检查是否有 k 个节点
            ListNode tail = prev;
            for (int i = 0; i < k; i++) {
                tail = tail.next;
                if (tail == null) return dummy.next;
            }
            ListNode next = tail.next;
            // 翻转 prev.next 到 tail
            ListNode curr = prev.next, newTail = curr;
            prev.next = null;
            for (int i = 0; i < k; i++) {
                ListNode temp = curr.next;
                curr.next = prev.next;
                prev.next = curr;
                curr = temp;
            }
            newTail.next = next;
            prev = newTail;
        }
    }
}
```

**关键点：**
- 先检查是否有 k 个
- 翻转区间内链表
- 连接前后段

**执行步骤：**
1. 检查是否有 k 个节点
1. 翻转这 k 个
1. 连接前段和后段
1. prev 移到当前段的末尾


## ⚠️ 易错点

- 翻转前的"数够不够 K 个"这步不能省。如果不检查就直接翻转，最后不足 K 个的组也会被翻转，答案就错了。
- 翻转之后，`head` 变成了这组的最后一个节点，它的 `next` 必须接上后面递归/迭代的返回值，否则链表在这里就断了。
- 迭代版本中，翻转一组时 `prev` 初始值应该是什么？是 `prevGroup`（前一组的尾/虚拟头），不是 `null`。如果写成 `null`，翻转后第一个节点会指向 `null` 而不是前一组，链表就断了。

- 不足 k 个不翻转

- 翻转后要正确连接前后段

## 📊 复杂度分析

- 时间复杂度：`O(n)`，每个节点最多被访问两次（一次数够不够，一次翻转）。
- 空间复杂度：`O(n/k)`（递归版本），递归深度为组数。迭代版本是 `O(1)`。

## 🏢 实际业务应用

- **大数据分批处理**：Apache Spark、Flink 在处理流式数据时，经常按固定大小的 batch 分组处理，每组内部可能需要倒序回放（比如时间窗口内的日志要按时间从新到旧展示），分组翻转的逻辑和本题一致。
- **消息队列的消费确认**：Kafka、RabbitMQ 在消费消息时，支持按批次确认（batch ack）。如果需要回滚某个批次中的消息顺序（比如按照某种优先级重排），就需要对链式存储的消息进行分组翻转。
- **区块链的交易池重排**：以太坊、比特币节点的交易池（mempool）中，矿工会按 gas price 对交易分组排序，某些策略需要对相邻组内的交易列表翻转（比如把低优先级的放到前面先处理以填满区块空间）。

# 23. 合并 K 个升序链表

> 难度：**Hard** | 分类：链表 | 标签：链表, 分治, 堆, 归并排序

> 🔗 力扣链接：[https://leetcode.cn/problems/merge-k-sorted-lists/](https://leetcode.cn/problems/merge-k-sorted-lists/)

## 📝 题目描述

给你 K 条已经排好序的链表，要把它们合成一条，合成之后也得是升序的。

重点不是"会不会合并"，而是"K 条链表的情况下，怎么每次都快速找到最小的那个节点"。

- 题号：23
- 题目名：合并 K 个升序链表
- 难度：困难
- 题型：链表
- 为什么选这题：它是"合并两个有序链表"的直接升级版，能让你一次学会堆（优先队列）和分治两种思路，是从中等跳到困难的经典跳板。

## 🧠 思考路径

最朴素的想法：每次从 K 条链表的当前头节点中，遍历找出最小值，取下来接到结果链表末尾，重复直到所有链表都为空。这样做每找一个节点要比较 K 次，总共 N 个节点，时间复杂度 `O(NK)`，K 大了就很慢。

能不能更快地找到"当前 K 个头节点中的最小值"？这就想到了最小堆（优先队列）。把 K 个头节点全扔进堆里，每次弹出的就是全局最小，再把弹出节点的 next 塞回堆里。堆操作是 `O(logK)`，总时间 `O(NlogK)`，大幅优化。

另一种思路是分治：两两合并，像锦标赛一样层层向上。第一轮 K 条变成 K/2 条，第二轮 K/2 变成 K/4 条……总共 `logK` 轮，每轮所有节点参与一次合并，也是 `O(NlogK)`。

两种方法复杂度一样，但堆的写法更直觉，分治的写法更优雅。

## 💻 Java 题解

### 解法 1：小顶堆（优先队列）

把每个链表的头节点放入小顶堆。每次弹出最小的，接到结果后面，然后把它的 next 放入堆。

- ⏱ 时间复杂度：`O(N log k)`
- 💾 空间复杂度：`O(k)`

```java
class Solution {
    public ListNode mergeKLists(ListNode[] lists) {
        PriorityQueue<ListNode> pq = new PriorityQueue<>((a, b) -> a.val - b.val);
        for (ListNode l : lists) if (l != null) pq.offer(l);
        ListNode dummy = new ListNode(0), tail = dummy;
        while (!pq.isEmpty()) {
            ListNode node = pq.poll();
            tail.next = node;
            tail = tail.next;
            if (node.next != null) pq.offer(node.next);
        }
        return dummy.next;
    }
}
```

**关键点：**
- 堆中始终有 k 个元素（每个链表一个）
- 弹出最小→接上→放入 next
- O(N log k)

**执行步骤：**
1. 所有头节点入堆
1. 弹最小节点接到结果
1. 该节点 next 入堆
1. 重复直到堆空


## ⚠️ 易错点

- Python 的 `heapq` 在元组比较时，如果 `val` 相同会继续比较下一个元素。`ListNode` 不支持比较，会报错。解决办法是在元组中加一个唯一的 `index` 字段来打破平局。
- 输入的 `lists` 可能包含空链表（`None`/`null`），入堆前必须判空，否则空指针异常。
- 分治写法里，`left == right` 是递归终止条件，漏掉这一步会无限递归。

- 空链表不要入堆

- Comparator 用 lambda 比较值

## 📊 复杂度分析

- 时间复杂度：`O(NlogK)`，其中 N 是所有节点的总数，K 是链表条数。每个节点入堆出堆各一次，每次堆操作 `O(logK)`。
- 空间复杂度：`O(K)`（堆方案），堆里最多同时存 K 个节点。分治方案的空间是 `O(logK)` 的递归栈深度。

# 21. 合并两个有序链表

> 难度：**Easy** | 分类：链表 | 标签：递归, 链表

> 🔗 力扣链接：[https://leetcode.cn/problems/merge-two-sorted-lists/](https://leetcode.cn/problems/merge-two-sorted-lists/)

## 📝 题目描述

给你两个已经按升序排好的链表 `list1` 和 `list2`，把它们合并成一个新的有序链表并返回。

- 题号：21
- 题目名：合并两个有序链表
- 难度：简单
- 题型：链表
- 为什么选这题：它是最基础的链表合并操作，是后续"合并 K 个链表"的基础。

## 🧠 思考路径

最自然的想法是用两个指针分别指向两个链表的头，每次比较两个指针指向的节点值，较小的那个接到结果链表上，然后该指针后移。重复直到其中一个链表走完，把另一个链表剩余部分直接接上。

可以用迭代或递归两种方式实现。递归更简洁：每次选较小的头节点，它的 `next` 指向剩余部分的合并结果。

## 💻 Java 题解

### 解法 1：迭代 + dummy 节点

创建 dummy 头节点，逐个比较两个链表的节点，较小的接到结果链表后面。

- ⏱ 时间复杂度：`O(m+n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
        ListNode dummy = new ListNode(0), tail = dummy;
        while (list1 != null && list2 != null) {
            if (list1.val <= list2.val) { tail.next = list1; list1 = list1.next; }
            else { tail.next = list2; list2 = list2.next; }
            tail = tail.next;
        }
        tail.next = list1 != null ? list1 : list2;
        return dummy.next;
    }
}
```

**关键点：**
- dummy 节点简化头节点处理
- 比较值小的先接
- 剩余部分直接拼接

**执行步骤：**
1. 创建 dummy 头
1. 比较两个链表当前节点
1. 较小的接到 tail 后面
1. 剩余直接拼接


## ⚠️ 易错点

- 用哨兵节点避免处理"第一个节点接谁"的边界情况。
- 某个链表走完后，另一个链表剩余部分直接接上即可，不需要逐个节点处理。
- 递归版本要注意栈深度，链表很长时可能栈溢出。

- 忘记 dummy 节点导致头节点处理复杂

- 剩余链表直接接上不用再遍历

## 📊 复杂度分析

- 时间复杂度：`O(m + n)`。两个链表各遍历一次。
- 空间复杂度：迭代 `O(1)`，递归 `O(m + n)`（栈空间）。

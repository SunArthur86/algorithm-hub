e:T192b,# 295. 数据流的中位数

> 难度：**Hard** | 分类：堆 | 标签：设计, 双指针, 数据流, 排序, 堆

> 🔗 力扣链接：[https://leetcode.cn/problems/find-median-from-data-stream/](https://leetcode.cn/problems/find-median-from-data-stream/)

## 📝 题目描述

设计一个数据结构，支持两种操作：`addNum(num)` 添加一个数字，`findMedian()` 返回当前所有数字的中位数。数据流中的数字数量最大为 10^5。

重点不在"怎么排序找中位数"，而在"怎么在 O(log n) 内插入并 O(1) 取中位数"。

- 题号：295
- 题目名：数据流的中位数
- 难度：困难
- 题型：堆
- 为什么选这题：它是"双堆"技巧的代表作，用一个最大堆和一个最小堆把数据流一分为二，中位数在堆顶直接取。

## 🧠 思考路径

如果每次都排序，`addNum` 是 O(n log n)，太慢。如果用有序数据结构（如平衡 BST），复杂度可以，但实现复杂。

最优雅的方案是双堆：一个大顶堆存较小的一半，一个小顶堆存较大的一半。保持两个堆大小差不超过 1，中位数就是较大的那个堆顶或两个堆顶的平均。

添加元素时：先和大顶堆顶比较，小的放大顶堆，大的放小顶堆。然后平衡两个堆的大小。

所以这题的关键词不是"排序找中间"，而是"大顶堆装左半、小顶堆装右半，中位数在堆顶"。

## 💻 Java 题解

### 解法 1：双堆（大顶+小顶）

大顶堆存较小的一半，小顶堆存较大的一半。两堆大小差不超过1。大顶堆可以多一个。

- ⏱ 时间复杂度：`O(log n) add`
- 💾 空间复杂度：`O(n)`

```java
class MedianFinder {
    PriorityQueue<Integer> left;  // 大顶堆（较小的一半）
    PriorityQueue<Integer> right; // 小顶堆（较大的一半）
    public MedianFinder() {
        left = new PriorityQueue<>(Collections.reverseOrder());
        right = new PriorityQueue<>();
    }
    public void addNum(int num) {
        left.offer(num);
        right.offer(left.poll()); // 先放大顶堆，再把大顶堆最大值移到小顶堆
        if (left.size() < right.size()) left.offer(right.poll()); // 平衡
    }
    public double findMedian() {
        return left.size() > right.size() ? left.peek() : (left.peek() + right.peek()) / 2.0;
    }
}
```

**关键点：**
- left大顶堆存较小半，right小顶堆存较大半
- 先入left→移到right→平衡回left
- left可以多一个

**执行步骤：**
1. 新元素先入left
1. left.poll移到right
1. 若left少则right.poll移回left
1. 中位数看堆顶


## ⚠️ 易错点

- Python 的 `heapq` 是小顶堆，大顶堆用存负数来模拟。
- 平衡时注意方向：大顶堆多出来的移到小顶堆，小顶堆多出来的移到大顶堆。
- 取中位数时注意整数除法和浮点除法的区别。

- 平衡条件：left.size() >= right.size()

- addNum的三步操作顺序不能错

## 📊 复杂度分析

- 时间复杂度：`addNum` 为 `O(log n)`，`findMedian` 为 `O(1)`。
- 空间复杂度：`O(n)`。

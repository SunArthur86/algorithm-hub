> 难度：**Easy** | 分类：二叉树 | 标签：树, 深度优先搜索, 广度优先搜索, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/invert-binary-tree/](https://leetcode.cn/problems/invert-binary-tree/)

## 📝 题目描述

给你一棵二叉树的根节点，翻转这棵树，即交换每个节点的左右子树。

重点不在"怎么翻转"，而在"怎么用递归让每个节点都做同样的交换操作"。

- 题号：226
- 题目名：翻转二叉树
- 难度：简单
- 题型：二叉树
- 为什么选这题：它是最经典的二叉树递归入门题——"对每个节点做同一件事"，三行代码就能解决，但理解它的递归思维是学好所有树题的基础。

## 🧠 思考路径

这题的直觉非常直接：对每个节点，把它的左孩子和右孩子交换一下就行了。问题是：怎么确保整棵树都被翻转？

答案就是递归。对根节点，先交换它的左右孩子。交换完之后，左子树和右子树各自内部还没有翻转，所以对左子树递归做同样的操作，再对右子树递归做。递归到叶子节点（空节点）时自然停止。

也可以用迭代的方式，借助队列做层序遍历：每个节点出队时交换它的左右孩子，如果有孩子就入队。效果完全一样。

这题之所以出名，还因为一个典故：Homebrew 的作者 Max Howell 在 Google 面试时没写出这题，被拒了。他在推特上说"Google: 90% of our engineers use the software you wrote (Homebrew), but you can't invert a binary tree on a whiteboard so f*** off"，让这道题成为了算法面试文化的标志。

这题的关键词不是"翻转"，而是"递归的本质就是让每个节点做同一件简单的事"。

## 💻 Java 题解

### 解法 1：递归交换

交换当前节点的左右子树，递归翻转左右子树。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    public TreeNode invertTree(TreeNode root) {
        if (root == null) return null;
        TreeNode temp = root.left;
        root.left = invertTree(root.right);
        root.right = invertTree(temp);
        return root;
    }
}
```

**关键点：**
- 交换左右子树
- 递归处理子树

**执行步骤：**
1. 保存左子树引用
1. left=递归翻转右子树
1. right=递归翻转保存的左子树


## ⚠️ 易错点

- Python 的 `root.left, root.right = root.right, root.left` 可以一行完成交换；Java 需要临时变量，不能连续赋值。
- 递归时交换和递归的顺序不影响结果——先交换再递归，或先递归再交换，都可以。
- 空节点必须判断，否则访问 `null.left` 会报错。

- 先保存 left 再赋值，否则会丢失引用

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点恰好被访问一次。
- 空间复杂度：递归法 `O(h)`（`h` 为树高，递归栈深度），最坏 `O(n)`（退化为链表）；迭代法 `O(n)`（队列最多存一层节点）。

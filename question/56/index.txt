> 难度：**Medium** | 分类：普通数组 | 标签：数组, 排序

> 🔗 力扣链接：[https://leetcode.cn/problems/merge-intervals/](https://leetcode.cn/problems/merge-intervals/)

## 📝 题目描述

给你一堆区间，每个区间有起点和终点。如果两个区间有重叠（或首尾相接），就把它们合并成一个更大的区间。最终返回一组互不重叠的区间，且这组区间恰好覆盖原来所有区间。

- 题号：56
- 题目名：合并区间
- 难度：中等
- 题型：数组
- 为什么选这题：区间合并是面试高频考点，"先排序再扫描"的套路在日历、排班等场景反复出现。

## 🧠 思考路径

直接的想法：两两比较区间，有重叠就合并。但这样 `O(n²)` 比较太慢，而且合并后产生的新区间可能又要和之前的比较，搞不清楚顺序。

关键观察：如果按起点从小到大排序，重叠的区间一定会相邻。排序后从左到右扫描，拿"当前区间"和"结果里最后一个区间"比较：如果当前起点 <= 上一条的终点，说明有重叠，合并（更新终点为两者终点的较大值）；否则，没有重叠，把当前区间直接加入结果。

排序保证了"前面的区间终点一定 <= 后面的区间终点"吗？不一定，但排序后前面的起点一定 <= 后面的起点，所以只需看终点即可。

## 💻 Java 题解

### 解法 1：排序 + 合并

按区间起点排序。遍历时如果当前区间起点<=上一个的终点，合并；否则新增。

- ⏱ 时间复杂度：`O(n log n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int[][] merge(int[][] intervals) {
        Arrays.sort(intervals, (a, b) -> a[0] - b[0]);
        List<int[]> result = new ArrayList<>();
        for (int[] interval : intervals) {
            if (!result.isEmpty() && interval[0] <= result.get(result.size()-1)[1])
                result.get(result.size()-1)[1] = Math.max(result.get(result.size()-1)[1], interval[1]);
            else
                result.add(interval);
        }
        return result.toArray(new int[0][]);
    }
}
```

**关键点：**
- 按起点排序
- 重叠条件：当前start<=上一end
- 合并时取较大的end

**执行步骤：**
1. 按起点排序
1. 遍历比较与前一个的终点
1. 重叠则合并，否则新增


## ⚠️ 易错点

- 必须先排序！不排序就直接扫描，会漏掉不相邻的重叠区间。
- 合并时终点要取 `max`，不是简单替换：比如 `[1, 5]` 和 `[2, 3]` 重叠，合并后是 `[1, 5]`，不是 `[1, 3]`。
- 题目说"首尾相接也算重叠"：`[1, 4]` 和 `[4, 5]` 要合并成 `[1, 5]`，判断条件是 `<=`，不是 `<`。

- 合并时要更新end为max不是覆盖

## 📊 复杂度分析

- 时间复杂度：`O(n log n)`。排序占主导，扫描是 `O(n)`。
- 空间复杂度：`O(n)`。最坏情况所有区间都不重叠，结果数组大小为 `n`（不算排序的额外空间）。

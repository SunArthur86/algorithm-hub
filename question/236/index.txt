e:T1c26,# 236. 二叉树的最近公共祖先

> 难度：**Medium** | 分类：二叉树 | 标签：树, 深度优先搜索, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/)

## 📝 题目描述

给你一棵普通二叉树和两个节点 `p`、`q`，找到它们的最近公共祖先（LCA）。

重点不在"怎么遍历"，而在"怎么利用递归的返回值来判断：当前节点是不是 LCA"。

- 题号：236
- 题目名：二叉树的最近公共祖先
- 难度：中等
- 题型：二叉树
- 为什么选这题：它是二叉树递归的集大成者——后序遍历"左右根"的顺序天然适合"从下往上找祖先"，是理解递归返回值的经典题目。

## 🧠 思考路径

对于 BST（235 题），可以利用大小关系直接决定搜索方向。但普通二叉树没有这个信息，必须遍历整棵树。

关键想法是后序遍历（左 → 右 → 根）。后序遍历天然是"自底向上"的：先处理完左右子树，再处理当前节点。这意味着当我们在某个节点时，已经知道 `p` 和 `q` 分别在左子树和右子树的哪里了。

递归函数的返回值设计：
- 如果当前节点是 `p` 或 `q`，返回当前节点。
- 如果当前节点是 `null`，返回 `null`。
- 递归左右子树，得到 `left` 和 `right`。
- 如果 `left` 和 `right` 都非空，说明 `p` 和 `q` 分布在两侧，当前节点就是 LCA。
- 如果只有 `left` 非空，说明 `p` 和 `q` 都在左子树（或其中一个就是 `left`），返回 `left`。
- 如果只有 `right` 非空，同理返回 `right`。

这题的关键词不是"找公共祖先"，而是"后序遍历的自底向上特性 + 返回值的设计"。

## 💻 Java 题解

### 解法 1：递归后序遍历

递归查找p和q。如果当前节点等于p或q返回。如果左右子树各返回非null，当前节点就是LCA。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(h)`

```java
class Solution {
    public TreeNode lowestCommonAncestor(TreeNode root, TreeNode p, TreeNode q) {
        if (root == null || root == p || root == q) return root;
        TreeNode left = lowestCommonAncestor(root.left, p, q);
        TreeNode right = lowestCommonAncestor(root.right, p, q);
        if (left != null && right != null) return root;
        return left != null ? left : right;
    }
}
```

**关键点：**
- root==p 或 root==q 时返回
- 左右都非null→当前是LCA
- 一边为null→答案在另一边

**执行步骤：**
1. 递归在左右子树中找p和q
1. 如果当前是p或q直接返回
1. 左右都找到→当前是LCA
1. 只有一边找到→返回那一边


## ⚠️ 易错点

- 基本条件 `root == p or root == q` 必须在递归之前判断，这是搜索终止的关键。
- "左右都非空"是唯一能确定 LCA 的情况——此时当前节点就是答案。
- 如果左为空右非空，返回右；右为空左非空，返回左。不要认为"找到了一个就是答案"——可能另一个还在更深处。
- `p` 可以是 `q` 的祖先，这种情况下递归会先找到 `p` 并向上返回，不会出错。

- p或q本身就是祖先的情况也能处理——因为遇到就返回

## 📊 复杂度分析

- 时间复杂度：`O(n)`。最坏情况需要遍历所有节点。
- 空间复杂度：`O(h)`，`h` 为树高。递归栈的深度。

e:T1f6e,# 1. 两数之和

> 难度：**Easy** | 分类：哈希 | 标签：数组, 哈希表

> 🔗 力扣链接：[https://leetcode.cn/problems/two-sum/](https://leetcode.cn/problems/two-sum/)

## 📝 题目描述

给你一个数组和一个目标值 `target`。你要在数组里找出两个数，让它们加起来正好等于 `target`，并返回这两个数的下标。

重点不在"会不会加法"，而在"怎么更快地找到那两个数"。

- 题号：1
- 题目名：两数之和
- 难度：简单
- 题型：哈希
- 为什么选这题：它是哈希表最好的入门题，能最快建立"边遍历，边查询"的感觉。

## 🧠 思考路径

最直接的想法，是把每两个数都试一遍，看它们加起来是不是 `target`。这种做法叫暴力枚举，时间复杂度是 `O(n²)`。数组一长，就会慢得很明显。

这时要换个角度想。假设我现在看到一个数 `x`，那我真正想知道的不是"它能和谁配"，而是"我还缺一个 `target - x`"。如果我能很快知道"前面有没有出现过这个缺的数"，那就不用回头把所有数再翻一遍。

这正是哈希表擅长的事。我们一边遍历数组，一边把已经见过的数记到哈希表里。每来一个新数，就先查它的补数在不在表里；在，就直接得到答案；不在，再把自己存进去。

所以这题的关键词不是"找两数"，而是"把找搭档这件事，变成一次哈希查询"。

## 💻 Java 题解

### 解法 1：哈希表一次遍历（最优）

边遍历边查哈希表。对每个元素 nums[i]，检查 target - nums[i] 是否已在表中。如果在，直接返回；不在则将当前元素存入表中。这样不会用到自己（因为存入发生在检查之后）。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i];
            if (map.containsKey(complement)) {
                return new int[]{map.get(complement), i};
            }
            map.put(nums[i], i);
        }
        return new int[0]; // 题目保证有解，不会执行到这
    }
}
```

**关键点：**
- 一次遍历完成
- 先查再存避免用到自己
- HashMap 查找 O(1)

**执行步骤：**
1. 初始化空 HashMap
1. 遍历每个元素 nums[i]
1. 计算 complement = target - nums[i]
1. 若 complement 在 map 中，返回结果
1. 否则将 nums[i]:i 存入 map


### 解法 2：暴力双重循环

枚举所有两两组合，检查和是否等于 target。简单直观但效率低。

- ⏱ 时间复杂度：`O(n²)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int[] twoSum(int[] nums, int target) {
        for (int i = 0; i < nums.length; i++) {
            for (int j = i + 1; j < nums.length; j++) {
                if (nums[i] + nums[j] == target) {
                    return new int[]{i, j};
                }
            }
        }
        return new int[0];
    }
}
```

**关键点：**
- 最简单的思路
- 无需额外空间
- 大数据量超时

**执行步骤：**
1. 外层循环 i 从 0 到 n-1
1. 内层循环 j 从 i+1 到 n-1
1. 检查 nums[i]+nums[j]==target


## ⚠️ 易错点

- 不要先把当前数放进哈希表，再查补数。否则在 `target = 2 × num` 的情况下，可能会把自己和自己配对。
- 哈希表里存的是"数值 → 下标"，不是"下标 → 数值"。
- 题目返回的是下标，不是两个数本身。

- 不要先存完再查——会导致可能用到自己

- 注意返回的是索引不是值

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个数只遍历一次，哈希查询平均是 `O(1)`。
- 空间复杂度：`O(n)`。最坏情况下，需要把所有数都记进哈希表。

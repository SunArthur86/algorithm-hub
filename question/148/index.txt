> 难度：**Medium** | 分类：链表 | 标签：链表, 双指针, 分治, 排序, 归并排序

> 🔗 力扣链接：[https://leetcode.cn/problems/sort-list/](https://leetcode.cn/problems/sort-list/)

## 📝 题目描述

给定链表的头节点 `head`，请将其按升序排列并返回排序后的链表。

要求时间复杂度 `O(n log n)`，空间复杂度 `O(1)`（即不能用递归，因为递归栈是 `O(log n)`）。重点不在"排序"，而在"怎么在链表上实现归并排序"。

- 题号：148
- 题目名：排序链表
- 难度：中等
- 题型：链表
- 为什么选这题：它要求 `O(n log n)` 时间 + `O(1)` 空间排序链表，是"归并排序在链表上的应用"的代表作。

## 🧠 思考路径

插入排序是 `O(n²)`，不符合要求。快速排序在链表上实现较复杂（不方便随机取 pivot）。

归并排序天然适合链表：
1. **分割**：用快慢指针找到链表中点，分成两半。
2. **递归排序**：对两半分别排序。
3. **合并**：把两个有序链表合并成一个。

但题目要求 `O(1)` 空间，递归的调用栈是 `O(log n)`。可以用**自底向上的迭代归并**：先 1 个 1 个合并，再 2 个 2 个合并，再 4 个 4 个合并……直到合并完整个链表。

这样不需要递归，空间是 `O(1)`。

所以这题的关键词不是"排序"，而是"链表上的归并排序：找中点 + 合并两个有序链表"。

## 💻 Java 题解

### 解法 1：归并排序（递归）

快慢指针找中点切分，递归排序左右两半，然后合并两个有序链表。

- ⏱ 时间复杂度：`O(n log n)`
- 💾 空间复杂度：`O(log n)`

```java
class Solution {
    public ListNode sortList(ListNode head) {
        if (head == null || head.next == null) return head;
        // 找中点
        ListNode slow = head, fast = head.next;
        while (fast != null && fast.next != null) {
            slow = slow.next; fast = fast.next.next;
        }
        ListNode mid = slow.next;
        slow.next = null;
        // 递归排序
        ListNode left = sortList(head);
        ListNode right = sortList(mid);
        // 合并
        return merge(left, right);
    }
    private ListNode merge(ListNode a, ListNode b) {
        ListNode dummy = new ListNode(0), tail = dummy;
        while (a != null && b != null) {
            if (a.val <= b.val) { tail.next = a; a = a.next; }
            else { tail.next = b; b = b.next; }
            tail = tail.next;
        }
        tail.next = a != null ? a : b;
        return dummy.next;
    }
}
```

**关键点：**
- 快慢指针找中点
- 递归分治
- 合并两个有序链表

**执行步骤：**
1. 快慢指针找中点
1. 断开成两段
1. 递归排序左右
1. 合并


## ⚠️ 易错点

- 找中点时，`fast` 从 `head.next` 开始（不是 `head`），确保奇数长度时 slow 停在左半的末尾。
- 断开链表时 `slow.next = None` 不能忘。
- 合并时用哨兵节点 `dummy` 简化操作。
- 自底向上归并中，`split` 函数需要在断开处截断（`head.next = null`）。

- fast 初始是 head.next 不是 head（避免栈溢出）

- slow.next=null 断开链表

## 📊 复杂度分析

- 时间复杂度：`O(n log n)`。归并排序的标准复杂度。
- 空间复杂度：递归版本 `O(log n)`（递归栈），迭代版本 `O(1)`。

## 🏢 实际业务应用

- 大规模日志排序：日志以链表形式存储（每条日志指向下一条），需要按时间戳排序时，归并排序是链表上最优的选择。
- 数据库的外排序：当数据量大到内存放不下时，数据以链表形式分块存储，块间用归并排序合并。
- 大文件合并：多个有序的小文件合并成一个大文件，核心就是链表归并。

e:T14f4,# 70. 爬楼梯

> 难度：**Easy** | 分类：动态规划 | 标签：数学, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/climbing-stairs/](https://leetcode.cn/problems/climbing-stairs/)

## 📝 题目描述

你在爬楼梯，一共 `n` 级台阶。每次可以爬 1 级或 2 级。问：有多少种不同的方法爬到顶？

比如 `n = 3`，有 3 种：`1+1+1`、`1+2`、`2+1`。

- 题号：70
- 题目名：爬楼梯
- 难度：简单
- 题型：动态规划
- 为什么选这题：它是最经典的 DP 入门题，递推关系极其简洁（`dp[i] = dp[i-1] + dp[i-2]`），能最快建立"状态定义 + 递推"的直觉。

## 🧠 思考路径

站在第 `i` 级台阶上，你只可能从第 `i-1` 级跨 1 步上来，或从第 `i-2` 级跨 2 步上来。所以到达第 `i` 级的方法数 = 到达第 `i-1` 级的方法数 + 到达第 `i-2` 级的方法数。

这不就是斐波那契数列吗？没错。`dp[1] = 1`，`dp[2] = 2`，`dp[i] = dp[i-1] + dp[i-2]`。

空间优化：计算 `dp[i]` 只需要前两个值，用两个变量滚动即可，空间 `O(1)`。

## 💻 Java 题解

### 解法 1：动态规划（空间优化）

f(n)=f(n-1)+f(n-2)。用两个变量滚动。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int climbStairs(int n) {
        if (n <= 2) return n;
        int prev2 = 1, prev1 = 2;
        for (int i = 3; i <= n; i++) {
            int curr = prev1 + prev2;
            prev2 = prev1; prev1 = curr;
        }
        return prev1;
    }
}
```

**关键点：**
- 斐波那契数列
- 两个变量滚动
- n<=2直接返回

**执行步骤：**
1. prev2=1, prev1=2
1. 滚动更新
1. 返回prev1


## ⚠️ 易错点

- `n = 1` 时只有 1 种方法，`n = 2` 时有 2 种方法，不要统一用 `dp[0] = 1, dp[1] = 1` 的方式搞混下标。
- 暴力递归会超时。`n = 45` 时递归深度 45 层，但重复子问题呈指数增长，必须用 DP 或记忆化。
- 滚动变量更新时要注意顺序，不能先覆盖 `prev` 再算 `curr`。

- n=1和n=2的base case

## 📊 复杂度分析

- 时间复杂度：`O(n)`。遍历一次。
- 空间复杂度：`O(1)`。只用了两个变量。

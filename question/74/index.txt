e:T17e3,# 74. 搜索二维矩阵

> 难度：**Medium** | 分类：二分查找 | 标签：数组, 二分查找, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/search-a-2d-matrix/](https://leetcode.cn/problems/search-a-2d-matrix/)

## 📝 题目描述

给你一个 `m × n` 的矩阵，每行从左到右递增，每行第一个数大于上一行最后一个数。再给一个目标值 `target`，判断它是否在矩阵中。

由于这个矩阵满足严格的全局递增（不只是局部递增），可以把它当成一个排序好的一维数组来做二分查找。

- 题号：74
- 题目名：搜索二维矩阵
- 难度：中等
- 题型：二分查找
- 为什么选这题：它展示了如何把二维结构"拍扁"成一维来做二分，是把一维算法推广到二维的经典案例。

## 🧠 思考路径

最暴力的方法是遍历所有元素，`O(m × n)`。但矩阵是有序的，有序就应该想到二分。

关键观察：如果把这个矩阵按行拼接成一个一维数组，它是严格递增的。那问题就变成了"在一个有序一维数组里查找目标值"。唯一需要处理的是坐标转换：一维下标 `mid` 对应矩阵中的 `(mid // n, mid % n)`。

这样时间复杂度直接降到 `O(log(m × n))`。

## 💻 Java 题解

### 解法 1：二分查找（展平）

将二维索引映射到一维：mid→(mid/cols, mid%cols)。标准二分。

- ⏱ 时间复杂度：`O(log(m×n))`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length, n = matrix[0].length;
        int left = 0, right = m * n - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            int val = matrix[mid / n][mid % n];
            if (val == target) return true;
            else if (val < target) left = mid + 1;
            else right = mid - 1;
        }
        return false;
    }
}
```

**关键点：**
- mid/n 得行，mid%n 得列
- 本质就是一个有序一维数组

**执行步骤：**
1. 二分0到m*n-1
1. mid映射到二维
1. 比较并移动


## ⚠️ 易错点

- 坐标映射是 `mid // n` 和 `mid % n`，不是 `mid // m`。除的是列数 `n`。
- `right` 的初始值是 `m * n - 1`，不是 `m * n`。
- 注意题目的条件是每行第一个数大于上一行最后一个数（全局有序），不是普通的行内递增。如果是后者，需要用另一种策略。

- 矩阵列数n不是m

- mid/n 和 mid%n 不要搞反

## 📊 复杂度分析

- 时间复杂度：`O(log(m × n))`，标准二分查找。
- 空间复杂度：`O(1)`，只用了几个变量。

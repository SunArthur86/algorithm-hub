> 难度：**Medium** | 分类：二叉树 | 标签：树, 深度优先搜索, 二叉树

> 🔗 力扣链接：[https://leetcode.cn/problems/path-sum-iii/](https://leetcode.cn/problems/path-sum-iii/)

## 📝 题目描述

给你一棵二叉树和一个整数 `targetSum`，求树中有多少条从任意节点到任意节点的路径，使得路径上所有节点值之和等于 `targetSum`。路径必须从父节点到子节点（向下走），不能回头。

重点不在"根到叶"，而在"任意起点到任意终点"——这意味着要枚举所有可能的路径。

- 题号：437
- 题目名：路径总和 III
- 难度：中等
- 题型：二叉树
- 为什么选这题：它把"前缀和"思想从数组搬到了树上，是理解"树上的路径问题"的关键一步。

## 🧠 思考路径

暴力做法：枚举每个节点作为路径起点，向下 DFS 枚举所有路径，统计和等于 `targetSum` 的数量。时间复杂度 `O(n²)`，对于较大的树会很慢。

优化思路：在数组中，我们用前缀和 + 哈希表来 `O(n)` 解决"子数组和等于 k"的问题。树上能不能做同样的事？

可以。在 DFS 遍历树时，维护从根到当前节点的"路径前缀和"。对于当前节点，如果存在某个祖先节点的前缀和恰好等于 `当前前缀和 - targetSum`，那么从那个祖先到当前节点的路径和就是 `targetSum`。用哈希表记录每种前缀和出现的次数，就能 `O(1)` 查询。

回溯时要把当前前缀和从哈希表中移除（计数减一），保证不同分支之间不互相干扰。

所以这题的本质是：树上 DFS + 前缀和 + 哈希表。

## 💻 Java 题解

### 解法 1：前缀和 + DFS

DFS过程中用 HashMap 记录前缀和出现次数。对每个节点，查找当前前缀和-target 出现次数。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int pathSum(TreeNode root, int targetSum) {
        Map<Long, Integer> prefixCount = new HashMap<>();
        prefixCount.put(0L, 1);
        return dfs(root, 0, targetSum, prefixCount);
    }
    private int dfs(TreeNode node, long currSum, int target, Map<Long, Integer> map) {
        if (node == null) return 0;
        currSum += node.val;
        int count = map.getOrDefault(currSum - target, 0);
        map.merge(currSum, 1, Integer::sum);
        count += dfs(node.left, currSum, target, map);
        count += dfs(node.right, currSum, target, map);
        map.merge(currSum, -1, Integer::sum); // 回溯
        return count;
    }
}
```

**关键点：**
- 前缀和差等于target
- HashMap记录前缀和频率
- 回溯时移除当前前缀和

**执行步骤：**
1. DFS遍历
1. 累加前缀和
1. 查找 currSum-target 的出现次数
1. 回溯移除


## ⚠️ 易错点

- 必须初始化 `prefix[0] = 1`，否则从根节点出发的路径无法被统计。
- 回溯时一定要将前缀和从哈希表中减回，否则左子树的前缀和会影响右子树的计算。
- 节点值可能为负数，前缀和不单调递增，不能用双指针替代哈希表。
- Java 中前缀和可能溢出 int 范围，要用 `long` 类型。

- 必须回溯——否则会统计非路径上的前缀和

- 用 long 防溢出

## 📊 复杂度分析

- 时间复杂度：`O(n)`。每个节点访问一次，哈希表查询 `O(1)`。
- 空间复杂度：`O(n)`。哈希表和递归栈最坏都是 `O(n)`。

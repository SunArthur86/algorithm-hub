e:T175b,# 64. 最小路径和

> 难度：**Medium** | 分类：多维动态规划 | 标签：数组, 动态规划, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/minimum-path-sum/](https://leetcode.cn/problems/minimum-path-sum/)

## 📝 题目描述

给你一个 `m × n` 的网格，每个格子里有一个非负整数，代表经过这个格子的代价。你从左上角出发，到右下角结束，每次只能向右或向下走一步。问：路径上所有数字加起来最小是多少？

和 62 题的区别是：62 题求"有多少条路"，这题求"所有路里哪条最便宜"。

- 题号：64
- 题目名：最小路径和
- 难度：中等
- 题型：动态规划
- 为什么选这题：和 62、63 题组成"路径三连"，从计数到条件计数到最小代价，展示 DP 递推式的灵活变化。

## 🧠 思考路径

还是那个核心观察：每个格子只能从上方或左方到达。那么到达当前格子的最小代价，就是"从上面来的最小代价"和"从左边来的最小代价"中较小的那个，再加上当前格子本身的代价。

递推式：`dp[i][j] = min(dp[i-1][j], dp[i][j-1]) + grid[i][j]`。

边界条件：第一行的格子只能从左边来，所以 `dp[0][j] = dp[0][j-1] + grid[0][j]`；第一列的格子只能从上面来，所以 `dp[i][0] = dp[i-1][0] + grid[i][0]`。起点格子 `dp[0][0] = grid[0][0]`。

空间优化：和前两题一样，用一维数组滚动更新。`dp[j] = min(dp[j], dp[j-1]) + grid[i][j]`，其中 `dp[j]` 是从上面来的，`dp[j-1]` 是从左边来的。

## 💻 Java 题解

### 解法 1：动态规划（原地修改）

直接在grid上修改。grid[i][j]+=min(grid[i-1][j], grid[i][j-1])。

- ⏱ 时间复杂度：`O(m×n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int minPathSum(int[][] grid) {
        int m = grid.length, n = grid[0].length;
        for (int i = 0; i < m; i++)
            for (int j = 0; j < n; j++) {
                if (i == 0 && j > 0) grid[i][j] += grid[i][j-1];
                else if (j == 0 && i > 0) grid[i][j] += grid[i-1][j];
                else if (i > 0 && j > 0) grid[i][j] += Math.min(grid[i-1][j], grid[i][j-1]);
            }
        return grid[m-1][n-1];
    }
}
```

**关键点：**
- 原地修改O(1)空间
- 第一行只加左边
- 第一列只加上边

**执行步骤：**
1. 第一行累加
1. 第一列累加
1. 内部取min累加


## ⚠️ 易错点

- 不要忘记处理起点 `dp[0][0]`，它直接等于 `grid[0][0]`，不是 0。
- 第一列的更新：`dp[j] = dp[j] + grid[i][j]`，这里右边的 `dp[j]` 是上一行的值（从上面来），不是本行新算的。
- 如果想直接在原数组上改，可以省掉 `dp` 数组，但会破坏原始数据，面试时先问清是否允许修改。

- 第一行和第一列的base case

## 📊 复杂度分析

- 时间复杂度：`O(m × n)`。遍历整个网格一次。
- 空间复杂度：`O(n)`。一维数组滚动更新。

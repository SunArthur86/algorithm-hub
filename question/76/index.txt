e:T1ea9,# 76. 最小覆盖子串

> 难度：**Hard** | 分类：子串 | 标签：哈希表, 字符串, 滑动窗口

> 🔗 力扣链接：[https://leetcode.cn/problems/minimum-window-substring/](https://leetcode.cn/problems/minimum-window-substring/)

## 📝 题目描述

给你两个字符串 `s` 和 `t`。在 `s` 中找一个最短的连续子串，使得这个子串包含了 `t` 中所有字符（包括重复字符）。返回这个子串，如果不存在就返回空串。

比如 `s = "ADOBECODEBANC"`，`t = "ABC"`，答案是 `"BANC"`，因为它包含了 `A`、`B`、`C` 各至少一个，而且是所有满足条件中最短的。

- 题号：76
- 题目名：最小覆盖子串
- 难度：困难
- 题型：滑动窗口
- 为什么选这题：它是滑动窗口的标杆题，涵盖了窗口扩张、收缩、合法性判断三大核心操作。

## 🧠 思考路径

暴力方法是枚举所有子串逐一检查，`O(n²)`。但可以用滑动窗口优化。

关键思路：用两个指针 `left` 和 `right` 维护一个窗口。先不断右移 `right` 扩大窗口，直到窗口包含了 `t` 的所有字符；然后尝试右移 `left` 缩小窗口，直到窗口不再满足条件。每次窗口满足条件时记录最小长度。

为了高效判断"窗口是否包含了 `t` 的所有字符"，用两个哈希表（或数组）分别统计 `t` 需要的字符频率和当前窗口中的字符频率。用一个计数器 `formed` 追踪已经满足要求的字符种类数。

## 💻 Java 题解

### 解法 1：滑动窗口（模板）

用两个指针维护窗口。右指针扩展直到窗口包含 t 的所有字符，然后左指针收缩到最短。用 needMap 和 have 计数器跟踪。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(m)`

```java
class Solution {
    public String minWindow(String s, String t) {
        Map<Character, Integer> need = new HashMap<>();
        Map<Character, Integer> window = new HashMap<>();
        for (char c : t.toCharArray()) need.merge(c, 1, Integer::sum);
        int left = 0, right = 0, valid = 0;
        int start = 0, minLen = Integer.MAX_VALUE;
        while (right < s.length()) {
            char c = s.charAt(right++);
            if (need.containsKey(c)) {
                window.merge(c, 1, Integer::sum);
                if (window.get(c).equals(need.get(c))) valid++;
            }
            // 所有字符都满足时收缩
            while (valid == need.size()) {
                if (right - left < minLen) {
                    start = left;
                    minLen = right - left;
                }
                char d = s.charAt(left++);
                if (need.containsKey(d)) {
                    if (window.get(d).equals(need.get(d))) valid--;
                    window.merge(d, -1, Integer::sum);
                }
            }
        }
        return minLen == Integer.MAX_VALUE ? "" : s.substring(start, start + minLen);
    }
}
```

**关键点：**
- need 记录需要的字符及数量
- valid 记录已满足的字符种类数
- 先扩展再收缩的滑动窗口模板

**执行步骤：**
1. 统计 t 的字符频率
1. 右指针扩展加入字符
1. valid==need.size() 时开始收缩
1. 收缩时更新最短长度


## ⚠️ 易错点

- `formed` 的更新时机：是 `window[ch] == need[ch]` 时才加一，不是 `>=`。否则会重复计数。
- 缩小窗口时 `formed--` 的条件：是 `window[ch] < need[ch]`，即某个字符的窗口数量跌破需求量。
- `t` 中可能有重复字符，所以需求量可能大于 `1`，不能用"字符是否存在"来判断。
- 空串或 `s` 比 `t` 短的情况要提前返回。

- valid 是满足的种类数不是总数

- 收缩后要检查 valid 是否仍满足

## 📊 复杂度分析

- 时间复杂度：`O(|s| + |t|)`。左右指针各最多遍历 `s` 一次。
- 空间复杂度：`O(1)`。字符集有限（ASCII 或 Unicode），哈希表大小固定。

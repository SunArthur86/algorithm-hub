e:T1f7a,# 138. 随机链表的复制

> 难度：**Medium** | 分类：链表 | 标签：哈希表, 链表

> 🔗 力扣链接：[https://leetcode.cn/problems/copy-list-with-random-pointer/](https://leetcode.cn/problems/copy-list-with-random-pointer/)

## 📝 题目描述

给你一个链表，每个节点除了 `next` 指针外，还有一个 `random` 指针，可以指向链表中的任意节点或 `null`。请返回这个链表的深拷贝。

新链表中的节点必须是全新的对象，结构和原链表完全一致（包括 `next` 和 `random` 的指向关系）。重点不在"顺着 next 复制"，而在"random 指针怎么正确指向新节点"。

- 题号：138
- 题目名：随机链表的复制
- 难度：中等
- 题型：链表
- 为什么选这题：它从普通的链表复制升级为带随机指针的复制，是链表操作和哈希映射的综合训练。

## 🧠 思考路径

如果只有 `next` 指针，顺着走一遍就能复制完。但 `random` 指针可能指向前面的节点——如果那个节点还没创建副本，`random` 就无法设置。

最直觉的做法是用哈希表：第一遍遍历创建所有新节点并存入映射 `原节点 → 新节点`，第二遍遍历设置 `next` 和 `random`，通过查表找到对应的新节点。时间和空间都是 `O(n)`。

还有一道巧妙的 `O(1)` 空间解法：把新节点"穿插"在原节点后面。即对每个原节点 `A`，创建新节点 `A'`，让 `A.next = A'`，`A'.next = B`。这样设置 `random` 时，`A'.random = A.random.next`（因为原节点的下一个就是它的副本）。最后再拆开两条链表。

所以这题的关键词不是"复制链表"，而是"用映射保证 random 指向正确的副本"。

## 💻 Java 题解

### 解法 1：插入复制节点 + 拆分

第一步：在每个节点后插入复制节点。第二步：设置 random 指针（copy.random = original.random.next）。第三步：拆分两个链表。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public Node copyRandomList(Node head) {
        if (head == null) return null;
        // 1. 插入复制节点
        Node curr = head;
        while (curr != null) {
            Node copy = new Node(curr.val);
            copy.next = curr.next;
            curr.next = copy;
            curr = copy.next;
        }
        // 2. 设置 random
        curr = head;
        while (curr != null) {
            if (curr.random != null) curr.next.random = curr.random.next;
            curr = curr.next.next;
        }
        // 3. 拆分
        curr = head;
        Node newHead = head.next;
        while (curr != null) {
            Node copy = curr.next;
            curr.next = copy.next;
            if (copy.next != null) copy.next = copy.next.next;
            curr = curr.next;
        }
        return newHead;
    }
}
```

**关键点：**
- 三步走：插入→设置random→拆分
- O(1) 空间不使用 HashMap
- copy.random = curr.random.next

**执行步骤：**
1. 每个节点后插入复制节点
1. 设置复制节点的 random
1. 拆分两个链表


## ⚠️ 易错点

- 哈希表法中，`mapping.get(cur.next)` 可能返回 `null`（当 `cur.next` 为 null 时），用 `get` 而不是 `getOrDefault` 就行，null 自动处理。
- 穿插法中，设置 `random` 时要检查 `cur.random != null`，因为 `null` 没有 `next`。
- 拆链表时，`cur.next = cur.next.next` 恢复原链表，`copyCur.next = cur.next` 提取新链表，两步顺序不能反。
- 两种方法都能通过，哈希表法更直观，穿插法更省空间。

- random 可能为 null 要检查

- 拆分时注意 copy.next 的更新

## 📊 复杂度分析

- 时间复杂度：`O(n)`。无论哪种方法都遍历 2-3 次。
- 空间复杂度：哈希表法 `O(n)`，穿插法 `O(1)`。

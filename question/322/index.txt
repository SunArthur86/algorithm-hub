> 难度：**Medium** | 分类：动态规划 | 标签：广度优先搜索, 数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/coin-change/](https://leetcode.cn/problems/coin-change/)

## 📝 题目描述

给你不同面额的硬币 `coins` 和一个总金额 `amount`。每种硬币有无限多个。你要凑出这个总金额，且使用的硬币数量最少。如果凑不出来，返回 -1。

关键不是"能不能凑出"（那只是一部分），而是"凑出来的方案中，硬币数最少的那个"。

- 题号：322
- 题目名：零钱兑换
- 难度：Medium
- 题型：动态规划
- 为什么选这题：它是完全背包的入门题，用最直白的场景帮你建立"dp[i] = min(dp[i-coin] + 1)"的直觉。

## 🧠 思考路径

暴力做法：递归尝试每种硬币，对每个金额枚举用 0 个、1 个、2 个……直到金额不够。这是指数级复杂度。

动态规划：定义 `dp[i]` 为凑出金额 `i` 需要的最少硬币数。转移方程：`dp[i] = min(dp[i - coin] + 1)`，对每种面额 `coin` 都试一下。如果 `i - coin >= 0` 且 `dp[i - coin]` 可达，就取最小值。

初始化 `dp[0] = 0`，其余 `dp[i] = ∞`。最终如果 `dp[amount]` 仍是 `∞`，返回 -1。

## 💻 Java 题解

### 解法 1：动态规划（完全背包）

dp[i]表示凑成金额i的最少硬币数。对每个金额枚举所有面值。

- ⏱ 时间复杂度：`O(n×amount)`
- 💾 空间复杂度：`O(amount)`

```java
class Solution {
    public int coinChange(int[] coins, int amount) {
        int[] dp = new int[amount + 1];
        Arrays.fill(dp, amount + 1);
        dp[0] = 0;
        for (int i = 1; i <= amount; i++)
            for (int coin : coins)
                if (coin <= i) dp[i] = Math.min(dp[i], dp[i - coin] + 1);
        return dp[amount] > amount ? -1 : dp[amount];
    }
}
```

**关键点：**
- 初始化amount+1表示不可达
- 完全背包：正向遍历
- 最后检查是否可达

**执行步骤：**
1. 初始化dp
1. 对每个金额枚举面值
1. 返回dp[amount]或-1


## ⚠️ 易错点

- `dp[0] = 0` 是边界条件，凑出金额 0 不需要任何硬币。
- Java 中用 `amount + 1` 代替无穷大更方便，因为最多只需要 `amount` 个 1 元硬币。
- 硬币可以重复使用（完全背包），所以内层循环是金额而不是硬币数量。

- 用amount+1初始化比MAX_VALUE更安全

- coin<=i 才能使用

## 📊 复杂度分析

- 时间复杂度：`O(amount × n)`，`n` 是硬币种类数。
- 空间复杂度：`O(amount)`，`dp` 数组。

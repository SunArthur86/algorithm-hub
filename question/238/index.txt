e:T1bb6,# 238. 除自身以外数组的乘积

> 难度：**Medium** | 分类：普通数组 | 标签：数组, 前缀和

> 🔗 力扣链接：[https://leetcode.cn/problems/product-of-array-except-self/](https://leetcode.cn/problems/product-of-array-except-self/)

## 📝 题目描述

给你一个整数数组 `nums`，返回一个数组 `answer`，使得 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积。要求时间复杂度 `O(n)`，且不使用除法。

重点不在"怎么算乘积"，而在"怎么把每个位置左边所有数的乘积和右边所有数的乘积组合起来"。

- 题号：238
- 题目名：除自身以外数组的乘积
- 难度：中等
- 题型：数组
- 为什么选这题：它是前缀思想的经典变体——从"前缀和"升级到"前缀积 × 后缀积"，不使用除法的约束让这道题的设计非常优雅。

## 🧠 思考路径

如果允许用除法，最简单的做法是先算所有数的总乘积，然后每个位置除以自己。但题目不允许，而且如果有零存在，除法也会出错。

换个角度：对于位置 `i`，`answer[i]` 等于"它左边所有数的乘积"乘以"它右边所有数的乘积"。如果我能预先算出每个位置左边和右边的累积乘积，直接相乘就是答案。

具体做法是两遍遍历：
1. 第一遍从左到右，计算每个位置左侧所有数的乘积（前缀积），存入 `answer` 数组。
2. 第二遍从右到左，用一个变量 `right` 维护右侧的累积乘积，直接乘到 `answer[i]` 上。

这样不需要额外的数组（除了返回的 `answer`），空间复杂度满足 `O(1)` 的要求。

这题的关键词不是"算乘积"，而是"前缀积从左扫一遍，后缀积从右扫一遍，两两相乘"。

## 💻 Java 题解

### 解法 1：前缀积 × 后缀积

result[i] = 左边所有数的积 × 右边所有数的积。两次遍历：先正向求前缀积，再反向乘后缀积。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int[] productExceptSelf(int[] nums) {
        int n = nums.length;
        int[] result = new int[n];
        result[0] = 1;
        // 前缀积
        for (int i = 1; i < n; i++)
            result[i] = result[i-1] * nums[i-1];
        // 后缀积
        int right = 1;
        for (int i = n - 1; i >= 0; i--) {
            result[i] *= right;
            right *= nums[i];
        }
        return result;
    }
}
```

**关键点：**
- 先填前缀积
- 再用right变量反向乘后缀积
- result数组不算额外空间→O(1)

**执行步骤：**
1. result[i] = nums[0..i-1]的积
1. 反向遍历乘以nums[i+1..n-1]的积


## ⚠️ 易错点

- 第一遍遍历时，`answer[i] = left` 必须在 `left *= nums[i]` 之前，否则会把自身也算进去。
- `answer` 数组初始值要全设为 `1`，不是 `0`。
- 第二遍遍历是从右往左（`n-1` 到 `0`），方向别写反。
- 题目要求不算额外空间，所以用 `answer` 数组同时充当前缀积的存储。

- right 变量初始化为1

- O(1)空间需要用result数组本身

## 📊 复杂度分析

- 时间复杂度：`O(n)`。两遍遍历，各 `O(n)`。
- 空间复杂度：`O(1)`（不算返回的 `answer` 数组）。只用两个变量 `left` 和 `right`。

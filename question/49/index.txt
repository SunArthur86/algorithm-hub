e:T22d7,# 49. 字母异位词分组

> 难度：**Medium** | 分类：哈希 | 标签：数组, 哈希表, 字符串

> 🔗 力扣链接：[https://leetcode.cn/problems/group-anagrams/](https://leetcode.cn/problems/group-anagrams/)

## 📝 题目描述

给你一堆字符串，要求把"字母异位词"分到同一组。字母异位词就是由相同字母重新排列组成的单词，比如 `"eat"`、`"tea"`、`"ate"` 互为异位词，它们含有相同的字母 `a`、`e`、`t`，只是顺序不同。

你需要返回一个列表，每个子列表是一组互为异位词的字符串。

- 题号：49
- 题目名：字母异位词分组
- 难度：中等
- 题型：哈希
- 为什么选这题：它是哈希表"特征归类"的典型应用，教你如何设计合适的 key 把同一类元素归到一组。

## 🧠 思考路径

最朴素的想法是：对每对字符串都检查一下是不是异位词，是就分到同一组。但两两比较太慢，`O(n² × m)`。

换个角度：两个字符串是异位词，等价于它们包含的字母种类和数量完全相同。那我们只需要找到一个"统一的标准形式"，让所有异位词都映射到同一个 key 上。

最自然的 key 有两种：
1. **排序后的字符串**：`"eat"` 排序后是 `"aet"`，`"tea"` 排序后也是 `"aet"`。排序是天然的"统一标准"。
2. **字母计数编码**：统计每个字母出现的次数，拼成 `"a1e1t1"` 这样的字符串作为 key。对于包含 Unicode 的情况更通用。

用哈希表存储 `key → 同组字符串列表`，遍历一次即可完成分组。

## 💻 Java 题解

### 解法 1：排序字符串作为哈希键

对每个字符串排序，排序结果相同的即为异位词，分到同一组。

- ⏱ 时间复杂度：`O(n·k·log k)`
- 💾 空间复杂度：`O(n·k)`

```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        for (String s : strs) {
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(map.values());
    }
}
```

**关键点：**
- 排序后字符串作为 key
- computeIfAbsent 简化代码
- 字母异位词排序后一定相同

**执行步骤：**
1. 遍历每个字符串
1. 转为 char[] 排序得到 key
1. 用 HashMap 分组
1. 返回所有分组列表


### 解法 2：字符计数数组作为键

统计每个字母出现次数，用 26 长度数组转字符串作为 key，避免排序开销。

- ⏱ 时间复杂度：`O(n·k)`
- 💾 空间复杂度：`O(n·k)`

```java
class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        for (String s : strs) {
            int[] count = new int[26];
            for (char c : s.toCharArray()) count[c - 'a']++;
            String key = Arrays.toString(count);
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        return new ArrayList<>(map.values());
    }
}
```

**关键点：**
- 26字母计数代替排序
- 时间复杂度更优
- key 用 Arrays.toString 转换

**执行步骤：**
1. 遍历每个字符串
1. 统计26个字母出现频率
1. 频率数组转字符串作为 key
1. HashMap 分组


## ⚠️ 易错点

- Python 中 `sorted(s)` 返回的是字符列表，需要 `''.join()` 拼成字符串才能当 key。
- Java 中用计数法时，`StringBuilder` 拼出的 key 格式要唯一。不能只拼计数不拼字母，否则 `"abb"` 和 `"aab"` 的计数数组不同但只拼数字可能冲突。
- `groups.values()` 返回的不是 `list`，需要 `list()` 或 `new ArrayList<>()` 包装一下。
- 空字符串也是合法输入，排序后还是空字符串，会被分到同一组。

- 排序用 Arrays.sort(chars) 不是 String 排序

- 注意空字符串的处理

## 📊 复杂度分析

- 时间复杂度：`O(n × k log k)`，其中 `n` 是字符串个数，`k` 是字符串的最大长度。每个字符串排序需要 `O(k log k)`。如果用计数法，时间降到 `O(n × k)`。
- 空间复杂度：`O(n × k)`。哈希表需要存储所有字符串。

- **文档查重**：两篇文章中的段落，如果打乱了句子顺序（洗稿），传统哈希无法识别。但如果把每个词的字母排序后作为指纹，异位词分组可以帮助发现这类"字面相同、顺序不同"的内容抄袭。
- **推荐系统特征提取**：在自然语言处理中，两个词如果包含相同的字母组合，可能在词形变化上有联系（如 `listen` 和 `silent`）。分组后可以作为特征输入推荐模型。
- **账号风控**：批量注册的垃圾账号可能用字母相同但顺序不同的用户名（如 `abc123`、`123abc`、`cba321`），用异位词分组可以快速发现这类批量注册行为。

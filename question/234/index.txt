> 难度：**Easy** | 分类：链表 | 标签：栈, 递归, 链表, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/palindrome-linked-list/](https://leetcode.cn/problems/palindrome-linked-list/)

## 📝 题目描述

给你一个单链表的头节点，判断它是否是回文链表（正着读和反着读一样）。

重点不在"回文的定义"，而在"怎么在 `O(1)` 空间内判断，而不是把链表复制到数组里"。

- 题号：234
- 题目名：回文链表
- 难度：简单
- 题型：链表
- 为什么选这题：它巧妙地结合了"找中点"、"反转链表"和"逐一比较"三个技巧，是链表操作的综合演练，也是面试中出现频率极高的题。

## 🧠 思考路径

最直接的做法是遍历链表把值存到数组里，然后在数组上用双指针判断回文。时间 `O(n)`，空间 `O(n)`，可以过但不够优雅。

要做到 `O(1)` 空间，关键想法是：如果能把链表的后半段反转，然后前半段和反转后的后半段逐一比较，就能判断是否回文。

具体步骤：
1. 用快慢指针找到链表的中点（快指针走两步，慢指针走一步）。
2. 反转后半段链表。
3. 前半段和反转后的后半段逐一比较。
4. （可选）把后半段再反转回来恢复原链表。

这个方案把链表的三个经典操作串联在一起：快慢指针找中点、反转链表、双指针比较。每一步都是 `O(n)`，整体 `O(n)` 时间 `O(1)` 空间。

这题的关键词不是"判断回文"，而是"找中点、反转后半段、前后比较——三步搞定"。

## 💻 Java 题解

### 解法 1：快慢指针 + 反转后半部分

快慢指针找中点，反转后半部分链表，然后双指针比较前后两半。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public boolean isPalindrome(ListNode head) {
        // 快慢指针找中点
        ListNode slow = head, fast = head;
        while (fast.next != null && fast.next.next != null) {
            slow = slow.next; fast = fast.next.next;
        }
        // 反转后半部分
        ListNode second = reverse(slow.next);
        // 比较
        ListNode p1 = head, p2 = second;
        while (p2 != null) {
            if (p1.val != p2.val) return false;
            p1 = p1.next; p2 = p2.next;
        }
        return true;
    }
    private ListNode reverse(ListNode head) {
        ListNode prev = null, curr = head;
        while (curr != null) {
            ListNode next = curr.next;
            curr.next = prev;
            prev = curr; curr = next;
        }
        return prev;
    }
}
```

**关键点：**
- 快慢指针找中点
- 反转后半部分与前半比较
- O(1) 空间

**执行步骤：**
1. 快慢指针找中点
1. 反转后半部分
1. 双指针逐个比较


## ⚠️ 易错点

- 反转后半段时，`prev` 初始为 `null`，不是 `slow`。
- 比较时只需要判断 `right != null`，因为如果链表长度为奇数，前半段会多一个中间节点，不影响判断。
- 快慢指针的循环条件是 `fast and fast.next`，缺一不可。

- 奇数长度时 slow 停在前半的最后一个

- fast.next 和 fast.next.next 都要判空

## 📊 复杂度分析

- 时间复杂度：`O(n)`。找中点 `O(n/2)`，反转 `O(n/2)`，比较 `O(n/2)`。
- 空间复杂度：`O(1)`。只用了几个指针变量。

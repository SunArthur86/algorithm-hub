e:T18fb,# 11. 盛最多水的容器

> 难度：**Medium** | 分类：双指针 | 标签：贪心, 数组, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/container-with-most-water/](https://leetcode.cn/problems/container-with-most-water/)

## 📝 题目描述

给你一个数组 `height`，每个元素表示竖线的长度。找出两条竖线，让它们和 x 轴围成的矩形面积最大。面积 = 两条线的距离 × 较短那条线的高度。

- 题号：11
- 题目名：盛最多水的容器
- 难度：中等
- 题型：双指针
- 为什么选这题：它是双指针"短板决定上限"思想的最佳入门题，训练"移动哪边"的决策能力。

## 🧠 思考路径

暴力做法是枚举所有两两组合，计算面积取最大值，时间 `O(n²)`。

观察面积公式：`面积 = min(h[i], h[j]) × (j - i)`。如果左右两个指针分别站在数组两端，每次移动较短的那个指针向内靠拢，面积有可能变大（因为距离在减小，但如果新的线更高，面积的损失可能被弥补）。而移动较长的那边没有意义——面积受短板限制，长板变长也没用。

这就是贪心策略：每次放弃短板，因为短板决定了当前面积的上限，留着它不可能得到更大的面积。

## 💻 Java 题解

### 解法 1：贪心双指针

左右指针从两端出发。每次移动较短的一边（因为移动较长边不可能获得更大水量），记录最大值。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int maxArea(int[] height) {
        int left = 0, right = height.length - 1;
        int maxWater = 0;
        while (left < right) {
            int h = Math.min(height[left], height[right]);
            int w = right - left;
            maxWater = Math.max(maxWater, h * w);
            if (height[left] < height[right]) left++;
            else right--;
        }
        return maxWater;
    }
}
```

**关键点：**
- 面积 = min(h[left],h[right]) × 宽度
- 移动较短的一边
- 一次遍历 O(n)

**执行步骤：**
1. left=0, right=n-1
1. 计算当前水量并更新最大值
1. 移动较短的一边
1. 直到 left >= right


## ⚠️ 易错点

- 移动较短的那边，不是移动任意一边。移动长板只会让距离变小而短板不变，面积不可能增大。
- 面积由短板决定，所以用 `min(height[left], height[right])`。
- 两边高度相等时，移动哪边都可以，不影响正确性。

- 不是移动较长边！移动较长边宽度减少但高度不会增加

- 宽度是 right-left 不是 right-left+1

## 📊 复杂度分析

- 时间复杂度：`O(n)`。左右指针各最多遍历一半。
- 空间复杂度：`O(1)`。只用常数个变量。

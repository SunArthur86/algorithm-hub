> 难度：**Easy** | 分类：链表 | 标签：哈希表, 链表, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/intersection-of-two-linked-lists/](https://leetcode.cn/problems/intersection-of-two-linked-lists/)

## 📝 题目描述

给你两个单链表的头节点 `headA` 和 `headB`，它们可能在某个节点开始相交（共享后面的所有节点）。找出并返回相交的起始节点。如果不相交，返回 `null`。

重点不在"遍历链表"，而在"两个不同长度的链表怎么同步走到交点"。

- 题号：160
- 题目名：相交链表
- 难度：简单
- 题型：链表
- 为什么选这题：它有一个极其优雅的双指针解法——走完自己的路再走对方的路，必然在交点相遇，是链表题里的思维艺术品。

## 🧠 思考路径

最直接的想法是让两个指针同步走，但两条链表长度不同，同步走就会错过交点。如果能先知道长度差，让长的那个先走几步就行。

但有个更巧妙的办法：让两个指针分别从 A 和 B 出发，走完自己的链表后，跳到另一条链表的头部继续走。这样两个指针各走了 `lenA + lenB` 步，必然同时到达交点（如果存在的话）。如果不存在交点，两个指针会同时到达 `null`。

为什么？因为走的总步数一样。如果存在交点，指针 A 走完 A 链表后从 B 链表头开始，指针 B 走完 B 链表后从 A 链表头开始。两者在交点处恰好走了相同的步数。

所以这题的关键词不是"找交点"，而是"走完自己的再走对方的，总步数一样必然相遇"。

## 💻 Java 题解

### 解法 1：双指针交叉走

pA 从 headA 出发走完 A 转 B，pB 从 headB 出发走完 B 转 A。如果没有交点，两指针最终同时为 null。

- ⏱ 时间复杂度：`O(m+n)`
- 💾 空间复杂度：`O(1)`

```java
public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        if (headA == null || headB == null) return null;
        ListNode pA = headA, pB = headB;
        while (pA != pB) {
            pA = pA == null ? headB : pA.next;
            pB = pB == null ? headA : pB.next;
        }
        return pA;
    }
}
```

**关键点：**
- pA 走完 A 转 B
- pB 走完 B 转 A
- 走的总路程相同，必在交点相遇

**执行步骤：**
1. pA=headA, pB=headB
1. pA 到 null 转 headB
1. pB 到 null 转 headA
1. pA==pB 时即为交点


## ⚠️ 易错点

- 跳转条件是 `a == null` 时跳到 `headB`，不是 `a.next == null`。要走到 null 才跳，而不是走到最后一个节点就跳。
- 如果没有交点，两个指针会在同一轮都走到 `null`，循环结束，返回 `null`。
- 不要修改链表结构，也不要用额外空间（哈希表存节点虽然能过，但空间是 O(n)）。

- 无交点时两指针最终都为 null（同时走完 m+n）

- 不要修改链表结构

## 📊 复杂度分析

- 时间复杂度：`O(m + n)`。两个指针各走 `m + n` 步。
- 空间复杂度：`O(1)`。只用了两个指针。

e:T13db,# 189. 轮转数组

> 难度：**Medium** | 分类：普通数组 | 标签：数组, 数学, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/rotate-array/](https://leetcode.cn/problems/rotate-array/)

## 📝 题目描述

给你一个数组 `nums` 和一个整数 `k`，把数组向右轮转 `k` 步。比如 `nums = [1,2,3,4,5,6,7]`，`k = 3`，结果为 `[5,6,7,1,2,3,4]`。

要求原地修改，空间复杂度 `O(1)`。

- 题号：189
- 题目名：轮转数组
- 难度：中等
- 题型：数组
- 为什么选这题：它的"三次反转"解法极为优雅——不需要额外空间，只靠三次翻转就完成轮转，是原地操作的经典技巧。

## 🧠 思考路径

暴力法：每次把最后一个元素移到最前面，重复 k 次。时间 `O(n × k)`，太慢。

三次反转法：观察轮转后的结果，前 `k` 个元素是原数组末尾的 `k` 个元素，后面是原数组前面的 `n - k` 个元素。所以：

1. 反转整个数组。
2. 反转前 `k` 个元素。
3. 反转后 `n - k` 个元素。

为什么有效？第一步反转后，原来的"末尾 k 个"变成了"前 k 个"，但顺序反了。第二步把它们再反转回来，顺序就对了。第三步同理处理后 `n - k` 个。

## 💻 Java 题解

### 解法 1：三次翻转（最优）

先翻转整个数组，再翻转前 k 个，再翻转后 n-k 个。k 需取模。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public void rotate(int[] nums, int k) {
        k %= nums.length;
        reverse(nums, 0, nums.length - 1);
        reverse(nums, 0, k - 1);
        reverse(nums, k, nums.length - 1);
    }
    private void reverse(int[] nums, int l, int r) {
        while (l < r) {
            int t = nums[l]; nums[l] = nums[r]; nums[r] = t;
            l++; r--;
        }
    }
}
```

**关键点：**
- k %= n 处理 k>n
- 三次翻转：全翻+前k翻+后n-k翻
- O(1) 空间

**执行步骤：**
1. k 取模
1. 翻转整个数组
1. 翻转前 k 个
1. 翻转剩余部分


## ⚠️ 易错点

- 必须先 `k %= n`，否则 `k` 可能大于数组长度。
- 三次反转的顺序和区间不能搞错：先整体，再前 k，再后 n-k。
- 反转函数的参数是闭区间 `[l, r]`。

- 忘记 k %= nums.length

- 翻转的边界不要搞错

## 📊 复杂度分析

- 时间复杂度：`O(n)`，每个元素最多被交换两次。
- 空间复杂度：`O(1)`，原地操作。

这段记忆的关键是：整体反转 + 两段分别反转 = 轮转。

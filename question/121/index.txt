e:T1765,# 121. 买卖股票的最佳时机

> 难度：**Easy** | 分类：贪心算法 | 标签：数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/](https://leetcode.cn/problems/best-time-to-buy-and-sell-stock/)

## 📝 题目描述

给你一个数组 `prices`，`prices[i]` 表示第 `i` 天的股票价格。你只能选择某一天买入，并在之后的某一天卖出。计算你能获得的最大利润。如果不能盈利，返回 0。

- 题号：121
- 题目名：买卖股票的最佳时机
- 难度：简单
- 题型：贪心
- 为什么选这题：它是股票系列的第一题，用最简单的"维护最小值"思路就能解决，是理解"一次遍历找最优"的经典模板。

## 🧠 思考路径

暴力方法是枚举每一天买入，再枚举后面每一天卖出，取最大差值。时间复杂度 `O(n²)`。

优化思路：遍历数组时，维护"到当前为止的最低价格"作为买入点，然后计算"如果今天卖出能赚多少"。如果今天的价格比之前的最低价还低，就更新最低价。不需要回头再看之前的价格，因为之前的价格已经不可能产生更大的利润了。

这就是贪心的核心：在每个位置都做出"目前为止最优"的决策。

## 💻 Java 题解

### 解法 1：贪心（一次遍历）

维护历史最低价minPrice。每天计算profit=price-minPrice，更新最大利润。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int maxProfit(int[] prices) {
        int minPrice = Integer.MAX_VALUE, maxProfit = 0;
        for (int price : prices) {
            minPrice = Math.min(minPrice, price);
            maxProfit = Math.max(maxProfit, price - minPrice);
        }
        return maxProfit;
    }
}
```

**关键点：**
- 维护历史最低价
- 每天算卖出的利润
- 更新最大值

**执行步骤：**
1. 遍历每天价格
1. 更新最低价
1. 计算利润取max


## ⚠️ 易错点

- 必须先更新最低价，再算利润。如果反过来，可能会用当天的价格既买又卖，利润为 0。
- 不能盈利时返回 0，不是负数。题目允许"不交易"。
- `min_price` 初始化为正无穷，不是 0。否则第一次比较会把 0 当作最低价。

- minPrice初始为MAX_VALUE

## 📊 复杂度分析

- 时间复杂度：`O(n)`。一次遍历。
- 空间复杂度：`O(1)`。只用了两个变量。

e:T1cef,# 33. 搜索旋转排序数组

> 难度：**Medium** | 分类：二分查找 | 标签：数组, 二分查找

> 🔗 力扣链接：[https://leetcode.cn/problems/search-in-rotated-sorted-array/](https://leetcode.cn/problems/search-in-rotated-sorted-array/)

## 📝 题目描述

给你一个升序数组，它在某个未知的下标处被旋转了。比如 `[0,1,2,4,5,6,7]` 在下标 3 处旋转后变成 `[4,5,6,7,0,1,2]`。给你一个目标值 `target`，你要在 `O(log n)` 时间内找到它在数组中的下标。找不到就返回 `-1`。

数组中不存在重复元素。

- 题号：33
- 题目名：搜索旋转排序数组
- 难度：中等
- 题型：二分查找
- 为什么选这题：它是二分查找的经典变体——数组不再全局有序，而是被旋转过，需要先判断哪一半有序再决定搜索方向，是掌握"二分不是只能用在排好序的数组上"的关键题。

## 🧠 思考路径

普通的二分查找要求整个数组有序，但现在数组被"折了一下"，变成了两段有序的区间。

关键观察：虽然整体无序，但取中点 `mid` 之后，左半边和右半边**必然有一半是有序的**。这是因为旋转点只有一个，它最多破坏了一半的有序性。

所以每次二分时，先判断左半边 `[left, mid]` 是否有序（条件是 `nums[left] <= nums[mid]`）。如果有序，再看 `target` 是否落在这个有序区间里，是就在左半边找，否则去右半边。如果左半边无序，那右半边一定有序，同理判断 `target` 是否在右半边。

这样每次都能排除一半，仍然是 `O(log n)`。

## 💻 Java 题解

### 解法 1：二分（判断有序半段）

每次二分时，判断左半还是右半有序。然后判断target是否在有序的那一半里。

- ⏱ 时间复杂度：`O(log n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int search(int[] nums, int target) {
        int left = 0, right = nums.length - 1;
        while (left <= right) {
            int mid = left + (right - left) / 2;
            if (nums[mid] == target) return mid;
            // 左半有序
            if (nums[left] <= nums[mid]) {
                if (target >= nums[left] && target < nums[mid]) right = mid - 1;
                else left = mid + 1;
            }
            // 右半有序
            else {
                if (target > nums[mid] && target <= nums[right]) left = mid + 1;
                else right = mid - 1;
            }
        }
        return -1;
    }
}
```

**关键点：**
- nums[left]<=nums[mid] 判断左半有序
- target在有序半段则搜该半
- 否则搜另一半

**执行步骤：**
1. 二分mid
1. 判断哪半有序
1. 判断target在哪个范围
1. 缩小搜索范围


## ⚠️ 易错点

- 判断左半边有序时用 `nums[left] <= nums[mid]`，注意是 `<=` 不是 `<`，因为当 `left == mid` 时（区间只剩两个元素），左半边只有一个元素，也算有序。
- 判断 `target` 在哪个范围时，边界条件要写清楚：左半边是 `nums[left] <= target < nums[mid]`，右半边是 `nums[mid] < target <= nums[right]`，注意哪边包含等号。
- 不要忘了找不到时返回 `-1`。

- nums[left]<=nums[mid] 用<=处理left==mid的情况

- 边界用>=和<=

## 📊 复杂度分析

- 时间复杂度：`O(log n)`。每次排除一半，标准二分。
- 空间复杂度：`O(1)`。只用了几个指针变量。

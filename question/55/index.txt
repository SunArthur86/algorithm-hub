e:T1a41,# 55. 跳跃游戏

> 难度：**Medium** | 分类：贪心算法 | 标签：贪心, 数组, 动态规划

> 🔗 力扣链接：[https://leetcode.cn/problems/jump-game/](https://leetcode.cn/problems/jump-game/)

## 📝 题目描述

你站在数组的第一个位置，每个位置的数字表示你从这里"最多能跳几步"。问：能不能跳到最后一个位置？能就返回 `true`，不能就返回 `false`。

- 题号：55
- 题目名：跳跃游戏
- 难度：中等
- 题型：贪心
- 为什么选这题：贪心策略"维护最远可达位置"的思想简洁有力，是贪心算法最经典的入门范例。

## 🧠 思考路径

最朴素的想法：从起点开始，每个位置都尝试跳 1 步、2 步……一直到最大步数，用 BFS 或 DFS 搜索所有路径，看有没有一条能到终点。但这样最坏是指数级的。

换个角度：我不需要知道"怎么跳"，只需要知道"最远能跳到哪"。从左往右扫描，维护一个变量 `farthest` 表示当前能到达的最远位置。每到一个位置 `i`，先看 `i` 是否在可达范围内（`i <= farthest`），如果是就更新 `farthest = max(farthest, i + nums[i])`。

如果扫描过程中 `farthest` 已经超过了最后一个位置，直接返回 `true`。如果扫描完了还没到，或者某个位置 `i` 超过了 `farthest`（说明中间断了，跳不过去），就返回 `false`。

这就是贪心：每一步都不做具体决策，只维护"目前能到的最远处"。

## 💻 Java 题解

### 解法 1：贪心（最远可达）

遍历时维护maxReach=i+nums[i]。如果i>maxReach则失败。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public boolean canJump(int[] nums) {
        int maxReach = 0;
        for (int i = 0; i < nums.length; i++) {
            if (i > maxReach) return false;
            maxReach = Math.max(maxReach, i + nums[i]);
        }
        return true;
    }
}
```

**关键点：**
- maxReach=i+nums[i]
- i>maxReach→无法到达

**执行步骤：**
1. 遍历
1. 更新maxReach
1. i超过maxReach返回false


## ⚠️ 易错点

- 检查 `i > farthest` 必须在更新 `farthest` 之前：先判断当前位置是否可达，再更新最远范围。
- 不要把 `farthest` 和 `i` 搞混：`farthest` 是"目前能到的最远位置"，`i + nums[i]` 是"从当前位置跳出去能到的位置"。
- `nums[i]` 可以是 `0`：跳到这个位置就"停住"了，如果此时 `farthest` 也等于 `i` 且 `i` 不是终点，就跳不动了。

- 先判断i>maxReach再更新

## 📊 复杂度分析

- 时间复杂度：`O(n)`。只遍历一次数组，每个位置做 O(1) 操作。
- 空间复杂度：`O(1)`。只用了 `farthest` 一个变量。

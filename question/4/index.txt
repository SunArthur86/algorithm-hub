e:T1e4d,# 4. 寻找两个正序数组的中位数

> 难度：**Hard** | 分类：二分查找 | 标签：数组, 二分查找, 分治

> 🔗 力扣链接：[https://leetcode.cn/problems/median-of-two-sorted-arrays/](https://leetcode.cn/problems/median-of-two-sorted-arrays/)

## 📝 题目描述

给你两个已经排好序的数组 `nums1` 和 `nums2`，找出它们合并后的中位数。要求时间复杂度 `O(log(m+n))`。

如果是奇数个元素，返回中间那个；如果是偶数个，返回中间两个的平均值。

- 题号：4
- 题目名：寻找两个正序数组的中位数
- 难度：困难
- 题型：二分查找
- 为什么选这题：它是二分查找应用的天花板级题目，训练"在两个有序结构上做分割线"的思维方式。

## 🧠 思考路径

最朴素的做法是把两个数组合并再排序，直接取中位数。但合并需要 `O(m+n)` 的时间，不满足题目的 `O(log(m+n))` 要求。

要达到对数级别，必须用二分。关键观察是：中位数的本质是一条"分割线"——把合并后的数组分成两半，左半边最大的数 ≤ 右半边最小的数，且左右两边的元素个数相等（或差一个）。

既然两个数组各自有序，我们可以在较短的数组上二分找分割线的位置。设 `nums1` 的分割点为 `i`，`nums2` 的分割点为 `j = (m+n+1)/2 - i`，只要保证 `nums1[i-1] ≤ nums2[j]` 且 `nums2[j-1] ≤ nums1[i]`，就找到了正确的分割线。

## 💻 Java 题解

### 解法 1：二分切分（O(log(min(m,n)))）

在较短数组上二分切分位置i，另一个数组切分位置j=(m+n+1)/2-i。检查边界条件。

- ⏱ 时间复杂度：`O(log(min(m,n)))`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public double findMedianSortedArrays(int[] nums1, int[] nums2) {
        if (nums1.length > nums2.length) return findMedianSortedArrays(nums2, nums1);
        int m = nums1.length, n = nums2.length;
        int left = 0, right = m;
        while (left <= right) {
            int i = (left + right) / 2, j = (m + n + 1) / 2 - i;
            int left1 = i == 0 ? Integer.MIN_VALUE : nums1[i-1];
            int right1 = i == m ? Integer.MAX_VALUE : nums1[i];
            int left2 = j == 0 ? Integer.MIN_VALUE : nums2[j-1];
            int right2 = j == n ? Integer.MAX_VALUE : nums2[j];
            if (left1 <= right2 && left2 <= right1) {
                if ((m + n) % 2 == 1) return Math.max(left1, left2);
                return (Math.max(left1, left2) + Math.min(right1, right2)) / 2.0;
            } else if (left1 > right2) right = i - 1;
            else left = i + 1;
        }
        return 0.0;
    }
}
```

**关键点：**
- 在较短数组上二分
- j=(m+n+1)/2-i 保证左半>=右半
- left1<=right2 && left2<=right1 为正确切分

**执行步骤：**
1. 二分切分位置i
1. 计算j
1. 检查边界条件
1. 奇数取max(left)，偶数取avg


## ⚠️ 易错点

- 必须在较短的数组上做二分。否则 `j = totalLeft - i` 可能为负数。
- 边界处理：`i = 0` 时 `nums1` 左半为空，`i = m` 时 `nums1` 右半为空，需要用 `-inf` 和 `+inf` 填充。
- 奇偶判断：`(m + n + 1) // 2` 保证了奇数时左半多一个元素，中位数直接取左半最大值。
- 二分循环中移动方向：`nums1LeftMax > nums2RightMin` 说明 `nums1` 分多了，往左移；反之往右移。

- 确保nums1是较短数组

- 边界用MIN_VALUE/MAX_VALUE处理

## 📊 复杂度分析

- 时间复杂度：`O(log(min(m, n)))`。只在较短数组上做二分。
- 空间复杂度：`O(1)`。只用了常数个变量。

## 🏢 实际业务应用

- 双数据源的实时中位数统计：比如两个机房各自上报延迟数据，需要实时计算全局中位数，不合并数组直接查中位数，避免数据搬迁。
- 分位数计算：将中位数的分割线思路推广到任意分位数（P90、P95、P99），监控系统中的延迟指标统计常用此方法。
- 数据库的排序归并优化：两个有序索引段的合并查询中，快速定位中间位置的记录。

e:T17b3,# 136. 只出现一次的数字

> 难度：**Easy** | 分类：技巧 | 标签：位运算, 数组

> 🔗 力扣链接：[https://leetcode.cn/problems/single-number/](https://leetcode.cn/problems/single-number/)

## 📝 题目描述

给你一个非空整数数组，其中除了一个元素只出现一次外，其余每个元素都出现了两次。找出那个只出现一次的元素。

要求线性时间、不使用额外空间。重点不在"用哈希表计数"，而在"用位运算一次扫描搞定"。

- 题号：136
- 题目名：只出现一次的数字
- 难度：简单
- 题型：位运算
- 为什么选这题：它是异或运算最经典的应用题，一句话就能让你记住"相同为 0，不同为 1"的含义。

## 🧠 思考路径

最直觉的做法是用哈希表统计每个数出现的次数，但题目要求 `O(1)` 空间，哈希表就不行了。

换个角度：异或运算 `XOR` 有三个关键性质：
1. 一个数和自己异或结果是 0：`a ^ a = 0`
2. 一个数和 0 异或结果是自身：`a ^ 0 = a`
3. 异或满足交换律和结合律

这意味着，如果把数组里所有数都异或起来，成对出现的数会互相抵消变成 0，最后只剩下那个只出现一次的数。

比如 `[4, 1, 2, 1, 2]`：`4 ^ 1 ^ 2 ^ 1 ^ 2 = 4 ^ (1 ^ 1) ^ (2 ^ 2) = 4 ^ 0 ^ 0 = 4`。

所以这题的关键词不是"找唯一"，而是"用异或把成对的数消掉"。

## 💻 Java 题解

### 解法 1：异或

所有数字异或，出现两次的互相抵消为0，最后剩下的就是出现一次的。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public int singleNumber(int[] nums) {
        int result = 0;
        for (int num : nums) result ^= num;
        return result;
    }
}
```

**关键点：**
- 异或性质：a^a=0
- 0^a=a
- 交换律和结合律

**执行步骤：**
1. 遍历所有数
1. 异或累加
1. 返回结果


## ⚠️ 易错点

- `result` 初始化为 0，不是数组的第一个元素（虽然从第一个元素开始也可以，但 0 更通用，因为 `x ^ 0 = x`）。
- 异或运算不关心顺序，所以即使数组无序也能正确消对。
- 这个方法只适用于"其余元素恰好出现两次"的情况。如果出现三次，需要用 137 题的方法。

- 初始值必须为0

## 📊 复杂度分析

- 时间复杂度：`O(n)`。遍历一次数组。
- 空间复杂度：`O(1)`。只用了一个变量。

## 🏢 实际业务应用

- 数据校验中，快速找出奇偶不匹配的数据项。比如分布式系统里，两个副本互相异或，结果为 0 表示一致。

> 难度：**Easy** | 分类：双指针 | 标签：数组, 双指针

> 🔗 力扣链接：[https://leetcode.cn/problems/move-zeroes/](https://leetcode.cn/problems/move-zeroes/)

## 📝 题目描述

给你一个数组，把所有的 0 移到数组末尾，同时保持非零元素的原始顺序。原地操作。

重点不在"怎么排序"，而在"怎么用 O(n) 时间、O(1) 空间一次到位"。

- 题号：283
- 题目名：移动零
- 难度：简单
- 题型：双指针
- 为什么选这题：它是双指针"快慢指针"最经典的应用，代码简洁，思想通用。

## 🧠 思考路径

最直接的想法是用额外数组，把非零元素复制过去再补零。但要求原地操作。

用双指针：慢指针 `slow` 指向"下一个非零元素应该放的位置"，快指针 `fast` 遍历数组。每遇到一个非零元素，就把它放到 `slow` 的位置，`slow` 后移。遍历结束后，从 `slow` 到末尾全部填零。

更简洁的写法：遇到非零就和 `slow` 交换（即使 `slow == fast` 也没关系），这样一步到位，不需要最后补零。

所以这题的关键词不是"移动零"，而是"快指针找非零，慢指针标位置，交换"。

## 💻 Java 题解

### 解法 1：双指针一次遍历

慢指针指向下一个非零位置。快指针遍历数组，遇到非零就交换到慢指针位置，慢指针前进。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public void moveZeroes(int[] nums) {
        int slow = 0;
        for (int fast = 0; fast < nums.length; fast++) {
            if (nums[fast] != 0) {
                int temp = nums[slow];
                nums[slow] = nums[fast];
                nums[fast] = temp;
                slow++;
            }
        }
    }
}
```

**关键点：**
- 原地操作 O(1) 空间
- 交换而非覆盖——保持非零元素顺序
- 慢指针始终指向下一个非零应放的位置

**执行步骤：**
1. slow=0, fast=0
1. fast 遍历数组
1. 遇到非零：交换 nums[slow] 和 nums[fast]，slow++
1. fast 继续前进


## ⚠️ 易错点

- 交换写法比"先移动非零再补零"更简洁，且保持稳定性（非零元素相对顺序不变）。
- `slow == fast` 时交换是自己和自己交换，不影响结果。
- 不要用覆盖写法（先移后补零），容易出错。

- 不能用覆盖法后补零——题目要求原地修改

- 交换确保非零元素顺序不变

## 📊 复杂度分析

- 时间复杂度：`O(n)`。
- 空间复杂度：`O(1)`。

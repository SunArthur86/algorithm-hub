e:T1b0f,# 128. 最长连续序列

> 难度：**Medium** | 分类：哈希 | 标签：并查集, 数组, 哈希表

> 🔗 力扣链接：[https://leetcode.cn/problems/longest-consecutive-sequence/](https://leetcode.cn/problems/longest-consecutive-sequence/)

## 📝 题目描述

给你一个未排序的整数数组 `nums`，找出数字连续的最长序列的长度。要求算法的时间复杂度为 `O(n)`。

"连续序列"是指整数之间差为 1 的序列，比如 `[3, 4, 5, 6]`。不要求在原数组中连续排列。

- 题号：128
- 题目名：最长连续序列
- 难度：中等
- 题型：哈希
- 为什么选这题：它是哈希集合"判存在"的经典应用，核心技巧是"只从序列起点开始扩展"，面试中考得很多。

## 🧠 思考路径

排序后找连续序列是最直观的思路，但排序需要 `O(n log n)`，不满足题目要求。

用哈希集合可以在 `O(1)` 时间内判断一个数是否存在。关键问题是：如何避免重复扫描同一段序列？

答案是：**只从序列的起点开始扩展**。如果一个数 `num` 的前驱 `num - 1` 不在集合中，说明 `num` 是某个连续序列的起点。从这个起点开始，逐个检查 `num + 1, num + 2, ...` 是否存在，统计长度。如果 `num - 1` 存在，就跳过——因为这段序列会从更小的起点被完整扫描。

这样每个数最多被访问两次（一次判断是否是起点，一次在某个序列中被扩展），总时间 `O(n)`。

## 💻 Java 题解

### 解法 1：HashSet 枚举序列起点

把所有数放入 HashSet。只从序列起点（num-1 不在集合中）开始向后枚举，保证 O(n)。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(n)`

```java
class Solution {
    public int longestConsecutive(int[] nums) {
        Set<Integer> set = new HashSet<>();
        for (int n : nums) set.add(n);
        int maxLen = 0;
        for (int n : set) {
            // 只有序列起点才开始计数
            if (!set.contains(n - 1)) {
                int cur = n, len = 1;
                while (set.contains(cur + 1)) {
                    cur++; len++;
                }
                maxLen = Math.max(maxLen, len);
            }
        }
        return maxLen;
    }
}
```

**关键点：**
- 只从序列起点开始计数
- 每个元素最多被访问两次 → O(n)
- set.contains(n-1) 判断是否为起点

**执行步骤：**
1. 全部元素放入 HashSet
1. 遍历集合中每个元素
1. 若 num-1 不在集合中 → 这是序列起点
1. 从起点向后数连续序列长度
1. 更新最大值


## ⚠️ 易错点

- 判断"是否是起点"的条件是 `num - 1 not in set`，不是 `num + 1 not in set`。前者保证只从序列最左端开始扩展。
- 遍历集合而不是数组，可以避免重复元素导致重复计算。
- 空数组的最大长度是 0，不是 1。
- 不要对数组排序，那样是 `O(n log n)`。

- 不要对每个元素都向后枚举——会变成 O(n²)

- 起点判断条件是 n-1 不在集合中

## 📊 复杂度分析

- 时间复杂度：`O(n)`。虽然有两层循环，但内层循环总共执行次数不超过 `n`，因为每个数字只会在一个序列中被扩展一次。
- 空间复杂度：`O(n)`。哈希集合存储所有数字。

e:T1bc3,# 240. 搜索二维矩阵 II

> 难度：**Medium** | 分类：矩阵 | 标签：数组, 二分查找, 分治, 矩阵

> 🔗 力扣链接：[https://leetcode.cn/problems/search-a-2d-matrix-ii/](https://leetcode.cn/problems/search-a-2d-matrix-ii/)

## 📝 题目描述

给你一个 `m × n` 的矩阵，满足每行从左到右递增、每列从上到下递增。判断目标值 `target` 是否在矩阵中。

重点不在"怎么遍历矩阵"，而在"怎么利用行列有序性，每一步排除一整行或一整列"。

- 题号：240
- 题目名：搜索二维矩阵 II
- 难度：中等
- 题型：二分查找
- 为什么选这题：它展示了"从右上角开始搜索"这个经典技巧——利用矩阵的行列有序性，每一步都能排除一行或一列，是"搜索空间逐步缩小"的完美示范。

## 🧠 思考路径

暴力遍历是 `O(m * n)`，但完全没有利用矩阵的有序性。

如果从左上角（最小值）开始，右边和下边都比它大，不知道该往哪走。如果从右下角（最大值）开始，左边和上边都比它小，同样不确定。

关键发现：从右上角开始看——当前元素左边的都比它小，下面的都比它大。所以：
- 如果当前元素等于 `target`，找到了。
- 如果当前元素大于 `target`，那这一列下面的数都更大，不可能了，排除这一列（`j--`）。
- 如果当前元素小于 `target`，那这一行左边的数都更小，不可能了，排除这一行（`i++`）。

每一步都排除一行或一列，最多走 `m + n` 步。也可以从左下角开始，逻辑对称。

这题的关键词不是"二分查找"，而是"从右上角出发，利用行列有序性每步排除一行或一列"。

## 💻 Java 题解

### 解法 1：右上角搜索（Z形查找）

从右上角出发，当前值比 target 大就左移（减小），比 target 小就下移（增大）。

- ⏱ 时间复杂度：`O(m+n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public boolean searchMatrix(int[][] matrix, int target) {
        int m = matrix.length, n = matrix[0].length;
        int row = 0, col = n - 1; // 从右上角开始
        while (row < m && col >= 0) {
            if (matrix[row][col] == target) return true;
            else if (matrix[row][col] > target) col--;
            else row++;
        }
        return false;
    }
}
```

**关键点：**
- 右上角是分界点：左小右大下大
- 每次排除一行或一列
- O(m+n) 时间

**执行步骤：**
1. 从右上角 (0,n-1) 开始
1. 等于 target 返回 true
1. 大于 target 左移 col--
1. 小于 target 下移 row++


## ⚠️ 易错点

- 起点必须是右上角或左下角，不能是左上角或右下角——左上角两个方向都是增大，无法决定走哪边。
- 矩阵为空时需要提前判断，否则访问 `matrix[0]` 会报错。
- 每一步只移动一个方向（行或列），不要同时移动。

- 不能从左上角或右下角开始——两个方向同增/同减

- 左下角也可以作为起点

## 📊 复杂度分析

- 时间复杂度：`O(m + n)`。最多走 `m` 步向下和 `n` 步向左。
- 空间复杂度：`O(1)`。只用两个指针。

## 🏢 实际业务应用

- 数据库索引中，多列联合索引的搜索可以用类似的"排除法"策略——每一步都能剪枝一部分搜索空间。
- 电子表格中在行列都有序的数据区域里查找某个值，从右上角开始是最自然的交互方式。
- 游戏地图中，如果地形数据按行列有序排列（比如高度从西北到东南递增），查找特定高度的地块可以用这个方法。

e:T1bbb,# 763. 划分字母区间

> 难度：**Medium** | 分类：贪心算法 | 标签：贪心, 哈希表, 双指针, 字符串

> 🔗 力扣链接：[https://leetcode.cn/problems/partition-labels/](https://leetcode.cn/problems/partition-labels/)

## 📝 题目描述

给你一个字符串 `s`，要把它分成尽可能多的片段，使得同一个字母不会出现在两个不同的片段里。返回每个片段的长度。

换句话说，每种字母只能出现在一个片段中，问最多能分几段、每段多长。

- 题号：763
- 题目名：划分字母区间
- 难度：中等
- 题型：贪心
- 为什么选这题：它是贪心策略的经典体现——"先把每条蛇的尾巴摸清，再一刀一刀切"。

## 🧠 思考路径

第一反应可能是：遇到一个字母就往前看它最后出现在哪里。但这样每个字母都要回溯，效率不高。

换一个角度：先做一遍预处理，记录每个字母最后出现在哪个位置。这一步 `O(n)` 搞定。

然后再从头遍历，维护一个"当前片段的右边界"。每走到一个字符，就把右边界更新为"这个字符最后出现位置"和"当前右边界"中更大的那个。当遍历到了右边界时，说明当前片段里所有字母的最后一次出现都在这个位置或之前了——可以切一刀。

这就是贪心：在能切的地方尽量切，每次切都保证当前片段是合法的最短片段。

## 💻 Java 题解

### 解法 1：贪心（记录最后位置）

先记录每个字母最后出现位置。遍历时维护当前片段的结束边界，到达边界时切割。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public List<Integer> partitionLabels(String s) {
        int[] last = new int[26];
        for (int i = 0; i < s.length(); i++) last[s.charAt(i) - 'a'] = i;
        List<Integer> result = new ArrayList<>();
        int start = 0, end = 0;
        for (int i = 0; i < s.length(); i++) {
            end = Math.max(end, last[s.charAt(i) - 'a']);
            if (i == end) { result.add(end - start + 1); start = i + 1; }
        }
        return result;
    }
}
```

**关键点：**
- 先记录每个字母最后出现位置
- end=max(end, last[char])
- i==end时切割

**执行步骤：**
1. 预记录最后位置
1. 遍历更新end
1. 到达end切割


## ⚠️ 易错点

- 必须先完成"记录最后位置"的预处理，再开始分段。如果边遍历边分，可能会漏掉后面出现的同字母。
- 判断"可以切一刀"的条件是 `i == end`，不是 `last[c] == i`。因为 `end` 是当前片段内所有字母的最后位置的最大值。
- 每次切完要更新 `start`，否则片段长度会算错。

- 必须先遍历一次记录所有最后位置

## 📊 复杂度分析

- 时间复杂度：`O(n)`。两遍扫描，每遍都是线性的。
- 空间复杂度：`O(1)`。只有 26 个字母，最后位置数组大小固定。

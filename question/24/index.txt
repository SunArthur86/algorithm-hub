e:T1ccd,# 24. 两两交换链表中的节点

> 难度：**Medium** | 分类：链表 | 标签：递归, 链表

> 🔗 力扣链接：[https://leetcode.cn/problems/swap-nodes-in-pairs/](https://leetcode.cn/problems/swap-nodes-in-pairs/)

## 📝 题目描述

给你一条链表，每相邻的两个节点交换位置。比如原来是 `1→2→3→4`，交换后变成 `2→1→4→3`。

注意是"交换节点"而不是"只交换值"，虽然交换值也能过测试，但面试官要考的是指针操作。

- 题号：24
- 题目名：两两交换链表中的节点
- 难度：中等
- 题型：链表
- 为什么选这题：它是链表指针操作的经典练习，既可以用递归优雅解决，也可以用迭代练手感，是理解"指针重连"这个核心动作的最好切入点。

## 🧠 思考路径

先看最简单的情况：只有两个节点 `1→2`。交换后变成 `2→1`，就是把 2 拉到前面，1 接到后面。这很容易。

再看四个节点 `1→2→3→4`。先处理前两个，把 1→2 变成 2→1，然后 1 的 next 应该指向后面的交换结果。后面的 3→4 变成 4→3，所以 1 接上 4→3 就好。这就是递归的影子：先把后面的交换好，再把前面的接上去。

迭代思路也一样。用三个指针：`prev` 指向前一个节点（或 dummy），`a` 和 `b` 是要交换的一对。每次把 `prev→a→b→b.next` 变成 `prev→b→a→b.next`，然后 `prev` 移到 `a` 的位置，继续处理下一对。

关键操作就三步：`prev.next = b`、`a.next = b.next`、`b.next = a`。顺序不能乱。

## 💻 Java 题解

### 解法 1：迭代交换

用 dummy 头。每次取两个节点交换，prev 指向交换后的第一个节点。

- ⏱ 时间复杂度：`O(n)`
- 💾 空间复杂度：`O(1)`

```java
class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummy = new ListNode(0, head);
        ListNode prev = dummy;
        while (prev.next != null && prev.next.next != null) {
            ListNode first = prev.next, second = prev.next.next;
            first.next = second.next;
            second.next = first;
            prev.next = second;
            prev = first;
        }
        return dummy.next;
    }
}
```

**关键点：**
- 三个指针：prev, first, second
- second.next=first 完成交换
- prev 移到 first（交换后的第二个）

**执行步骤：**
1. 取 first 和 second
1. first.next=second.next
1. second.next=first
1. prev.next=second
1. prev=first


## ⚠️ 易错点

- 指针重连的顺序：必须先 `a.next = b.next`（保存后半段），再 `b.next = a`。如果先做 `b.next = a`，那 `b.next` 原来指向的后半段就丢了。
- 奇数个节点时，最后一个不用交换。循环条件 `prev.next and prev.next.next` 自动处理了这种情况，但写递归终止条件时容易漏掉 `not head.next` 这个判断。
- 忘记用 `dummy` 节点。如果直接操作 `head`，第一对交换后头节点就变了，最后返回的还是原来的 `head`，答案就错了。

- 指针操作顺序错误导致环

- prev 要移动到交换后的第二个节点

## 📊 复杂度分析

- 时间复杂度：`O(n)`，每个节点被访问一次。
- 空间复杂度：`O(1)`（迭代版本）。递归版本是 `O(n)` 的递归栈空间。

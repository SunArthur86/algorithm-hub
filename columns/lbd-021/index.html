函数本⾝的复杂度就是忽略递归部分的复杂度，这⾥ dp 函数中⽤了⼀个 ⼆分搜索，所以函数本⾝的复杂度是 O(logN)。 ⼦问题个数也就是不同状态组合的总数，显然是两个状态的乘积，也就是 O(KN)。 所以算法的总时间复杂度是 O(KNlogN), 空间复杂度 O(KN)。效率上⽐之 前的算法 O(KN^2) 要⾼效⼀些。 重新定义状态转移 前⽂「不同定义有不同解法」就提过，找动态规划的状态转移本就是⻅仁⻅ 智，⽐较⽞学的事情，不同的状态定义可以衍⽣出不同的解法，其解法和复 杂程度都可能有巨⼤差异。这⾥就是⼀个很好的例⼦。 再回顾⼀下我们之前定义的 dp 数组含义： def dp(k, n) -> int
